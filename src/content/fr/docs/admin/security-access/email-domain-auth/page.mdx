# Authentification par domaine d'e-mail

*Disponible sur les plans **Team** et supérieurs*

L'authentification par domaine d'e-mail contrôle comment les nouveaux utilisateurs rejoignent votre Organisation. Lorsqu'elle est activée, toute personne qui s'inscrit ou se connecte avec une **adresse e-mail correspondant à votre domaine vérifié** est automatiquement ajoutée à votre Organisation.

<SectionTab as="h1" sectionId="how-it-works">Fonctionnement</SectionTab>

- Les utilisateurs qui s'inscrivent ou se connectent avec un e-mail correspondant à votre domaine vérifié sont **automatiquement ajoutés** à votre Organisation.
- Les utilisateurs **créent et gèrent toujours leurs propres identifiants** (e-mail et mot de passe). Cette fonctionnalité contrôle l'appartenance à l'Organisation, pas l'authentification.
- Pour une authentification par authentification unique **sans identifiants séparés**, utilisez plutôt le [SSO](/admin/security-access/sso) (plans Enterprise).

<SectionTab as="h1" sectionId="prerequisites">Prérequis</SectionTab>

Vous devez **vérifier la propriété de votre domaine** avant d'activer cette fonctionnalité. La vérification du domaine utilise un enregistrement DNS TXT. Consultez [vérification du domaine](/admin/security-access/sso#domain-verification) pour les étapes complètes de configuration.

<SectionTab as="h1" sectionId="enabling">Activer l'authentification par domaine d'e-mail</SectionTab>

1. Ouvrez les paramètres <AppLink href="https://admin.mistral.ai" app="admin">Accès</AppLink>.
2. Dans la section **Authentification**, cliquez sur **Activer l'authentification par domaine d'e-mail**.
3. Confirmez en cliquant sur **Activer l'authentification par domaine d'e-mail** dans la boîte de dialogue.

La page Accès se met à jour pour indiquer que la fonctionnalité est active.

<SectionTab as="h1" sectionId="disabling">Désactiver l'authentification par domaine d'e-mail</SectionTab>

Vous pouvez désactiver l'authentification par domaine d'e-mail à tout moment depuis la page de paramètres **Accès**.

- Les nouveaux utilisateurs avec des e-mails de domaine correspondants ne seront plus ajoutés automatiquement.
- Les membres existants ne sont pas affectés.
- Vous devrez inviter manuellement les nouveaux utilisateurs, sauf si le [SSO](/admin/security-access/sso) est configuré.

:::caution
Si vous réactivez la fonctionnalité ultérieurement, vous devrez recommencer le processus de configuration.
:::
