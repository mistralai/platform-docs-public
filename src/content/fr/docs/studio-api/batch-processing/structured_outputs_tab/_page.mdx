En tant que fichier JSONL, chaque ligne représente une requête vers l'endpoint API et le modèle.

<SectionTab as="h2" variant="secondary" sectionId="structured-outputs-explained">Explication</SectionTab>

Le corps de la requête suit le même format que l'endpoint que vous souhaitez utiliser pour votre traitement batch, à l'exception de l'identifiant du modèle qui sera fourni uniquement lors de la création du job pour lancer l'exécution du batch. Ci-dessous, nous fournissons un exemple de ligne avec sorties structurées :

<Tabs>
    <TabItem value="file" label="Exemple" default>

```py
{
    "custom_id": "2", # Un ID comme métadonnée qui sera renvoyé dans le fichier de sortie pour identifier la requête
    "body": { # Le corps de la requête
        "max_tokens": 256, # Max tokens correspondant au nombre maximal de tokens générés pour les complétions de chat
        "temperature": 0, # La température à utiliser pour l'échantillonnage
        "messages": [ # Les messages pour générer des complétions de chat avec sorties structurées
            {
                "role": "system", # Le rôle de l'auteur du message
                "content": "Extract the books information." # Le contenu du message
            },
            {
                "role": "user", # Le rôle de l'auteur du message
                "content": "I recently read To Kill a Mockingbird by Harper Lee." # Le contenu du message, ici nous voulons extraire les informations du livre
            }
        ],
        "response_format": { # Le format de réponse à utiliser pour la complétion de chat
            "type": "json_schema", # Définir le type sur json_schema pour utiliser des sorties structurées personnalisées, "json_mode" si vous voulez utiliser le mode JSON uniquement par défaut
            "json_schema": { # Le schéma json à utiliser pour les sorties structurées
                "schema": { # Le schéma correspondant
                    "properties": { # Les propriétés du schéma
                        "name": { # Le nom du livre
                            "title": "Name",
                            "type": "string"
                        },
                        "authors": { # Les auteurs du livre
                            "items": {
                                "type": "string"
                            },
                            "title": "Authors",
                            "type": "array"
                        }
                    },
                    "required": [ # Les propriétés requises
                        "name", 
                        "authors"
                    ],
                    "title": "Book", # Le titre du schéma
                    "type": "object", # Le type du schéma
                    "additionalProperties": false # Autoriser ou non des propriétés supplémentaires
                },
                "name": "book", # Le nom du schéma
                "strict": true # Utiliser ou non le mode strict, toujours recommandé de définir comme true pour les sorties structurées
            }
        }
    }
}
```
