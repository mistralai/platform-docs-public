# Échantillonnage : introduction aux paramètres d’échantillonnage

Nous allons aborder ici les paramètres d’échantillonnage qui influencent les réponses des modèles de langage à grande échelle (LLM). Ce guide présente des paramètres comme la **Température**, **N**, **Top P**, la **Presence Penalty** et la **Frequency Penalty**, et explique comment les régler. Que vous souhaitiez générer du contenu original ou obtenir des réponses précises, comprendre ces réglages est essentiel.

Voyons en détail chaque paramètre et comment adapter le comportement des LLM.

<details>

<summary><b>Nombre de complétions N</b></summary>

## N Completions

**N** représente le nombre de complétions à retourner pour chaque requête. Ce paramètre est utile lorsque vous souhaitez générer plusieurs réponses pour une seule entrée. Chaque complétion sera une réponse unique générée par le modèle, offrant une variété de résultats parmi lesquels choisir.

### Points clés

- **Réponses multiples** : En définissant `N` sur une valeur supérieure à 1, vous pouvez obtenir plusieurs réponses pour la même entrée.
- **Efficacité des coûts** : Les tokens d'entrée ne sont facturés qu'une seule fois, quel que soit le nombre de complétions demandées. Cela rend rentable l'exploration de différentes possibilités.

### Exemple

Voici un exemple d'utilisation du paramètre `N` dans l'API :

<Tabs groupId="sdk-version">
    <TabItem value="v1" label="V1" default>

```py
import os
from mistralai.client import Mistral

api_key = os.environ["MISTRAL_API_KEY"]
model = "ministral-3b-latest"

client = Mistral(api_key=api_key)

chat_response = client.chat.complete(
    model=model,
    messages=[
        {
            "role": "user",
            "content": "What is the best mythical creature? Answer with a single word.",
        },
    ],
    temperature = 1, # Increasing randomness and diversity of the output, this is required to be higher than 0 to have diverse outputs
    n = 10 # Number of completions
)

for i, choice in enumerate(chat_response.choices):
    print(choice.message.content)
```

<summary><b>Temperature</b></summary>

## Temperature

**Temperature** dans les grands modèles de langage (LLM) contrôle la diversité des résultats. Des valeurs plus faibles rendent le modèle plus déterministe, en se concentrant sur les réponses probables pour plus de précision. Des valeurs plus élevées augmentent la créativité et la diversité. Lors de la génération de texte, les LLM prédisent des tokens avec des probabilités associées à l'aide d'une fonction softmax. Temperature met à l'échelle ces probabilités : des températures plus élevées aplatissent la distribution, rendant les résultats plus variés, tandis que des températures plus faibles amplifient les différences, favorisant les tokens les plus probables.

**Temperature** affecte significativement la distribution de probabilité dans les LLM. Avec une Temperature de 0, le modèle génère toujours le token le plus probable, par exemple « **Dragon** ». Augmenter la Temperature à 0,2 introduit de la variabilité, permettant des tokens comme « **Un** » (comme dans « **Un**icorn »). Des augmentations supplémentaires révèlent des tokens plus divers : le troisième token pourrait toujours être « **Drag** » (pour « **Drag**on »), mais le quatrième pourrait commencer « **Peg**asus », et le cinquième, « **Phoenix** ». Des températures plus élevées rendent les tokens moins probables plus susceptibles d'être sélectionnés, augmentant ainsi la diversité des résultats du modèle.

## La meilleure Temperature

Il n'existe pas de Temperature universelle pour tous les cas d'usage, mais certaines directives peuvent vous aider à trouver la meilleure pour vos applications.

### Déterminisme

- **Exigences** : Tâches nécessitant des réponses cohérentes et précises, telles que les mathématiques, la classification, la santé ou le raisonnement.
- **Temperature** : Utilisez des valeurs très faibles, parfois non nulles pour ajouter une légère unicité.

Par exemple, un agent de classification doit utiliser une Temperature de 0 pour toujours choisir le meilleur token. Un assistant de chat mathématique peut utiliser des valeurs de Temperature très faibles pour éviter les répétitions tout en maintenant la précision.

### Créativité

- **Exigences** : Tâches nécessitant du texte diversifié et unique, comme le brainstorming, l'écriture de romans, la création de slogans ou le jeu de rôle.
- **Temperature** : Utilisez des valeurs élevées, mais évitez des températures excessivement élevées pour prévenir les résultats aléatoires et incohérents.

Considérez le compromis : des températures plus élevées augmentent la créativité mais peuvent diminuer la qualité et la précision.

<summary><b>Top P</b></summary>

# Top P

**Top P** est un paramètre qui limite les tokens considérés par un modèle de langage en fonction d'un seuil de probabilité. Il aide à se concentrer sur les tokens les plus probables, améliorant la qualité des résultats.

## Visualisation

Pour ces exemples, nous définissons d'abord la Temperature, puis appliquons un Top P de 50 %. Notez qu'une Temperature de 0 est déterministe, rendant Top P non pertinent dans ce cas.

Le processus est le suivant :

1. Appliquer la Temperature.
2. Utiliser Top P (0,5) pour ne conserver que les tokens les plus probables.
3. Ajuster les probabilités des tokens restants.

Top P garantit que seuls les tokens de haute qualité sont considérés, maintenant la qualité des résultats en excluant les tokens peu probables. Il est difficile d'équilibrer Temperature et Top P, il est donc recommandé d'en fixer un et d'ajuster l'autre. Vous devez toutefois expérimenter pour trouver les meilleurs paramètres pour votre cas d'usage !

### En résumé

1. **Rôle de Top P** : Top P limite les tokens considérés en fonction d'un seuil de probabilité, en se concentrant sur les tokens les plus probables pour améliorer la qualité de la sortie.
2. **Interaction avec Temperature** : Top P est appliqué après Temperature.
3. **Impact sur les sorties** : Top P évite de considérer les tokens très improbables, préservant ainsi la qualité et la cohérence de la sortie.
4. **Équilibre entre Temperature et Top P** : Il est difficile d'équilibrer les deux paramètres. Commencez par fixer un paramètre et ajustez l'autre, expérimentez pour trouver les réglages optimaux.

# Pénalité de présence/fréquence

## Pénalité de présence

**Pénalité de présence** détermine dans quelle mesure le modèle pénalise la répétition de mots ou de phrases. Elle encourage le modèle à utiliser un plus large éventail de mots et de phrases, rendant la sortie plus diverse et créative.

- **Plage** : [-2, 2]
- **Par défaut** : 0

Une pénalité de présence plus élevée encourage le modèle à éviter de répéter des mots ou des phrases qui sont déjà apparus dans la sortie, assurant un texte plus varié et créatif.

La pénalité de présence est spécifiquement un **ajustement ponctuel** appliqué à tous les tokens qui ont été utilisés au moins une fois. Elle réduit la probabilité de répéter un token qui est déjà apparu. Cela encourage le modèle à utiliser une gamme diversifiée de tokens, favorisant la créativité et la variété dans la sortie.

## Pénalité de fréquence

**Pénalité de fréquence** est un paramètre qui pénalise la répétition de mots en fonction de leur fréquence dans le texte généré. Elle contribue à promouvoir la diversité et à réduire la répétition dans la sortie.

- **Plage** : [-2, 2]
- **Par défaut** : 0

Une pénalité de fréquence plus élevée décourage le modèle de répéter des mots qui sont déjà apparus fréquemment dans la sortie. Cela garantit que le texte généré est plus varié et moins répétitif.

La pénalité de fréquence est spécifiquement une valeur qui augmente avec la fréquence d'apparition d'un token dans le texte généré, **une pénalité cumulative**, plus le token est échantillonné plus la pénalité est élevée. Elle réduit la probabilité de répéter un token qui est déjà apparu fréquemment. Cela garantit que le texte généré est plus varié et moins répétitif.

### Différences entre pénalité de présence et pénalité de fréquence

- **Pénalité de présence** : Il s'agit d'une contribution additive ponctuelle qui s'applique à tous les tokens qui ont été échantillonnés au moins une fois. Elle encourage le modèle à inclure une gamme diversifiée de tokens dans le texte généré.
- **Pénalité de fréquence** : Il s'agit d'une contribution proportionnelle à la fréquence à laquelle un token particulier a déjà été échantillonné. Elle décourage le modèle de répéter trop fréquemment les mêmes mots ou phrases dans le texte généré.

Les deux paramètres peuvent être ajustés pour façonner la qualité et la diversité du texte généré. Les meilleures valeurs pour ces paramètres peuvent différer selon la tâche spécifique et le résultat souhaité.
