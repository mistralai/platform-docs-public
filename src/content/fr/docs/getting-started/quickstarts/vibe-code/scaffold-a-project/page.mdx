# Créer un projet avec Vibe Code

Utilisez le <AppLink href="/vibe/code/overview">Vibe CLI</AppLink> pour générer un projet complet à partir d'une description en langage naturel, examiner chaque modification avant application et exécuter le résultat.

- **Génération en langage naturel** : décrivez ce que vous voulez, Vibe crée les fichiers.
- **Validation étape par étape** : Vibe divise les tâches complexes en étapes et demande une confirmation à chaque fois.
- **Aperçu complet** : visualisez chaque changement de fichier avant qu'il ne soit enregistré sur le disque.

À la fin, vous disposerez d'une base de projet fonctionnelle créée à partir d'une seule instruction.

**Durée estimée :** ~10 minutes

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

- Vibe CLI doit être installé et configuré. Si ce n'est pas déjà fait, commencez par lancer le <AppLink href="/getting-started/quickstarts/vibe-code/install-cli">guide d'installation du Vibe CLI</AppLink>.

<SectionTab as="h2" sectionId="step-1">Étape 1 : Créez un dossier vide et lancez le CLI</SectionTab>

Commencez dans un nouveau dossier pour que Vibe génère le projet depuis zéro.

```bash
mkdir my-project && cd my-project
vibe
```

Vibe démarre avec un contexte vide, prêt à créer la structure du projet.

<SectionTab as="h2" sectionId="step-2">Étape 2 : Décrivez votre projet</SectionTab>

Faites une description claire à Vibe. Précisez le langage, le framework et les fonctionnalités principales. Par exemple :

> Créez une API Flask en Python avec deux endpoints : un endpoint `/health` qui renvoie `{"status": "ok"}` et un endpoint `/predict` qui accepte un corps JSON contenant un champ `"text"` et renvoie `{"label": "positive", "score": 0.95}`. Incluez un `requirements.txt` et un `README`.

Vibe divise la demande en étapes et commence à générer les fichiers. Avant toute création ou modification, il affiche un aperçu.

![Vibe affichant un aperçu des modifications et demandant une confirmation](/assets/quickstarts/vibe/demo.png)

<SectionTab as="h2" sectionId="step-3">Étape 3 : Passez en revue et validez les modifications</SectionTab>

Pour chaque étape, Vibe affiche :

1. Les fichiers qu'il prévoit de créer ou modifier.
2. Le contenu de chaque fichier.
3. Une demande de confirmation.

Passez en revue les changements et tapez `y` pour valider, ou précisez votre requête pour ajuster le résultat. Après chaque validation, Vibe affiche le résultat obtenu.

![Vibe affichant la sortie générée et les étapes complétées](/assets/quickstarts/vibe/results.png)

Continuez à valider chaque étape jusqu'à la fin du projet.

<SectionTab as="h2" sectionId="verify">Vérification</SectionTab>

Vérifiez que le projet a bien été créé :

```bash
ls -la
```

Vous devriez voir les fichiers générés par Vibe (par exemple, `app.py`, `requirements.txt`, `README.md`).

Lancez le projet pour vérifier son bon fonctionnement :

```bash
pip install -r requirements.txt
python app.py
```

Testez les endpoints :

```bash
curl http://localhost:5000/health
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"text": "Mistral is great"}'
```

Les deux endpoints doivent renvoyer une réponse JSON valide.
