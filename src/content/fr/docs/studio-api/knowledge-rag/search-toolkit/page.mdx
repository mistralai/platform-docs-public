# Search Toolkit

Search Toolkit est un framework Python pour concevoir des systèmes de recherche d'information (IR) prêts pour la production. Il fournit des composants modulaires, extensibles pour l'ingestion, la recherche et l'évaluation, indépendants du backend et interchangeables à volonté.

Les LLM ne sont pas entraînés sur vos données privées. Pour que leurs réponses s'appuient sur vos propres documents, il vous faut une chaîne de récupération capable d'ingérer, d'indexer et d'interroger vos contenus. Search Toolkit réunit tous les modules nécessaires pour bâtir cette chaîne de valeur.

<SectionTab as="h1" sectionId="key-features">Fonctionnalités clés</SectionTab>

### [Ingestion](search-toolkit/ingestion)

- **Extraction multi-format** : PDF/DOCX/PPTX via Mistral OCR, HTML, feuilles de calcul, e-mails, texte brut
- **Chargement de fichiers** : Importation depuis le système de fichiers local ou implémentation de chargeurs sur mesure pour toute source
- **Découpage flexible** : à partir de caractères, de jetons, en tenant compte du markdown, ou par séparateur
- **Enrichissement** : Ajoute des métadonnées personnalisées ou des résumés générés par LLM au document ou à chaque segment
- **Indexation** : Indexation simultanée dans des bases vectorielles et à mots-clés

### [Recherche](search-toolkit/retrieval)

- **Multiples stratégies** : vectorielle (sémantique), mots-clés (BM25) ou hybride avec fusion RRF
- **Prétraitement des requêtes** : Optimisez les recherches par reformulation ou extension des requêtes via LLM
- **Rerankers** : Reranker LLM, cross-encoder ou personnalisés
- **Cache sémantique** : Mise en cache des résultats selon la similarité des requêtes, pour éviter les appels redondants


<SectionTab as="h1" sectionId="architecture">Architecture</SectionTab>

L’**ingestion** transforme des documents bruts en segments exploitables pour la recherche. Un **FileLoader** lit les données brutes à la source, un **DocumentExtractor** les convertit en documents structurés, un **TextSplitter** segmente ces documents, un **ChunkEnricher** (optionnel) ajoute des métadonnées, puis un **Embedder** produit les vecteurs pour indexer les segments dans une base vectorielle ou à mots-clés.

La **recherche** permet de retrouver les segments pertinents pour une requête donnée. Un **préprocesseur de requête** (optionnel) réécrit ou enrichit la requête, un ou plusieurs **Retrievers** interrogent l’index (vectoriel, mots-clés, hybride), puis un **Reranker** (optionnel) réordonne les résultats avant la restitution.

Les deux workflows sont orchestrés par des classes de haut niveau (`Pipeline` pour l’ingestion, `QueryEngine` pour la recherche) qui assurent la connexion et l’exécution des composants. Chaque composant est interchangeable : utilisez ceux inclus ou intégrez les vôtres.

### Composants

| Composant | Options intégrées |
|-----------|------------------|
| **File Loaders** | `FilesystemFileLoader`, chargeurs sur mesure |
| **Extractors** | `MistralOCRExtractor`, `PlainTextExtractor`, `HTMLExtractor`, `SpreadsheetExtractor`, `EmailExtractor`, `NumbersExtractor`, `LegacyOfficeExtractor` |
| **Text Splitters** | `CharacterTextSplitter`, `TokenTextSplitter`, `MarkdownTextSplitter`, `SeparatorTextSplitter` |
| **Enrichers** | `SummaryEnricher`, `ChunkEnricher` personnalisé |
| **Embedders** | `MistralEmbedder`, `Embedder` personnalisé |
| **Storage** | `VespaClient` (vectoriel + mots-clés), `VectorStoreClient` / `KeywordStoreClient` personnalisés |
| **Retrievers** | `VectorRetriever`, `KeywordRetriever` |
| **Rerankers** | `LLMReRanker`, `CrossEncoderReRanker`, `RRFRanker` |
| **Preprocessing** | `LLMQueryRewriter`, `LLMQueryExtension` |
| **Caching** | `SemanticCache` avec `InMemoryCacheBackend` |

<SectionTab as="h1" sectionId="packages">Packages</SectionTab>

| Package | Description |
|---------|-------------|
| `mistralai-search-toolkit` | SDK principal : pipelines, retrievers, rerankers, embedders, évaluation |
| `mistralai-search-toolkit[vespa]` | Extension Vespa pour stockage vectoriel et mots-clés |
| `mistralai-search-toolkit[html2md]` | Conversion HTML vers Markdown |
| `mistralai-search-toolkit[email]` | Analyse des fichiers e-mail (EML, MSG) |

Installez le package principal et les extensions dont vous avez besoin :

```bash
pip install mistralai-search-toolkit
pip install "mistralai-search-toolkit[vespa]"
```

Les packages sont disponibles sur [PyPI](https://pypi.org/project/mistralai-search-toolkit/).

:::info
Les packages requièrent **Python 3.12+**. Nous recommandons d’utiliser [uv](https://docs.astral.sh/uv/) pour la gestion des dépendances.
:::
