# Commandes et commandes slash

Utilisez les **commandes slash** et les **raccourcis clavier** pour contrôler Vibe Code directement depuis l’éditeur. Quelques commandes VS Code sont également accessibles depuis la palette de commandes pour la configuration et le débogage.

<SectionTab as="h2" sectionId="slash-commands">Commandes slash</SectionTab>

L’extension expose un sous-ensemble des commandes slash Vibe. Tapez « / » dans le panneau Vibe pour afficher les commandes disponibles. Les [compétences](/vibe/code/cli/skills) compatibles sont également proposées dans ce sélecteur.

| Commande | Suggestion de saisie | Description |
|---|---|---|
| `/help` |  | Affiche les commandes et les raccourcis clavier disponibles. |
| `/compact` | Instructions de résumé facultatives | Compresse l’historique de conversation en le résumant. |
| `/reload` |  | Recharge la configuration, les instructions des agents et les compétences à partir du disque. |
| `/log` |  | Affiche le chemin du répertoire de logs de la session en cours. |
| `/proxy-setup` | `KEY value`, `KEY` ou vide pour l’aide | Configure le proxy et les certificats SSL. |
| `/leanstall` |  | Installe l’agent Lean 4, `leanstral`. |
| `/unleanstall` |  | Désinstalle l’agent Lean 4. |
| `/data-retention` |  | Affiche les informations sur la conservation des données. |

Pour la liste complète des commandes CLI, consultez [Commandes et raccourcis](/vibe/code/cli/commands-shortcuts).

<SectionTab as="h2" sectionId="hotkeys">Raccourcis clavier</SectionTab>

Dans le panneau Vibe, « Mod » correspond à « Command » sous macOS et à « Ctrl » sous Windows ou Linux.

| Raccourci | Action |
|---|---|
| `Mod+Y` | Approuver une action d’outil. |
| `Mod+Alt+Y` | Approuver une action alternative d’outil. |
| `Mod+Shift+Y` | Approuver et appliquer une autorisation plus large. |
| `Mod+Alt+N` | Rejeter une action d’outil. |
| `Mod+N` | Démarrer une nouvelle conversation. |
| `Escape` | Annuler ou désélectionner l’état de saisie actuel. |
| `Shift+Tab` | Parcourir les modes de saisie actifs. |

<SectionTab as="h2" sectionId="cli-parity">Parité CLI</SectionTab>

L’extension n’expose pas toutes les commandes slash du CLI, mais elle permet de contrôler la plupart des paramètres d’exécution depuis l’interface :

- **`/model` et `/thinking`** ne sont pas disponibles en commandes slash. Modifiez le modèle actif ou le niveau de raisonnement via l’interface de l’extension.
- **`/resume`** n’est pas disponible en commande slash, mais vous pouvez charger les sessions précédentes via le panneau.
- **`/rename`** dispose d’un équivalent dans l’UI du panneau. Utilisez-le pour renommer la session actuelle.
- Utilisez le CLI pour les autres contrôles de session et d’exécution : `/rewind`, `/loop`, `/mcp`, `/voice`.

<SectionTab as="h2" sectionId="vscode-commands">Commandes VS Code</SectionTab>

Un petit ensemble de commandes est accessible depuis la palette de commandes de VS Code pour la configuration et le débogage.

| Commande | Fonction |
|---|---|
| **Mistral Vibe : Ouvrir Mistral Vibe** | Place le focus sur le panneau Mistral Vibe. |
| **Mistral Vibe : Afficher les logs** | Ouvre les journaux de l’extension. |

:::note
L’extension ne définit pas de raccourcis clavier VS Code par défaut. Ajoutez vos propres raccourcis dans `keybindings.json` pour accéder directement à ces commandes.
:::
