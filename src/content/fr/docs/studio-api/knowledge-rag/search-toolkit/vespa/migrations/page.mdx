Gérez les schémas de votre application Vespa grâce aux migrations Python. Pour comprendre les principes derrière les schémas, les champs et le ranking, consultez [Anatomie d’une application Vespa](anatomy).

<SectionTab as="h2" sectionId="creating-and-evolving-schemas">Création et évolution des schémas</SectionTab>

Utilisez les migrations pour définir et modifier vos schémas. Les migrations sont des fichiers Python dans `vespa_app/migrations/` : chaque fichier est en lecture seule (append-only), ordonné par horodatage, et réexécuté à chaque déploiement.

Générez une nouvelle migration :

```bash
uv run mistral-vespa generate-migration --app-dir ./vespa_app <nom>
```

Cela crée un fichier horodaté contenant un squelette de classe :

```python
from mistralai.search.toolkit.plugins.vespa.migration import VespaMigration


class MyMigration(VespaMigration):
    def migrate(self) -> None:
        pass  # Définissez vos modifications ici
```

<SectionTab as="h2" sectionId="defining-a-schema">Définir un schéma</SectionTab>

Utilisez `create_default_schema` pour des schémas directement compatibles avec `VespaSearchIndex` :

```python
from mistralai.search.toolkit.plugins.vespa.app.schemas.app import FieldDefinition, SearchMode
from mistralai.search.toolkit.plugins.vespa.migration import VespaMigration, create_default_schema, set_app_name


class InitialSchema(VespaMigration):
    def migrate(self) -> None:
        set_app_name("myapp")
        create_default_schema(
            name="articles",
            mode=SearchMode.INDEX,
            embedding_dimensions=1024,
            schema_version=1,
            additional_fields=[
                FieldDefinition.TextField(name="title"),
                FieldDefinition.TimestampField(name="created_at"),
            ],
        )
```

Cela crée un schéma contenant tous les champs nécessaires à la pipeline (fragments, embeddings, métadonnées, etc.), ainsi que les champs spécifiques à votre projet via `additional_fields`.

Utilisez `create_schema` avec des champs explicites si vous souhaitez un contrôle complet sur la définition du schéma.

### Ajouter des champs à un schéma existant

Utilisez `add_field` dans une nouvelle migration pour ajouter des champs après la création du schéma initial :

```python
from mistralai.search.toolkit.plugins.vespa.app.schemas.app import FieldDefinition
from mistralai.search.toolkit.plugins.vespa.migration import VespaMigration, add_field


class AddViewCount(VespaMigration):
    def migrate(self) -> None:
        add_field("articles", FieldDefinition.CountField(name="view_count"))
```

Activez `multi_dimensional=True` pour les champs de type tableau (par exemple, embeddings ou fragments de texte par fragment). Les dimensions d’embedding sont définies une seule fois via `create_schema(embedding_dimensions=...)`.

### Schémas multiples

Appelez plusieurs fois `create_schema()` ou `create_default_schema()` pour enregistrer plusieurs types de documents au sein d’une même application. Chaque schéma génère son propre fichier `.sd` et profil de requête. Les noms doivent être uniques.

<SectionTab as="h2" sectionId="migration-helpers">Aides à la migration</SectionTab>

| Helper | Description |
|---|---|
| `set_app_name(name)` | Définit le nom de l’application (requis lors de la première migration) |
| `create_default_schema(...)` | Crée un schéma avec les champs par défaut nécessaires à `VespaSearchIndex` (recommandé) |
| `create_schema(...)` | Crée un schéma avec des champs explicites (pour un contrôle total) |
| `add_field(schema, field)` | Ajoute un champ à un schéma existant |
| `add_query_profiles(...)` | Ajoute ou met à jour des profils de requêtes |
| `add_schema_rank_profiles(schema, paths)` | Ajoute des fichiers de profils de ranking personnalisés |
| `add_schema_model_files(schema, paths)` | Ajoute des fichiers de modèles ML |
| `add_schema_custom_document_summary(...)` | Ajoute des résumés de documents personnalisés |

<SectionTab as="h2" sectionId="deploying">Déploiement</SectionTab>

La commande `mistral-vespa migrate` détecte les migrations, les exécute dans l’ordre, construit le package de l’application et le télécharge. Consultez les guides [Développement local](local-development) et [Déploiement et exploitation](operations) pour plus de détails.

<SectionTab as="h2" sectionId="optional-snapshot">Snapshot optionnel</SectionTab>

```bash
uv run mistral-vespa generate \
  --app-dir ./vespa_app \
  --path ./vespa.lock
```

Écrit le package de l’application sur le disque, pour inspection ou validation CI. Ce fichier n’est pas utilisé lors du déploiement.
