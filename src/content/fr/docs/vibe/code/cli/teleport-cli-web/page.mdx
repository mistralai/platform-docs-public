# Téléportation du CLI vers le web

Téléportez une session CLI dans une **sandbox Vibe Code Web** afin de poursuivre la tâche sans devoir garder votre terminal ouvert. Utilisez cette fonction pour les tâches longues, pour suivre une exécution depuis un autre appareil, ou pour permettre à l’agent de pousser des commits et d’ouvrir une pull request depuis le cloud.

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

Avant de téléporter, assurez-vous de&nbsp;:

- Utiliser une clé API liée à un compte **Pro, Team ou Enterprise**.
- Avoir installé l’**[application GitHub Mistral](https://github.com/apps/mistralai/installations/new)** sur le dépôt où vous travaillez.
- Utiliser un **modèle Mistral** pour votre session locale (par exemple, Mistral Medium 3.5 ou Devstral). Les sessions basées sur des modèles tiers ne peuvent pas être téléportées.
- Lancer le CLI depuis un **dépôt Git**.

Mettez à jour le CLI avant de commencer&nbsp;:

```bash
uv tool upgrade mistral-vibe
```

<SectionTab as="h2" sectionId="two-ways">Deux manières d’utiliser Teleport</SectionTab>

<SectionTab as="h3" variant="secondary" sectionId="single-prompt">Envoyer un prompt unique vers une session cloud</SectionTab>

Depuis votre dépôt Git, préfixez toute commande par `&`&nbsp;:

```bash
vibe
> & fix the failing tests
```

La tâche s’exécute dans une sandbox cloud. Le CLI retourne une URL Vibe Code Web pour suivre la session.

<SectionTab as="h3" variant="secondary" sectionId="move-session">Transférer une session locale en cours vers le cloud</SectionTab>

Dans une session locale active, lancez&nbsp;:

```text
/teleport
```

La session se poursuit dans Vibe Code Web, propulsée par Mistral Medium 3.5. Vous pouvez la suivre depuis n’importe quel appareil connecté à Vibe.

<SectionTab as="h2" sectionId="what-happens">Que se passe-t-il après la téléportation&nbsp;?</SectionTab>

Lorsqu’une session cloud démarre, l’agent en général&nbsp;:

1. **Clone votre dépôt** dans la sandbox.
2. **Réalise la tâche**&nbsp;: lit les fichiers, exécute les commandes, modifie le code.
3. **Diffuse la progression** dans Vibe Code Web en temps réel.
4. **Pose des questions de clarification** si besoin.
5. **Pousse une branche et ouvre une pull request brouillon** à la fin.

Commits, branches et pull requests créés durant la session restent dans votre dépôt. La sandbox est supprimée à la fin de la session.

<SectionTab as="h2" sectionId="limits">Limites</SectionTab>

| Limite | Valeur |
|---|---|
| Durée maximale d’une session | 24&nbsp;heures |
| Délai d’inactivité (temps d’attente pour votre réponse) | 3&nbsp;heures |
| Délai maximum par commande | 30 secondes |
| Sessions simultanées par utilisateur | Selon l’abonnement |

<SectionTab as="h2" sectionId="caveats">Points à noter</SectionTab>

- **Teleport est unidirectionnel.** Une fois la session déplacée vers le cloud, il n’est plus possible de la ramener dans le CLI local. Continuez via Vibe Code Web.
- **Fonctionne uniquement dans un dépôt Git.** Lancer la commande en dehors d’un dossier de code échoue.
- **Les sessions cloud s’exécutent sur Mistral Medium 3.5**, quel que soit le modèle local.
