# Référence des journaux d’audit

Cette référence répertorie les champs, filtres, types d’acteurs, types de cibles et événements pouvant apparaître dans les journaux d’audit.

Les entrées de journal d’audit sont générées à partir du schéma Admin API et des constantes de journal d’audit du tableau de bord. Les champs de métadonnées varient selon l’événement et le type de ressource.

## Champs de réponse

| Champ               | Type                     | Obligatoire | Signification                                    |
| ------------------- | ------------------------ | -------- | ------------------------------------------ |
| `created_at`        | `chaîne date-heure`       | Oui      | Date et heure de création de l’entrée de journal d’audit. |
| `actor_type`        | `ActorType`              | Oui      | Type d’acteur ayant effectué l’action.   |
| `actor_metadata`    | `objet<string, string>` | Oui      | Détails sur l’acteur ayant effectué l’action.    |
| `event_type`        | `EventType`              | Oui      | Type d’action enregistrée dans le journal d’audit.  |
| `event_metadata`    | `objet<string, string>` | Oui      | Détails concernant l’action enregistrée.         |
| `target_type`       | `TargetType`             | Oui      | Type de ressource affectée par l’action.   |
| `target_metadata`   | `objet<string, string>` | Oui      | Détails concernant la ressource affectée.       |
| `log_id`            | `entier`                | Oui      | ID de l’entrée de journal d’audit.                        |
| `organization_uuid` | `chaîne UUID` ou `null`  | Non       | ID de l’Organization pour l’entrée de journal d’audit.   |
| `workspace_uuid`    | `chaîne UUID` ou `null`  | Non       | ID de l’workspace pour l’entrée de journal d’audit.      |

## Champs de métadonnées

Les champs `actor_metadata`, `event_metadata` et `target_metadata` sont des objets. Ils contiennent des données supplémentaires sur l’acteur, l’action et la ressource affectée.

Le schéma API brut accepte des valeurs de chaîne. Le tableau de bord peut enrichir les ID connus en objets pour l’affichage.

| Clé                       | Apparaît dans                       | Signification                                                                                                                                                            |
| ------------------------- | -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `user_uuid`               | Métadonnées de l’acteur                   | ID utilisateur de l’acteur humain. Le tableau de bord peut enrichir cette valeur avec `uuid`, `e-mail` et `name`.                                                                 |
| `enterprise_account_uuid` | Métadonnées de l’acteur                   | ID du compte entreprise d’une clé API en tant qu’acteur. Le tableau de bord peut enrichir cette valeur avec `uuid` et `name`.                                                            |
| `organization_uuid`       | Métadonnées de l’acteur, de l’événement ou de la cible | ID de l’Organization lié à l’acteur, à l’action ou à la ressource affectée. Le tableau de bord peut enrichir cette valeur avec `uuid` et `name`.                                    |
| `workspace_uuid`          | Métadonnées de l’acteur, de l’événement ou de la cible | ID de l’espace de travail lié à l’acteur, à l’action ou à la ressource affectée. Le tableau de bord peut enrichir cette valeur avec `uuid`, `name`, `description`, `icon` et `is_default`. |
| `uuid`                    | Métadonnées enrichies                | ID de l’utilisateur, de l’Organization, de l’espace de travail ou du compte entreprise enrichi.                                                                                           |
| `email`                   | Métadonnées d’acteur enrichies          | Adresse e-mail d’un acteur humain enrichi.                                                                                                                         |
| `name`                    | Métadonnées enrichies                | Nom d’affichage pour un utilisateur, une Organization, un espace de travail ou un compte entreprise enrichi.                                                                                 |
| `description`             | Métadonnées d’espace de travail enrichies      | Description de l’espace de travail.                                                                                                                                             |
| `icon`                    | Métadonnées d’espace de travail enrichies      | Valeur de l’icône de l’espace de travail.                                                                                                                                              |
| `is_default`              | Métadonnées d’espace de travail enrichies      | Indique si l’espace de travail est l’espace de travail par défaut.                                                                                                                    |

## Filtres

Utilisez ces paramètres de requête avec l’[endpoint des journaux d’audit de l’Admin API](/api/endpoint/beta/admin/audit-logs).

| Filtre            | Type                         | Valeur par défaut      | Signification                                                       |
| ----------------- | ---------------------------- | ------------ | ------------------------------------------------------------- |
| `actor_type`      | `ActorType[]` ou `null`      | `Aucun`       | Types d’acteurs à inclure dans les résultats du journal d’audit.              |
| `event_type`      | `EventType[]` ou `null`      | `Aucun`       | Types d’événements à inclure dans les résultats du journal d’audit.              |
| `target_type`     | `TargetType[]` ou `null`     | `Aucun`       | Types de ressources cibles à inclure dans les résultats du journal d’audit.    |
| `actor_user_uuid` | `chaîne` ou `null`           | `Aucun`       | Filtre les journaux par UUID de l’utilisateur ayant effectué l’action. |
| `sort`            | `ascending` ou `descending`  | `descending` | Ordre de tri pour les entrées de journal d’audit.                             |
| `after`           | `chaîne date-heure` ou `null` | `Aucun`       | Retourne les entrées de journal d’audit après ce moment.                     |
| `before`          | `chaîne date-heure` ou `null` | `Aucun`       | Retourne les entrées de journal d’audit avant ce moment.                    |
| `limit`           | `entier`                    | `20`         | Nombre maximal de résultats à retourner.                          |

## Types d’acteurs

| Type d’acteur | Signification                                                                                               |
| ---------- | ----------------------------------------------------------------------------------------------------- |
| `HUMAN`    | Une personne a effectué l’action dans le tableau de bord, Studio, Le Chat ou une surface de produit authentifiée. |
| `API_KEY`  | Une clé API a effectué l’action.                                                                      |
| `OTHER`    | Un processus système, un système externe ou une opération du personnel Mistral a effectué l’action.                   |
