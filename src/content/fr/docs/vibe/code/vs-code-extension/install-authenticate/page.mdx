# Installer et s’authentifier

Installez l’**extension VS Code Mistral Vibe**, ouvrez le panneau Vibe et enregistrez la clé API utilisée par l’agent intégré. L’extension inclut l’agent Vibe, il n’est donc **pas nécessaire d’installer le CLI séparément**.

:::tip
**Vous changez de surface ?** Cette page traite de l’**extension VS Code**. Pour utiliser Vibe dans votre terminal, consultez [Installer le CLI](/vibe/code/cli/install-setup) ; pour une session cloud sur un dépôt GitHub, consultez [Vibe Code Web](/vibe/code/vibe-code-web/get-started). La configuration et les sessions sont partagées entre le CLI et l’extension.
:::

<SectionTab as="h2" sectionId="requirements">Prérequis</SectionTab>

| Prérequis | Détails |
|---|---|
| **VS Code** | Version 1.94.0 ou ultérieure. |
| **Compte Mistral** | Requis pour créer ou gérer des clés API. |
| **Droit d’accès** | Un [plan Mistral](https://mistral.ai/pricing) (Free, Pro ou supérieur), ou un plan API avec une clé que vous gérez. Des limites d’utilisation et de débit s’appliquent. |

<SectionTab as="h2" sectionId="install">Installer l’extension</SectionTab>

Installez **Mistral Vibe** depuis le panneau Extensions de VS Code :

1. Ouvrez VS Code.
2. Appuyez sur `Cmd+Shift+X` sous macOS ou `Ctrl+Shift+X` sous Windows ou Linux.
3. Recherchez `Mistral Vibe`.
4. Sélectionnez l’extension publiée par **Mistral AI**.
5. Cliquez sur `Install`.

Si l’extension n’apparaît pas après installation, lancez **Developer: Reload Window** via la palette de commandes ou redémarrez VS Code.

<SectionTab as="h2" sectionId="open">Ouvrir Mistral Vibe</SectionTab>

Après installation, ouvrez l’extension via l’un des points d’entrée suivants :

- Cliquez sur l’icône **Mistral Vibe** dans la barre latérale de VS Code.
- Lancez **Mistral Vibe : Ouvrir Mistral Vibe** depuis la palette de commandes.
- Utilisez l’action en haut de l’éditeur lorsque l’icône Mistral Vibe s’affiche pour le fichier actif.

L’extension ouvre un panneau de chat webview intégré à VS Code.

<SectionTab as="h2" sectionId="authenticate">S’authentifier</SectionTab>

La première fois que l’extension nécessite des identifiants, elle ouvre un **flux de configuration en terminal** :

1. Lancez la configuration depuis l’extension lorsque cela vous est proposé.
2. Dans le terminal **Mistral Vibe Setup**, collez votre clé API Mistral.
3. Finalisez l’assistant et attendez que l’extension revienne à l’état prêt.

Ce processus lance l’agent Vibe intégré avec `--setup`. Les identifiants sont stockés dans le répertoire d’accueil Vibe pour de futures sessions.

:::info
Un nouveau **flux de connexion via navigateur** arrive, aussi bien pour le CLI que l’extension VS Code. Il permettra de vous authentifier sans copier de clé API. Cette page sera mise à jour à sa sortie.
:::

<SectionTab as="h2" sectionId="verify">Vérifier la configuration</SectionTab>

Votre configuration est prête lorsque :

- Le panneau Mistral Vibe s’ouvre sans demande d’authentification.
- Vous pouvez envoyer un prompt depuis le panneau VS Code.
- L’extension accède à l’espace de travail et au contexte de l’éditeur actifs.

Testez avec un premier prompt simple :

```text
Expliquez le fichier actuellement ouvert.
```

<SectionTab as="h2" sectionId="vsix">Installer depuis un VSIX</SectionTab>

Si votre équipe reçoit un build `.vsix` avant la publication officielle, installez-le via **Extensions > Plus d’actions > Installer depuis un VSIX...**.

:::warning
N’utilisez que des fichiers VSIX provenant d’un **canal de diffusion Mistral approuvé**.
:::
