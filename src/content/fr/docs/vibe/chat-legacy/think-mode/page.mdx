# Mode Think

:::warning
Le mode "Réflexion" est une fonctionnalité héritée du Chat. Il reste accessible dans l’onglet **Chat**, sans date de fin de prise en charge annoncée.

Dans Vibe Work, le raisonnement étape par étape démarre automatiquement lorsque la tâche l’exige. Aucune activation manuelle via un interrupteur n’est nécessaire. Pour commencer avec Work, voir [Prise en main de Work](/vibe/work/get-started).
:::

Le mode Think permet à Vibe de **résoudre des problèmes complexes** par raisonnement.

Propulsé par nos [modèles Magistral](https://mistral.ai/news/magistral/), il décompose votre question en sous-étapes, avance étape par étape et fournit une réponse claire accompagnée de l’**ensemble du raisonnement**.

:::note
Cette méthode s’appelle l’*enchaînement de pensées* (« chain-of-thought ») : le modèle explicite son raisonnement avant de répondre. Pour les travaux de référence, voir l’[article original](https://arxiv.org/abs/2201.11903).
:::

<SectionTab as="h2" sectionId="how-to-use">Comment utiliser</SectionTab>

1. Sélectionnez l’onglet **Chat** dans la barre latérale.
2. Choisissez le mode `Think` à droite de la fenêtre de chat (`Fast` est le mode par défaut).
3. Saisissez votre prompt et envoyez-le comme d’habitude.
4. Vibe commence à raisonner et affiche sa réflexion en temps réel.
5. À la fin, Vibe fournit la réponse finale.

Le mode Think reste activé tant que vous ne changez pas de mode. Vous pouvez l’activer/désactiver à tout moment dans une conversation.

<SectionTab as="h2" sectionId="when-to-use">Quand utiliser le mode Think</SectionTab>

Le mode Think est idéal pour des tâches nécessitant une **logique en plusieurs étapes** ou une **analyse structurée** :

- **Planification et stratégie** : décomposer un lancement de produit, prioriser une feuille de route ou évaluer des plans de recrutement.
- **Débogage de production** : retracer des chaînes d’erreurs, comparer les causes possibles et cibler les correctifs.
- **Décisions à facteurs multiples** : évaluer des options avec des contraintes contradictoires (choix de fournisseur, compromis d’architecture, etc.).
- **Raisonnement mathématique et quantitatif** : modéliser des scénarios financiers, faire des calculs statistiques, vérifier des démonstrations.
- **Analyse avancée de code** : raisonner sur les logiques, les cas limites, les régressions potentielles dans une base de code.


:::tip
Toutes les questions ne nécessitent pas un raisonnement détaillé. Pour une recherche factuelle, une génération de texte rapide ou une tâche simple, le chat standard est plus direct et tout aussi fiable.
:::

<SectionTab as="h2" sectionId="tips">Astuces pour de meilleurs résultats</SectionTab>

- **Soyez précis dans votre prompt.** Plus vous donnez de contexte, plus le raisonnement sera pertinent. Plutôt que *« Quel modèle de tarification choisir ? »*, essayez *« Comparez le freemium et le paiement à l’usage pour un SaaS B2B, en tenant compte du churn et de l’expansion de revenus. »*
- **Utilisez le raisonnement comme documentation.** L’enchaînement de pensées fournit une trace utile pour la discussion en Team. Copiez-le dans un [Canvas](/vibe/work/files-and-canvas) ou partagez la conversation pour montrer le cheminement décisionnel à vos collègues.
- **Combinez avec d’autres outils.** Le mode Think fonctionne avec [la recherche web](/vibe/work/web-search-open-url), [l’import de fichiers](/vibe/work/files-and-canvas) et les [Libraries](/vibe/work/libraries) pour raisonner à partir de vos propres données.
