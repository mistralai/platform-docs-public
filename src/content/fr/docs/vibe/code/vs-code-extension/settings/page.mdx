# Paramètres de l’extension

L’extension VS Code ne propose pas de panneau de configuration dédié. **Tout le comportement de Vibe provient de l’agent Vibe intégré et du même modèle de configuration Vibe utilisé par la CLI** (fichiers de configuration, variables d’environnement, surcharges projet).

<SectionTab as="h2" sectionId="telemetry">Télémétrie</SectionTab>

L’extension réutilise le paramètre global de télémétrie de VS Code ainsi que le réglage de télémétrie de la CLI Vibe :

- **Télémétrie VS Code** : contrôlée par `telemetry.telemetryLevel` dans les paramètres de VS Code. L’extension n’envoie de la télémétrie que si celle de VS Code est activée.
- **Télémétrie Vibe** : contrôlée par `enable_telemetry` dans `~/.vibe/config.toml`. Voir [Paramètres de mises à jour et de télémétrie](/vibe/code/cli/configuration#updates-telemetry).

Pour désactiver la télémétrie de l’extension, désactivez l’un des deux réglages.

<SectionTab as="h2" sectionId="shared-vibe-configuration">Configuration Vibe partagée</SectionTab>

L’extension exécute l’agent Vibe ACP. **Le comportement au niveau de l’agent** (modèles, fournisseurs, outils, agents, compétences et serveurs MCP) est défini par le modèle de configuration partagé de Vibe. Utilisez la documentation de configuration CLI pour connaître le format partagé :

- **[Configuration](/vibe/code/cli/configuration)** : référence complète pour `config.toml`.
- **[Clés API et profils](/vibe/code/cli/api-keys-profiles)** : gestion des identifiants et sélection des fournisseurs.
- **[Serveurs MCP](/vibe/code/cli/mcp-servers)** : connexion des outils externes via le Model Context Protocol.
- **[Agents](/vibe/code/cli/agents)** : agents intégrés et agents personnalisés.
- **[Compétences](/vibe/code/cli/skills)** : comportements réutilisables et ciblés.
