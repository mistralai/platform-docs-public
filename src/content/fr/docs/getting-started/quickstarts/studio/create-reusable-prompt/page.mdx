# Créer un prompt réutilisable

Utilisez les <AppLink href="https://console.mistral.ai/build/prompts" app="studio">Prompts</AppLink> pour enregistrer des modèles de prompt réutilisables en privé ou à partager avec votre espace de travail dans Studio et dans les applications qui référencent des définitions de prompt partagées.

- **Instructions réutilisables** : stockez les instructions système, les modèles de tâches et les formats de sortie au même endroit.
- **Versionnage** : mettez à jour un prompt sans perdre les versions antérieures.

À la fin, vous aurez un prompt enregistré, prêt à être testé, réutilisé et partagé selon vos besoins.

**Temps nécessaire** : ~5 minutes

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

- Un compte Studio actif.
- Un prompt que vous souhaitez réutiliser. De bons premiers candidats sont les consignes de ton pour le support, les instructions d’extraction ou un format de résumé structuré.

<SectionTab as="h2" sectionId="step-1">Étape 1 : Ouvrir les prompts</SectionTab>

1. Ouvrez <AppLink href="https://console.mistral.ai" app="studio">Studio</AppLink>.
2. Dans la barre latérale de gauche, ouvrez **Créer** > **Prompts**.
3. Cliquez sur **Nouveau prompt**.

<SectionTab as="h2" sectionId="step-2">Étape 2 : Ajouter les détails du prompt</SectionTab>

Donnez un titre et une description clairs au prompt pour que vos collaborateurs sachent quand l’utiliser.

| Champ | Exemple |
|---|---|
| **Titre** | `Ton du support` |
| **Description** | `Ton et structure des réponses pour les échanges avec le support client.` |

Gardez la description orientée vers la tâche. Elle doit expliquer le cas d’usage, et non répéter le titre.

<SectionTab as="h2" sectionId="step-3">Étape 3 : Rédiger le modèle</SectionTab>

Ajoutez le texte du prompt. Gardez les instructions suffisamment précises pour qu’un autre collaborateur puisse réutiliser le prompt sans contexte supplémentaire :

> Vous êtes un assistant du support client. Reconnaissez le problème, répondez clairement et proposez la prochaine action. Utilisez un ton calme et concis pour les administrateurs d’entreprise.

<SectionTab as="h2" sectionId="step-4">Étape 4 : Enregistrer et tester</SectionTab>

1. Cliquez sur **Enregistrer** ou **Créer le prompt**.
2. Ouvrez le prompt enregistré et consultez sa première version.
3. Testez-le dans une surface Studio compatible, comme **l’espace de test**, avant de l’utiliser dans une application.

<SectionTab as="h2" sectionId="step-5">Étape 5 : Choisir la visibilité</SectionTab>

Depuis la page des détails du prompt, cliquez sur `Partager` pour choisir qui peut l’utiliser :

- Gardons-le privé si seul vous devez y accéder.
- Partagez-le avec l’ensemble de l’espace de travail si vos coéquipiers doivent pouvoir le trouver et le réutiliser.

Vous pouvez modifier cette visibilité plus tard depuis le même bouton `Partager`.

<SectionTab as="h2" sectionId="step-6">Étape 6 : Créer une nouvelle version</SectionTab>

Créez une nouvelle version lorsque vous souhaitez modifier le prompt sans écraser la version utilisée par vos collaborateurs ou les applications. Studio crée la nouvelle version après l’enregistrement des modifications.

1. Ouvrez le prompt depuis la liste **Prompts**.
2. Mettez à jour le texte du prompt.
3. Ajoutez des notes de version qui expliquent les modifications, par exemple `Réponses du support plus courtes` ou `Ajout de consignes d’escalade`.
4. Cliquez sur **Enregistrer**.
5. Confirmez que l’historique des versions affiche la nouvelle version.

Les anciennes versions restent disponibles pour consultation et retour en arrière. Testez la nouvelle version avant de la promouvoir pour une utilisation en production.

Si votre espace de travail utilise des alias, promouvez la version testée vers un alias tel que `stable`.

<SectionTab as="h2" sectionId="verify">Vérification</SectionTab>

Votre prompt est prêt si :

- Il apparaît dans la liste des prompts avec le bon titre et la bonne description.
- Un test produit le ton, la structure et les contraintes attendus.
- Sa visibilité est correcte : privée pour vous ou partagée avec l’espace de travail.
- L’historique des versions affiche votre version originale et la version mise à jour.
