# Installer et configurer

Installez la CLI Vibe, exécutez `vibe` dans votre projet et terminez le prompt de configuration.

:::tip
**Sur une autre plateforme&nbsp;?** Cette page traite **spécifiquement de la CLI**. Si vous préférez travailler dans votre éditeur ou avec un dépôt distant, consultez l'[extension VS Code](/vibe/code/vs-code-extension/install-authenticate) ou [Vibe Code Web](/vibe/code/vibe-code-web/get-started). Retrouvez une comparaison détaillée dans [Choisir CLI, VS Code ou sessions web](/vibe/code/choose-cli-vscode-web-sessions).
:::

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

Avant d'installer la CLI, assurez-vous de disposer&nbsp;:

- **macOS, Linux ou Windows.**
- **Python 3.12 ou supérieur** pour l'installation manuelle.
- *(Optionnel)* d'un **compte Mistral** pour utiliser les modèles hébergés Mistral. La CLI fonctionne aussi entièrement hors ligne avec des [modèles locaux](/vibe/code/cli/offline-models) ou avec n'importe quelle clé API compatible.

Si vous choisissez d'utiliser les modèles hébergés Mistral, deux méthodes existent pour obtenir une clé API, selon votre plan&nbsp;:

- Avec un **[plan Mistral](https://mistral.ai/pricing)** (Free, Pro ou supérieur), Vibe Code est inclus&nbsp;; des limites d'utilisation et de débit s'appliquent.
- Avec un **plan API**, vous gérez vous-même la clé API depuis la console.

<SectionTab as="h2" sectionId="install">Installer la CLI</SectionTab>

<ExplorerTabs>
<ExplorerTab value="macos-linux" label="macOS / Linux" default>

Utilisez le programme d'installation en une ligne&nbsp;:

```bash
curl -LsSf https://mistral.ai/vibe/install.sh | bash
```

Le programme d'installation&nbsp;:

- Vérifie si `uv` est installé et l'installe si nécessaire.
- Installe ou met à jour `mistral-vibe` avec `uv tool install` ou `uv tool upgrade`.
- Rends les commandes `vibe` et `vibe-acp` disponibles, à condition que votre variable `PATH` soit configurée.

Si le programme d'installation signale que `vibe` n'est pas dans votre `PATH`, ajoutez le dossier indiqué à votre profil shell et redémarrez votre terminal.

</ExplorerTab>
<ExplorerTab value="manual" label="Installation manuelle">

Utilisez l'installation manuelle si vous préférez gérer vous-même les outils Python.

Avec `uv`&nbsp;:

```bash
uv tool install mistral-vibe
```

Ou avec `pip`&nbsp;:

```bash
pip install mistral-vibe
```

</ExplorerTab>
<ExplorerTab value="windows" label="Windows">

Installez d'abord `uv`&nbsp;:

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

Puis installez Vibe&nbsp;:

```powershell
uv tool install mistral-vibe
```

:::note
La prise en charge de Windows est au mieux-effort. Certains raccourcis clavier et comportements du terminal dépendent de l'émulateur. Consultez les [terminaux recommandés](/vibe/code/cli/commands-shortcuts#keyboard-shortcuts).
:::

</ExplorerTab>
</ExplorerTabs>

<SectionTab as="h2" sectionId="start">Démarrer Vibe dans votre projet</SectionTab>

Ouvrez un terminal à la racine de votre projet et exécutez&nbsp;:

```bash
vibe
```

Au premier lancement, Vibe&nbsp;:

- Crée un fichier de configuration par défaut à `~/.vibe/config.toml` s'il n'existe pas déjà.
- Lance un **assistant de configuration** pour enregistrer votre clé API.

<SectionTab as="h2" sectionId="choose-access-path">Choisir votre mode d'accès</SectionTab>

L'assistant de configuration demande une [clé API](/vibe/code/cli/api-keys-profiles). Choisissez l'option qui correspond à votre plan&nbsp;:

- **Se connecter avec un compte Mistral**&nbsp;: choisissez un [plan Mistral](https://mistral.ai/pricing) (Free, Pro ou supérieur). Vibe Code est inclus&nbsp;; des limites d'utilisation et de débit s'appliquent, avec paiement à l'usage au-delà du budget inclus lorsque disponible.
- **Utiliser votre plan API**&nbsp;: générez une clé API depuis [Studio](https://console.mistral.ai/home?profile_dialog=api-keys) ou utilisez une clé déjà existante.

Collez votre clé API lorsque cela est demandé. L'assistant l'enregistre pour les prochaines sessions dans le dossier personnel Vibe. Vous pouvez également définir `MISTRAL_API_KEY` dans votre environnement shell si vous préférez éviter le prompt interactif.

:::tip
Vous pouvez relancer l'assistant de configuration à tout moment avec `vibe --setup`.
:::

<SectionTab as="h2" sectionId="verify">Vérifier la configuration</SectionTab>

Votre configuration est correcte lorsque&nbsp;:

- `vibe --version` renvoie une version.
- `vibe` ouvre l'interface terminal interactive à la racine de votre projet.
- Vibe accepte un prompt sans redemander de clé API.

Essayez un premier prompt simple&nbsp;:

```text
Trouver les commentaires TODO dans ce projet.
```

<SectionTab as="h2" sectionId="reset-config">Réinitialiser votre configuration</SectionTab>

Si la CLI ne démarre pas ou fonctionne de façon inattendue après modification du fichier de configuration, déplacez-le de côté et relancez Vibe. Un nouveau fichier `~/.vibe/config.toml` sera généré au prochain lancement&nbsp;:

```bash
mv ~/.vibe/config.toml ~/.vibe/config.toml.bkp
vibe
```
