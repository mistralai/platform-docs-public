# Commencer avec Vibe Code Web

Vibe Code Web est la **console de pilotage pour les agents de développement à distance**. Lancez une tâche de développement depuis le web, suivez l’agent pendant l’exécution, puis examinez la branche ou la pull request générée.

Même moteur que le [Vibe CLI](/vibe/code/cli/install-setup), mais dans un **sandbox cloud gérée** accessible **depuis n’importe quel navigateur**.

:::info
Le déploiement de Vibe Code Web est progressif sur tous les abonnements. Consultez [Limitations et cycle de vie](/vibe/code/vibe-code-web/limits-and-lifecycle) pour les quotas, le comportement du modèle et les limites de session.
:::

<SectionTab as="h2" sectionId="what-you-can-do">Ce que vous pouvez faire</SectionTab>

Avec Vibe Code Web, vous pouvez&nbsp;:

- Créer des projets à partir d’un ou plusieurs dépôts GitHub sous le même propriétaire GitHub.
- Démarrer des sessions de développement à distance depuis le web.
- Envoyer un prompt à une session cloud via le Vibe CLI, ou téléporter une session CLI active vers le web.
- Exécuter des agents dans des sandboxes cloud gérées.
- Générer des branches et pull requests GitHub.
- Revoir le code final sur GitHub.

Idéal pour les tâches qui&nbsp;:

- S’exécutent à distance sans fichiers locaux, services ou secrets locaux.
- Sont assez longues pour ne pas vouloir laisser votre machine allumée.
- Se terminent par une branche ou une pull request à examiner.

:::tip
Privilégiez le [CLI](/vibe/code/cli/install-setup) ou l’[extension VS Code](/vibe/code/vs-code-extension/install-authenticate) si la tâche dépend de fichiers locaux, de secrets sur votre poste, de runtimes non pris en charge ou d’un accès direct à votre machine.
:::

<SectionTab as="h2" sectionId="core-concepts">Principes de base</SectionTab>

Vibe Code Web structure le travail autour de trois entités&nbsp;: **projets**, **dépôts** et **sessions**. Un projet regroupe un ou plusieurs dépôts GitHub d’un même propriétaire&nbsp;; chaque session correspond à une exécution de l’agent sur ce projet.

<ExplorerTabs>
<ExplorerTab value="project" label="Project" default>

Un **projet** est un espace de travail ciblé, visible dans la barre latérale de Vibe Code Web. Il inclut un ou plusieurs dépôts GitHub appartenant au même propriétaire (utilisateur ou organisation). Les sessions que vous lancez depuis ce projet partagent ce périmètre de dépôt.

Au lancement, la configuration du projet se limite à sélectionner un ou plusieurs dépôts du même propriétaire GitHub et à nommer le projet.

Voir [Projets](/vibe/code/vibe-code-web/projects) pour la création, modification, suppression et gestion des sessions.

</ExplorerTab>
<ExplorerTab value="repository" label="Repository">

Un **dépôt** est un dépôt GitHub que l’agent clone dans la sandbox, lit, édite, sur lequel il applique des commits et ouvre des pull requests. Un projet peut regrouper un ou plusieurs dépôts, qui doivent tous appartenir au **même propriétaire GitHub**. Pour travailler sur plusieurs propriétaires, créez des projets distincts.

Voir [Dépôts et autorisations GitHub](/vibe/code/vibe-code-web/github-repositories-permissions) pour l’installation de l’application, la gestion des tokens, l’attribution et la gestion des accès.

</ExplorerTab>
<ExplorerTab value="session" label="Session">

Une **session** est une exécution unique de l’agent sur un projet. Le déroulement est le même&nbsp;:

1. **Démarrage.** Créez une session depuis le web ou envoyez-en une via la CLI. Fournissez un prompt précis.
2. **Exécution.** L’agent travaille à distance dans une sandbox isolée&nbsp;: clonage, plan, modifications, commandes et tests.
3. **Suivi.** Surveillez les commandes et modifications de fichiers en temps réel.
4. **Validation.** Ouvrez la branche ou pull request générée sur GitHub pour revue de code, commentaires, états requis et décisions de fusion.

Vous pouvez consulter les sessions passées, mais une fois la sandbox supprimée, il n’est plus possible de reprendre une session.

Voir [Sessions](/vibe/code/vibe-code-web/sessions) pour le guide complet d’exécution et [Limitations et cycle de vie](/vibe/code/vibe-code-web/limits-and-lifecycle) pour les statuts et délais.

</ExplorerTab>
</ExplorerTabs>

<SectionTab as="h2" sectionId="quickstart">Guide de démarrage</SectionTab>

Vous pouvez démarrer une session Vibe Code Web depuis le web ou via le Vibe CLI.

<ExplorerTabs>
<ExplorerTab value="web" label="Web" default>

1. Ouvrez Vibe Code Web depuis l’onglet **Code** dans Vibe.
2. Créez un projet et connectez un ou plusieurs dépôts GitHub du même propriétaire. Voir [Projets](/vibe/code/vibe-code-web/projects).
3. Cliquez sur **Nouvelle session** et saisissez un prompt concis. Précisez le nom du fichier, du test échoué, la trace de pile ou le comportement attendu si possible.
4. Observez l’agent cloner le dépôt, analyser le code et exécuter les commandes.
5. Ouvrez la branche ou pull request générée sur GitHub pour la relecture et la fusion.

:::tip
Commencez par une tâche courte et bien délimitée. Citez le fichier, la fonction ou le test échoué si vous le connaissez.
:::

</ExplorerTab>
<ExplorerTab value="cli" label="CLI">

Depuis le [Vibe CLI](/vibe/code/cli/install-setup), préfixez un prompt avec `&` pour envoyer la tâche vers une session cloud au lieu de l’exécuter en local&nbsp;:

```bash
vibe
& fix the failing auth tests
```

Le CLI retourne une URL web pour suivre la session. La tâche s’exécute dans le cloud, pas sur votre ordinateur.

Pré-requis&nbsp;:

- Vibe CLI installé (mise à jour via `uv tool upgrade mistral-vibe`).
- Un compte Mistral avec accès à Vibe Code Web.
- L’[application GitHub Mistral](https://github.com/apps/mistralai/installations/new) installée sur le dépôt cible.

:::tip
Déjà dans une session CLI locale&nbsp;? Utilisez `/teleport` dans la session pour la transférer dans le cloud, avec l’historique et l’état des branches conservés. Voir [Téléportation de CLI vers le web](/vibe/code/cli/teleport-cli-web).
:::

</ExplorerTab>
</ExplorerTabs>
