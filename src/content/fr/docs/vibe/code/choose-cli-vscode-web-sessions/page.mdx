# Choisissez CLI, VS Code ou sessions web

**Un agent, trois interfaces.** Vibe Code fonctionne en ligne de commande (CLI), comme extension VS Code, ou via Vibe Code Web. Les trois partagent le même moteur, la même configuration et la même couche de personnalisation (agents, compétences, serveurs MCP), pour vous laisser choisir l’interface la plus adaptée à votre besoin.

:::info
Vibe implémente aussi le [Agent Client Protocol (ACP)](https://agentclientprotocol.com/) et est référencé dans le [registre ACP](https://github.com/agentclientprotocol/registry/tree/main/mistral-vibe), ce qui permet de l’utiliser avec d’autres [clients compatibles ACP](https://agentclientprotocol.com/get-started/clients), y compris les IDE JetBrains. Pour les étapes de configuration, voir [Utiliser Vibe dans d’autres IDE](/vibe/code/use-vibe-in-other-ides). Cette page se concentre sur les interfaces prises en charge.
:::

<SectionTab as="h2" sectionId="get-started">Premiers pas</SectionTab>

<ExplorerTabs>
<ExplorerTab value="cli" label="CLI" default>

**Choisissez le CLI si vous voulez utiliser Vibe directement dans votre terminal.**

Le CLI est l’interface la plus flexible. Il fonctionne dans n’importe quel terminal, prend en charge les sessions interactives et les scripts, et offre un accès complet à votre shell et à l’arborescence de travail.

Exemples de tâches :

- *« Refactoriser `src/api/handlers/` pour utiliser le nouveau pattern de gestion des erreurs. »*
- *« Expliquer le fonctionnement de l’authentification dans ce repo. »*
- *« Lancer la suite de tests, corriger les échecs, puis valider les modifications. »*
- Exécutions automatisées ou sans surveillance via `vibe --prompt` pour l’intégration continue ou l’automatisation.

[Premiers pas avec le CLI](/vibe/code/cli/install-setup)

</ExplorerTab>
<ExplorerTab value="vscode" label="Extension VS Code">

**Choisissez l’extension VS Code si vous voulez Vibe dans votre éditeur.**

Pas besoin d’installer le CLI. L’extension vous donne accès à toutes les fonctions du CLI via le [Agent Client Protocol](https://agentclientprotocol.com/) et ajoute le contexte de l’éditeur : fichier actif, sélection, diff, et gestion de source, le tout dans le même espace de travail.

Exemples de tâches :

- *« Expliquer le fichier que je consulte actuellement. »*
- *« Relire ce diff pour identifier les cas limites avant d’ouvrir une pull request. »*
- *« Refactoriser cette sélection pour rendre la gestion d’erreur explicite. »*

[Premiers pas avec l’extension VS Code](/vibe/code/vs-code-extension/install-authenticate)

</ExplorerTab>
<ExplorerTab value="web" label="Vibe Code Web">

**Choisissez Vibe Code Web pour des sessions cloud sur un dépôt GitHub, directement dans votre navigateur.**

Aucun clonage local nécessaire. Vibe Code Web s’exécute dans un environnement géré, interagit directement avec un dépôt GitHub, et peut préparer des branches ou des pull requests pour vous.

Exemples de tâches :

- *« Lire et modifier un repo qui n’est pas cloné en local. »*
- *« Préparer une branche et ouvrir une PR depuis une session distante. »*
- *« Continuer la session que j’ai commencée dans le CLI, mais sur le web. »*
- Laisser un agent travailler de manière autonome sur votre code pendant que vous faites autre chose.

[Premiers pas avec Vibe Code Web](/vibe/code/vibe-code-web/get-started)

</ExplorerTab>
</ExplorerTabs>

:::tip
**Le CLI et l’extension VS Code partagent la même configuration et les mêmes sessions** (agents, compétences, serveurs MCP, profils, historique de conversations). Vous pouvez commencer une session dans le CLI puis la reprendre dans l’extension, ou l’inverse, sans jamais perdre le contexte.

Pour transférer une session vers Vibe Code Web, utilisez la commande `/teleport`. Voir [l'article dédié](/vibe/code/cli/teleport-cli-web) pour le fonctionnement actuel et les limites.
:::

<SectionTab as="h2" sectionId="what-each-surface-gives-you">Ce que chaque interface permet</SectionTab>

| | CLI | Extension VS Code | Vibe Code Web |
|---|---|---|---|
| Installation locale | Requise | Requise | Aucune |
| Accès au dépôt | Clone local | Clone local | Direct sur GitHub |
| Accès au système de fichiers et shell | Arborescence locale | Arborescence locale via ACP | Environnement distant |
| Agents, compétences, serveurs MCP et connecteurs personnalisés | Oui | Oui | Oui |
| Mode non interactif | Oui (`vibe --prompt`) | Non | Non |
| Durée de session | Terminal local (partagée avec VS Code) | VS Code local (partagée avec CLI) | Cloud géré |
| Usage idéal | Dev terminal, automatisation, CI | Dev éditeur, relecture visuelle | Tâches dépôt distant sans config locale |
