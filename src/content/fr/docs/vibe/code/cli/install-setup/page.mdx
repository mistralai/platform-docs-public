# Installation et configuration

Installez le CLI Vibe, exécutez `vibe` dans votre projet et suivez l’assistant de configuration.

:::tip
**Sur un autre environnement&nbsp;?** Cette page détaille l’utilisation du **CLI**. Si vous préférez travailler dans votre éditeur ou sur un dépôt distant, rendez-vous sur l’[extension VS Code](/vibe/code/vs-code-extension/install-authenticate) ou [Vibe Code Web](/vibe/code/vibe-code-web/get-started). Consultez [Choisir CLI, VS Code ou sessions web](/vibe/code/choose-cli-vscode-web-sessions) pour une comparaison directe.
:::

<SectionTab as="h2" sectionId="prerequisites">Pré-requis</SectionTab>

Avant d’installer le CLI, assurez-vous de disposer&nbsp;:

- **macOS, Linux ou Windows.**
- **Python 3.12 ou supérieur** pour l’installation manuelle.
- *(Optionnel)* d’un **compte Mistral** pour utiliser les modèles hébergés par Mistral. Le CLI fonctionne aussi hors ligne avec des [modèles locaux](/vibe/code/cli/offline-models) ou avec n’importe quelle clé API compatible de votre choix.

Si vous choisissez d’utiliser les modèles hébergés par Mistral, deux options existent pour obtenir une clé API selon votre plan&nbsp;:

- Avec un **[plan Mistral](https://mistral.ai/pricing)** (Free, Pro ou supérieur), Vibe Code est inclus&nbsp;: limites d’utilisation et de débit applicables.
- Avec un **plan API**, vous gérez la clé API vous-même depuis la console.

<SectionTab as="h2" sectionId="install">Installer le CLI</SectionTab>

<ExplorerTabs>
<ExplorerTab value="macos-linux" label="macOS / Linux" default>

Utilisez le programme d’installation en une ligne&nbsp;:

```bash
curl -LsSf https://mistral.ai/vibe/install.sh | bash
```

Le programme d’installation&nbsp;:

- Vérifie si `uv` est installé et l’installe si besoin.
- Installe ou met à jour `mistral-vibe` avec `uv tool install` ou `uv tool upgrade`.
- Rend les commandes `vibe` et `vibe-acp` disponibles, à condition que votre `PATH` soit bien configuré.

Si le programme d’installation indique que `vibe` n’est pas dans votre `PATH`, ajoutez le dossier indiqué à votre profil de shell et redémarrez votre terminal.

</ExplorerTab>
<ExplorerTab value="manual" label="Installation manuelle">

Choisissez l’installation manuelle si vous souhaitez gérer vous-même les outils Python.

Avec `uv`&nbsp;:

```bash
uv tool install mistral-vibe
```

Ou avec `pip`&nbsp;:

```bash
pip install mistral-vibe
```

</ExplorerTab>
<ExplorerTab value="windows" label="Windows">

Installez d’abord `uv`&nbsp;:

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

Puis installez Vibe&nbsp;:

```powershell
uv tool install mistral-vibe
```

:::note
La prise en charge de Windows est fournie «&nbsp;au mieux&nbsp;». Certains raccourcis clavier et comportement du terminal dépendent de l’émulateur utilisé. Consultez les [terminaux recommandés](/vibe/code/cli/commands-shortcuts#keyboard-shortcuts).
:::

</ExplorerTab>
</ExplorerTabs>

<SectionTab as="h2" sectionId="start">Démarrer Vibe dans votre projet</SectionTab>

Ouvrez un terminal à la racine de votre projet et exécutez&nbsp;:

```bash
vibe
```

Au premier lancement, Vibe&nbsp;:

- Crée un fichier de configuration par défaut dans `~/.vibe/config.toml` s’il n’existe pas déjà.
- Vous guide via un **assistant de configuration** pour enregistrer votre clé API.

<SectionTab as="h2" sectionId="choose-access-path">Choisir votre mode d’accès</SectionTab>

L’assistant demande une [clé API](/vibe/code/cli/api-keys-profiles). Sélectionnez l’option qui correspond à votre plan&nbsp;:

- **Connexion avec un compte Mistral**&nbsp;: choisissez un [plan Mistral](https://mistral.ai/pricing) (Free, Pro ou supérieur). Vibe Code est inclus&nbsp;: limites d’utilisation et de débit applicables, paiement à l’usage au-delà du quota inclus quand disponible.
- **Utiliser votre plan API**&nbsp;: générez une clé API depuis [Studio](https://console.mistral.ai/codestral/vibe) ou utilisez-en une que vous gérez déjà.

Collez votre clé API dans le terminal. L’assistant l’enregistre pour les prochaines sessions dans le répertoire Vibe. Vous pouvez aussi définir `MISTRAL_API_KEY` dans votre environnement shell si vous ne souhaitez pas utiliser l’assistant interactif.

:::tip
Vous pouvez relancer à tout moment l’assistant de configuration avec `vibe --setup`.
:::

<SectionTab as="h2" sectionId="verify">Vérifier la configuration</SectionTab>

Votre configuration est prête lorsque&nbsp;:

- `vibe --version` affiche un numéro de version.
- `vibe` lance l’interface terminal interactive depuis la racine de votre projet.
- Vibe accepte un prompt sans redemander de clé API.

Essayez un premier prompt simple&nbsp;:

```text
Trouver les commentaires TODO dans ce projet.
```

<SectionTab as="h2" sectionId="reset-config">Réinitialiser votre configuration</SectionTab>

Si le CLI ne démarre pas ou se comporte de façon inattendue après modification du fichier de configuration, mettez-le de côté puis relancez Vibe. Un nouveau `~/.vibe/config.toml` est créé au prochain lancement&nbsp;:

```bash
mv ~/.vibe/config.toml ~/.vibe/config.toml.bkp
vibe
```
