# Vibe Code

Vibe Code est le **mode développement** de Vibe. Avec un accès en lecture/écriture à votre système de fichiers, un shell et un ensemble d'outils configurable, il peut lire des fichiers, exécuter des commandes, écrire du code et ouvrir des pull requests pour vous, sous votre supervision.

Utilisez Vibe Code sur un dépôt local via le [CLI](/vibe/code/cli/install-setup) ou l'[extension VS Code](/vibe/code/vs-code-extension/install-authenticate), ou sur un dépôt GitHub dans un environnement distant via [Vibe Code Web](/vibe/code/vibe-code-web/get-started).

:::info
Vous gardez le contrôle. Vibe Code affiche son plan, demande votre validation avant chaque action sensible (commandes shell, modifications de fichiers, pull requests), et peut être interrompu à chaque étape. Le comportement est personnalisable par agent et par environnement. Voir [Sécurité, validations et autorisations](/vibe/code/safety-approvals-permissions).
:::

<SectionTab as="h2" sectionId="why-use-vibe-code">Pourquoi utiliser Vibe Code</SectionTab>

Cas d'usage courants&nbsp;:

- **Écrire du code&nbsp;:** décrivez ce que vous voulez construire, Vibe Code applique des modifications conformes à la structure et aux conventions de votre projet.
- **Explorer et comprendre un codebase&nbsp;:** demandez à Vibe Code d'inspecter les fichiers, d'expliquer l'architecture, et d'analyser comment les systèmes s'articulent.
- **Relire du code&nbsp;:** identifiez bugs, erreurs logiques, cas limites ou changements risqués avant déploiement.
- **Déboguer&nbsp;:** partagez une erreur, un test en échec ou une stack trace, et laissez Vibe Code analyser le contexte et proposer une correction ciblée.
- **Automatiser les tâches de développement&nbsp;:** déléguez les tâches répétitives comme les refactors, les tests, les migrations, la configuration ou la préparation des pull requests.

:::tip
**Vous travaillez en dehors du code&nbsp;?** Vibe Code est conçu principalement pour le code source (fichiers, dépôts, commandes, diffs, pull requests). Pour des tâches transverses (applications, documents, échanges, réunions ou outils métier), [Vibe Work](/vibe/work/get-started) est généralement plus adapté.
:::
