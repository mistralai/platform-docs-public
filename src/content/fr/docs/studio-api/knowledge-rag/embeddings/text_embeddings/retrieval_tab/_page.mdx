Notre modèle d'embeddings donne de bons résultats pour les tâches de recherche. Les embeddings sont utiles pour les systèmes de génération augmentée par recherche (RAG), qui utilisent les informations récupérées à partir d’une base de connaissances pour générer des réponses.

À haut niveau, vous transformez une base de connaissances, comme un répertoire local, des fichiers texte ou des wikis internes, en embeddings textuels et les stockez dans une base de données vectorielle. Ensuite, en fonction de la requête de l'utilisateur, vous récupérez les embeddings les plus similaires et transmettez les fragments pertinents à un modèle pour générer une réponse ancrée dans la requête et le Contexte de l'utilisateur.

Si vous souhaitez une fonctionnalité gérée qui ingère, vectorise et recherche des documents pour vous, utilisez [Bibliothèques](/studio-api/libraries). Si vous souhaitez rechercher dans des sources connectées comme Google Drive ou SharePoint, utilisez [Connecteurs](/studio-api/connectors). Pour apprendre comment fonctionne le RAG, consultez le [guide de démarrage RAG](/studio-api/knowledge-rag/rag_quickstart).
