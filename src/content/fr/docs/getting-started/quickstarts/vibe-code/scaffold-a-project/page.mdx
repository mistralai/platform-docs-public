# Créer un projet avec Vibe Code

Utilisez la <AppLink href="/vibe/code/overview">CLI Vibe</AppLink> pour générer un projet complet à partir d’une description en langage naturel, examinez chaque modification avant son application, et exécutez le résultat.

- **Génération à partir d’une description naturelle** : décrivez ce que vous souhaitez et la CLI crée les fichiers.
- **Validation étape par étape** : Vibe Code divise les tâches importantes en étapes et demande une approbation.
- **Prévisualisation complète** : visualisez chaque modification de fichier avant son écriture sur le disque.

À la fin, vous disposerez d’un projet fonctionnel généré à partir d’un seul prompt.

**Durée estimée** : ~10 minutes

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

- CLI Vibe installée et configurée. Si ce n’est pas encore fait, exécutez d’abord le <AppLink href="/getting-started/quickstarts/vibe-code/install-cli">guide de démarrage Installer la CLI Vibe</AppLink>.

:::tip
Scaffolding is a good use case for the **`plan`** or **`accept-edits`** [agents](/vibe/code/cli/agents). `plan` asks Vibe to outline what it will do before generating anything; `accept-edits` lets Vibe write files without prompting per step. The default agent asks for approval on every edit, which is safest for a first run.
:::

<SectionTab as="h2" sectionId="step-1">Étape 1 : Créer un répertoire vide et lancer la CLI</SectionTab>

Commencez dans un répertoire vierge pour que la CLI génère le projet à partir de zéro.

```bash
mkdir my-project && cd my-project
vibe
```

La CLI démarre avec un contexte vide, prêt à générer le projet.

:::note
If you run `vibe` in an existing project with a `.vibe/` directory, the CLI asks whether you trust the folder before loading any local configuration. See [Dossiers fiables](/vibe/code/safety-approvals-permissions#trusted-folders).
:::

<SectionTab as="h2" sectionId="step-2">Étape 2 : Décrire votre projet</SectionTab>

Donnez à la CLI une description claire. Soyez précis concernant le langage, le framework et les fonctionnalités clés. Par exemple :

> Créer une API Python Flask avec deux endpoints : un endpoint `/health` qui retourne `{"status": "ok"}` et un endpoint `/predict` qui accepte un corps JSON avec un champ `"text"` et retourne `{"label": "positive", "score": 0,95}`. Ajouter un fichier `requirements.txt` et un `README`.

Vibe Code décompose la demande en étapes et commence à générer les fichiers. Avant de créer ou de modifier un fichier, il affiche une prévisualisation.

![Vibe Code affichant une prévisualisation des modifications et demandant une confirmation](/assets/quickstarts/vibe/demo.png)

<SectionTab as="h2" sectionId="step-3">Étape 3 : Examiner et approuver les modifications</SectionTab>

Pour chaque étape, la CLI affiche les fichiers qu’elle prévoit de créer ou de modifier, le contenu de chaque fichier et une demande de confirmation. Appuyez sur l’une des [touches d’approbation](/vibe/code/safety-approvals-permissions#approval-model) :

- **`Entrée`**, **`1`** ou **`O`** pour approuver cette modification spécifique.
- **`2`** pour approuver toutes les modifications de ce fichier lors de la session en cours.
- **`3`** pour approuver toutes les modifications dans ce dossier.
- **`4`** pour approuver toutes les modifications dans le répertoire de travail.
- **`N`** pour refuser et précisez votre requête.

Après chaque étape approuvée, la CLI affiche le résultat.

![Vibe Code affichant le résultat généré et les étapes terminées](/assets/quickstarts/vibe/results.png)

Continuez à approuver les étapes jusqu’à ce que le projet soit complet.

<SectionTab as="h2" sectionId="verify">Vérification</SectionTab>

Vérifiez que le projet a bien été créé :

```bash
ls -la
```

Vous devriez voir les fichiers générés par Vibe Code (par exemple, `app.py`, `requirements.txt`, `README.md`).

Demandez à la CLI de démarrer le serveur et de tester les endpoints pour vous :

> Installez les dépendances, démarrez le serveur, puis exécutez une requête curl sur `/health` et `/predict` pour vérifier leur fonctionnement.

Ou exécutez-le vous-même :

```bash
pip install -r requirements.txt
python app.py
```

Testez les endpoints :

```bash
curl http://localhost:5000/health
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"text": "Mistral is great"}'
```

Les deux endpoints doivent retourner des réponses JSON valides.
