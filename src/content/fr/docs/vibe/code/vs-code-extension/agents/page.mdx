# Agents

Les agents sont des **surcouches de configuration** appliquées par-dessus la configuration globale. Ils regroupent un prompt système, un prompt de condensation, un modèle actif, un ensemble d’outils disponibles et des règles d’approbation. L’extension VS Code utilise le même système d’agents que la CLI, avec un mode supplémentaire `Chat` pour discuter sans outils.

<SectionTab as="h2" sectionId="built-in-agents">Agents intégrés</SectionTab>

| Agent | Comportement |
|---|---|
| `Default` | Agent polyvalent. Demande une approbation avant d’exécuter des outils. |
| `Plan` | Agent en lecture seule pour explorer et planifier. Approuve automatiquement les outils de lecture sûrs. |
| `Accept Edits` | Approuve automatiquement les modifications de fichiers dans le dossier de travail. Demande tout de même une approbation pour d’autres actions (par exemple, des commandes shell). |
| `Auto Approve` | Approuve automatiquement toutes les exécutions d’outils. À utiliser **uniquement dans un environnement isolé et de confiance** : cet agent peut exécuter n’importe quelle commande, comme `rm -rf`, sur n’importe quel chemin accessible par Vibe. |
| `Chat` | Mode conversation uniquement, sans exécution d’outils. |

<SectionTab as="h2" sectionId="select">Sélectionner un agent</SectionTab>

Cliquez sur le sélecteur d’agents en bas du panneau Mistral Vibe pour choisir un agent dans la liste déroulante. Celle-ci comporte tous les agents intégrés et personnalisés disponibles dans votre configuration.

Vous pouvez aussi parcourir rapidement les agents avec `Shift+Tab`, comme dans la CLI.

Le sélecteur sélectionne par défaut `Default` et la sélection reste active durant toute la session.

<SectionTab as="h2" sectionId="custom-agents">Agents personnalisés</SectionTab>

L’extension VS Code lit les mêmes définitions d’agents que la CLI, depuis `~/.vibe/agents/` (niveau utilisateur) et `./.vibe/agents/` (niveau projet). Les agents personnalisés déclarés à ces emplacements apparaissent dans le sélecteur à côté des agents intégrés.

Consultez la [référence Agents de la CLI](/vibe/code/cli/agents) pour le schéma de configuration complet, des exemples, et la différence entre `agent_type = "agent"` et `agent_type = "subagent"`.
