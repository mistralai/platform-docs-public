# Vision

Les fonctionnalités de vision permettent aux modèles **d’analyser des images et de fournir des informations à partir de contenus visuels**, en plus du texte. Cette approche multimodale ouvre de nouvelles perspectives pour les applications exigeant une compréhension à la fois textuelle et visuelle.

Nous proposons une gamme de modèles dotés de capacités de vision, tous disponibles via l’API Chat Completions.

:::tip
Pour la lecture de documents, l’OCR et l’extraction de données, consultez [Document AI](/studio-api/document-processing/overview).
:::

<SectionTab as="h1" sectionId="before-you-start">Avant de commencer</SectionTab>

### Modèles recommandés avec capacités vision

- **Mistral Large 3** via `mistral-large-2512`
- **Mistral Medium 3.1** via `mistral-medium-2508`
- **Mistral Small 3.2** via `mistral-small-2506`
- **Ministral 3** :
  - Ministral 3 14B via `ministral-14b-2512`
  - Ministral 3 8B via `ministral-8b-2512`
  - Ministral 3 3B via `ministral-3b-2512`

### Utiliser les modèles vision

Il existe deux façons d’envoyer une image à l’API Chat Completions : soit en passant une URL, soit en passant une image encodée en base64.

:::tip
Avant de continuer, nous vous recommandons de lire la documentation [Chat Completions](/studio-api/conversations/chat-completion) pour en savoir plus sur cette API et son utilisation.

<Faq defaultValue={['price-per-image', 'tokens-per-image']}>
  <FaqItem question="Quel est le prix par image ?">

    Le tarif est calculé de la même façon qu’avec les tokens d’entrée par image, chaque image étant tokenisée.

  </FaqItem>
  <FaqItem question="Combien de tokens correspondent à une image et/ou quelle est la résolution maximale ?">

    Selon le modèle et la résolution, une image sera tokenisée différemment. Résumé ci-dessous.

    | Modèle             | Résolution max. | ≈ Formule                             | ≈ Nb max tokens |
    | ----------------- | --------------- | ------------------------------------- | --------------- |
    | Mistral Large 3   | 1540x1540       | `≈ (RésolutionX * RésolutionY) / 784` | ≈ 3025          |
    | Mistral Small 3.2 | 1540x1540       | `≈ (RésolutionX * RésolutionY) / 784` | ≈ 3025          |
    | Mistral Medium 3  | 1540x1540       | `≈ (RésolutionX * RésolutionY) / 784` | ≈ 3025          |
    | Ministral 3       | 1540x1540       | `≈ (RésolutionX * RésolutionY) / 784` | ≈ 3025          |
    | Pixtral Large     | 1024x1024       | `≈ (RésolutionX * RésolutionY) / 256` | ≈ 4096          |
    | Pixtral 12B       | 1024x1024       | `≈ (RésolutionX * RésolutionY) / 256` | ≈ 4096          |

    Si la résolution de l’image envoyée dépasse la résolution maximale du modèle, l’image sera redimensionnée à sa résolution maximale. Une erreur sera retournée si la résolution dépasse **10000x10000**.

  </FaqItem>
  
  <FaqItem question="Puis-je les utiliser pour générer des images ?">

    Non, ces modèles sont conçus pour comprendre et analyser des images, pas pour en créer.

  </FaqItem>
  <FaqItem question="Quels types de fichiers image sont pris en charge ?">

    Les formats d’image suivants sont actuellement pris en charge :

    - PNG (.png)
    - JPEG (.jpeg et .jpg)
    - WEBP (.webp)
    - GIF non animé, contenant une seule image (.gif)

  </FaqItem>

  <FaqItem question="Existe-t-il une limite de taille pour l’image ?">

    La taille maximale autorisée pour un fichier image est de 10 Mo.

  </FaqItem>
  <FaqItem question="Quel est le nombre maximal d’images par requête ?">

    Via l’API, une requête peut inclure jusqu’à 8 images.

  </FaqItem>
  <FaqItem question="Quel est le seuil de débit (rate limit) ?">

    Pour plus d’informations sur le débit autorisé, consultez <AppLink href="https://console.mistral.ai/limits/" app="studio">Limits</AppLink>.

  </FaqItem>
</Faq>
