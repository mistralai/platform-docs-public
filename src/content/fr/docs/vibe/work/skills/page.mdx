# Réutiliser le travail avec les Skills

Les Skills sont des **instructions et ressources réutilisables** qui confèrent à Work des capacités spécialisées pour des tâches précises. Utilisez-les lorsqu'une tâche suit une méthode connue, nécessite une checklist, un modèle, un format de sortie ou une procédure métier : formalisez une approche une fois, Work l'applique chaque fois que la tâche correspond, en gardant le même ton, le même format et la même procédure.

:::info
Les Skills respectent le standard ouvert [Agent Skills](https://agentskills.io/home), initialement conçu par Anthropic et désormais adopté par plusieurs outils à agents. Chaque Skill est un dossier comprenant un fichier `SKILL.md` (instructions + métadonnées) et tout fichier supplémentaire que Work doit utiliser avec (références, modèles, exemples, données).
:::

<SectionTab as="h2" sectionId="how-skills-work">Fonctionnement des Skills</SectionTab>

Work charge les Skills grâce à un **chargement progressif** en trois étapes :

1. **Découverte** : au lancement de la session, Work charge uniquement le nom et la description de chaque Skill (~100 tokens chacun), juste assez pour déterminer quand ils sont pertinents.
2. **Activation** : lorsqu'une tâche correspond à la description d'un Skill, Work lit l'intégralité des instructions du `SKILL.md` dans le contexte.
3. **Exécution** : Work suit les instructions, charge éventuellement les fichiers référencés ou exécute des exemples si besoin.

Vous pouvez donc activer de nombreux Skills sans saturer le contexte : seuls les Skills pertinents sont développés sous forme d'instructions complètes.

:::tip
La **description** du Skill détermine quand il s'active. Rédigez des descriptions qui précisent **quand** utiliser le Skill, pas seulement ce qu'il fait (par exemple : *« À utiliser lors de la relecture de contrats clients pour analyser les risques de renouvellement »* plutôt que *« Aide à gérer les contrats »*).
:::

<SectionTab as="h2" sectionId="built-in-skills">Skills intégrés</SectionTab>

Vibe Work propose un ensemble de **Skills intégrés** adaptés aux principales tâches de productivité. Ils apparaissent dans votre liste de Skills, aux côtés de vos Skills personnels et de ceux de l'espace de travail, s'activent automatiquement si la tâche correspond, et peuvent être invoqués explicitement avec `/{skill-name}`. Désactivez ceux dont vous n'avez pas besoin depuis `Contexte` > `Skills`.

| Skill | Déclencheurs |
|---|---|
| `challenge-my-thinking` | Tester un plan, jouer l'avocat du diable, « Qu'ai-je oublié ? », analyse de risques. |
| `data-analysis` | Examiner, nettoyer, transformer ou agréger des données (`.csv`, `.json`, tableaux, métriques, anomalies). |
| `deep-research` | Recherche approfondie, synthèse appuyée sur les sources, analyse du marché ou de la technique. |
| `doc-coauthoring` | Rédiger à plusieurs des propositions, spécifications techniques, documents de décision, RFC. |
| `document-review` | Relire des documents pour vérifier leur complétude, conformité, cohérence ou qualité. |
| `internal-comms` | Rapports d'avancement, communications pour direction ou partenaires, FAQ, rapports d'incident. |
| `meeting-prep` | Préparation des réunions, briefing sur les points de calendrier. |
| `research-synthesis` | Synthétiser plusieurs sources, notes ou fichiers en une synthèse structurée. |
| `skill-creator` | Créer, mettre à jour ou supprimer des Skills (chargé avant toute modification de Skill). |
| `stakeholder-translator` | Adapter un contenu à un public différent (direction, technique, management). |
| `structured-extraction` | Extraire des données structurées (tableaux, JSON) à partir d'e-mails, de PDFs ou de texte libre. |
| `vibe-work-onboarding` | Découverte de Vibe Work pour les nouveaux utilisateurs. |

<SectionTab as="h3" variant="secondary" sectionId="example-deep-research">Exemple : `/deep-research`</SectionTab>

Pour voir un Skill intégré en action, invoquez explicitement le Skill `deep-research` :

> `/deep-research` Quelles sont les approches principales pour les LLM à long contexte en 2026 ?

Work élabore la recherche, exécute plusieurs requêtes web, synthétise les sources et rend une note structurée accompagnée de références, ouverte dans [Canvas](/vibe/work/files-and-canvas).

<SectionTab as="h2" sectionId="personal-vs-workspace">Skills personnels et d'espace de travail</SectionTab>

Ouvrez le panneau latéral puis `Contexte` > `Skills` pour voir vos Skills, organisés selon deux sections supplémentaires :

- **Personnel** : Skills que vous avez créés pour votre usage. Vous seul pouvez les voir.
- **Espace de travail** : Skills partagés dans tout votre espace de travail. Tous les membres de l'espace peuvent les utiliser.

Activez ou désactivez chaque Skill individuellement pour maîtriser lesquels sont chargés dans le contexte de Work.

:::note
Les administrateurs d'espace de travail peuvent **forcer l'activation** de certains Skills pour tout l'espace (par exemple, des Skills métier pour une équipe finance). Les Skills forcés sont toujours actifs et ne peuvent pas être désactivés par les utilisateurs individuels.
:::

<SectionTab as="h2" sectionId="using-skills">Utiliser un Skill dans une tâche</SectionTab>

Une fois le Skill activé, Work l'utilise automatiquement si la tâche correspond à sa description. Vous pouvez aussi invoquer un Skill explicitement :

- **Commande slash** : tapez `/` dans la fenêtre de conversation, puis choisissez un Skill dans la liste ou saisissez directement `/{skill-name}`.
- **Mention naturelle** : donnez le nom du Skill dans votre prompt (ex. : *« Utilise le Skill contract-review sur ce PDF »*).

Lorsqu'un Skill est actif dans une tâche, Work l'affiche dans la conversation pour indiquer lequel est appliqué.

<SectionTab as="h2" sectionId="creating-skills">Créer un Skill</SectionTab>

Deux façons de créer un Skill : le construire manuellement dans l'éditeur, ou laisser Work le générer à partir d'une conversation.

<SectionTab as="h3" variant="secondary" sectionId="create-from-ui">Créer depuis l'éditeur</SectionTab>

1. Ouvrez `Contexte` > `Skills` via le panneau latéral.
2. Cliquez sur `Nouvelle tâche`.
3. Complétez le formulaire :
   - **Titre** : nom court et descriptif (par exemple : `Rédaction conforme à la charte graphique`).
   - **Description** : quand Work doit utiliser ce Skill. Ce texte est utilisé à l'étape de découverte pour décider de l'activer : rendez-le précis.
   - **SKILL.md** : instructions complètes que Work suivra à l'activation du Skill. Rédigez-les comme pour un collègue : ton, format, étapes, exemples.
4. (Optionnel) Ajoutez des **fichiers ou dossiers** associés au Skill (fiches de référence, modèles, charte graphique, échantillons de sortie, jeux de données). Work les charge à la demande lors de l'activation du Skill.
5. Cliquez sur `Créer Skill`.

<SectionTab as="h3" variant="secondary" sectionId="create-from-chat">Créer un Skill à partir d'une tâche (recommandé)</SectionTab>

La manière la plus rapide de créer un Skill est de demander à Work d'en rédiger un directement à partir d'une tâche réelle :

1. Exécutez une tâche où vous affinez une méthode à réutiliser (style d'écriture, checklist de relecture, méthode de recherche).
2. Demandez à Work *« transforme ceci en Skill »* ou *« enregistre ceci comme Skill »*.
3. Work génère un brouillon de `SKILL.md`, liste le contenu et propose une relecture.
4. Modifiez si besoin, puis enregistrez.

:::tip
Créer un Skill à partir de l'usage réel capture les vraies méthodes qui fonctionnent, y compris les détails auxquels vous n'auriez pas pensé lors d'une rédaction à blanc. Itérez lors d'une tâche, enregistrez quand le résultat est satisfaisant, puis améliorez au fil du temps.
:::

<SectionTab as="h2" sectionId="managing-skills">Modifier et gérer les Skills</SectionTab>

Depuis la page `Contexte` > `Skills`, vous pouvez :

- **Modifier** le titre, la description, le contenu du `SKILL.md` ou les fichiers liés.
- **Désactiver** un Skill sans le supprimer.
- **Supprimer** un Skill personnel (les Skills d'espace de travail nécessitent la permission adaptée).
- **Publier** un Skill personnel sur votre espace de travail pour le rendre disponible à d'autres membres.

Les modifications d’un Skill sont effectives dans les prochaines conversations. Les conversations déjà actives continuent d'utiliser la version précédente jusqu'à ce que vous en lanciez une nouvelle.

<SectionTab as="h2" sectionId="best-practices">Bonnes pratiques</SectionTab>

- **Construisez les Skills à partir de tâches réelles.** Voir [Créer un Skill à partir d'une tâche](#create-from-chat) ci-dessus. C’est la méthode la plus efficace pour commencer.
- **Formulez les descriptions comme des déclencheurs.** La description indique à Work **quand** activer le Skill. Une tournure du type *« Utilisé lorsque... »* est bien plus efficace que *« Ce Skill aide à... »*.
- **Gardez les Skills focalisés.** Un Skill par objectif. Si un Skill couvre trop de procédures différentes, scindez-le.
- **Évitez les doublons.** Deux Skills au périmètre trop proche perturbent Work et surchargent le contexte : modifiez le Skill existant plutôt que de créer une variante.
- **Rassemblez les fichiers de référence.** Modèles, charte graphique, échantillons ou données associés doivent se trouver à côté du `SKILL.md` et sont utilisés à l’activation du Skill.
- **Itérez.** Testez les Skills sur des tâches réelles, repérez les manques, modifiez et améliorez.
