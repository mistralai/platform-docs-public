# Choose CLI, VS Code, or web sessions

**One agent, three surfaces.** Vibe Code runs as the CLI, the VS Code extension, and Vibe Code Web. They share the same engine, configuration, and customization layer (agents, skills, MCP servers), so you can pick whichever fits the task at hand.

:::info
Vibe also implements the [Agent Client Protocol (ACP)](https://agentclientprotocol.com/) and is published in the [ACP registry](https://github.com/agentclientprotocol/registry/tree/main/mistral-vibe), so you can run it in other [ACP-compatible clients](https://agentclientprotocol.com/get-started/clients), including JetBrains IDEs. For setup steps, see [Use Vibe in other IDEs](/vibe/code/use-vibe-in-other-ides). This page focuses on the supported surfaces.
:::

<SectionTab as="h2" sectionId="get-started">Get started</SectionTab>

<ExplorerTabs>
<ExplorerTab value="cli" label="CLI" default>

**Pick the CLI when you want Vibe directly in your terminal.**

The CLI is the most flexible surface. It runs in any terminal emulator, supports interactive sessions and scripting, and has full access to your shell and working tree.

Typical tasks:

- *"Refactor `src/api/handlers/` to use the new error handler pattern."*
- *"Explain how authentication works in this repo."*
- *"Run the test suite, fix what fails, and commit the result."*
- Scripted or unattended runs with `vibe --prompt` for CI and automation.

[Get started with the CLI](/vibe/code/cli/install-setup)

</ExplorerTab>
<ExplorerTab value="vscode" label="VS Code extension">

**Pick the VS Code extension when you want Vibe in your editor.**

You don't need to install the CLI. The extension gives you access to all CLI capabilities through the [Agent Client Protocol](https://agentclientprotocol.com/), and adds **editor context** on top: active file, selection, diffs, and source control, all in the same workspace.

Typical tasks:

- *"Explain the file I'm currently looking at."*
- *"Review this diff for edge cases before I open a pull request."*
- *"Refactor this selection to make error handling explicit."*

[Get started with the VS Code extension](/vibe/code/vs-code-extension/install-authenticate)

</ExplorerTab>
<ExplorerTab value="web" label="Vibe Code Web">

**Pick Vibe Code Web for a cloud session on a GitHub repository, straight from your browser.**

No local checkout required. Vibe Code Web runs in a managed sandbox, works directly against a GitHub repository, and can prepare branches and pull requests for you.

Typical tasks:

- *"Inspect and change a repo I don't have cloned locally."*
- *"Prepare a branch and open a PR from a remote session."*
- *"Continue the session I started in the CLI on the web."*
- Let an agent work autonomously on your codebase while you do something else.

[Get started with Vibe Code Web](/vibe/code/vibe-code-web/get-started)

</ExplorerTab>
</ExplorerTabs>

:::tip
**CLI and VS Code extension share the same configuration and sessions** (agents, skills, MCP servers, profiles, and conversation history). You can start a session in the CLI and resume it in the extension, or the other way around, without losing context.

To hand off work to Vibe Code Web, use the `/teleport` command from the CLI. See the [dedicated article](/vibe/code/cli/teleport-cli-web) for the current flow and limits.
:::

<SectionTab as="h2" sectionId="what-each-surface-gives-you">What each surface gives you</SectionTab>

| | CLI | VS Code extension | Vibe Code Web |
|---|---|---|---|
| Local install | Required | Required | None |
| Repository access | Local clone | Local clone | Directly via GitHub |
| Filesystem and shell access | Local working tree | Local working tree, through ACP | Remote sandbox |
| Custom agents, skills, MCP servers, and connectors | Yes | Yes | Yes |
| Non-interactive mode | Yes (`vibe --prompt`) | No | No |
| Session lifetime | Local terminal (shared with VS Code) | Local VS Code (shared with CLI) | Managed cloud |
| Best fit | Terminal-first development, automation, CI | Editor-native development and visual review | Remote repository tasks without local setup |
