# Your First Workflow

We'll walk through creating a simple workflow that executes a single activity.

<SectionTab as="h1" sectionId="prerequisites">
  Prerequisites
</SectionTab>

To complete this quickstart, you will need:

1. A [Mistral account](https://console.mistral.ai/).
1. [Python](https://www.python.org/downloads/) 3.12 installed on your machine.
1. [uvx](https://github.com/astral-sh/uvx) installed in your environment.

<SectionTab as="h1" sectionId="scaffold-your-project">
  Scaffold your project
</SectionTab>

To get started, run the following `uvx` command in your terminal:

```bash
uvx mistralai-workflows-cli setup
```

This scaffolds a ready-to-run Python project with the Workflows SDK already configured, a minimal example workflow, and helper commands to run your worker and trigger executions.

The command prompts you to [generate a Mistral API key in the Mistral Console](https://console.mistral.ai/home?profile_dialog=api-keys).
Follow the prompts to generate the API key, and then pass it to the command when requested. Note that API keys are only accessible once.

Open the `my-workflow` directory in the IDE of your choice.

<SectionTab as="h1" sectionId="step-1-define-your-workflow-and-worker">
  Step 1: Define Your Workflow and Worker
</SectionTab>

Navigate to `src/workflows/hello.py` to see an example workflow. This file contains the following code:

<Tabs>
  <TabItem value="python" label="Python">

```python
"""Minimal example workflow — edit this file or create new ones."""

from pydantic import BaseModel

import mistralai.workflows as workflows


class HelloInput(BaseModel):
    name: str = "World"


@workflows.activity()
async def greet(name: str) -> str:
    """A simple activity that returns a greeting."""
    return f"Hello, {name}! Welcome to Mistral Workflows."


@workflows.workflow.define(
    name="hello-world",
    workflow_display_name="Hello World",
    workflow_description="A minimal hello-world workflow.",
)
class HelloWorkflow:
    @workflows.workflow.entrypoint
    async def run(self, input: HelloInput) -> str:
        return await greet(input.name)

```

  </TabItem>
</Tabs>
This code defines a Mistral Workflow that takes a `name` as input and returns a greeting.

Open `src/discover.py`. Focus on the following snippet:

<Tabs>
  <TabItem value="python" label="Python">

```python
await workflows.run_worker(discovered)
```
  </TabItem>
</Tabs>

This code auto-discovers all workflows in the `src/workflows` directory, and then auto-watches them using the Mistral Workflows `run_worker` function.

<SectionTab
  as="h1"
  sectionId="step-2-run-your-worker-with-the-helloworld-workflow"
>
  Step 2: Run Your Worker with the `hello-world` Workflow
</SectionTab>

To start the worker, run the following command in your terminal:

```bash
make start-worker
```

This command starts the worker, connects to the Mistral API, and registers your workflow to wait for a task to do.
For details on this command, check out the `Makefile`.
To see the workflow running, you need to trigger execution using one of the methods in the following step.

<SectionTab as="h1" sectionId="step-3-trigger-execution-of-your-workflow">
  Step 3: Trigger execution of Your Workflow
</SectionTab>

You can trigger your workflow using the Mistral Console, the Mistral Python SDK, or the Mistral API. 
The `my-workflow` project also includes a `Makefile` command to easily trigger workflows.

<Tabs>
  <TabItem value="console" label="Console">

1. Visit [https://console.mistral.ai/](https://console.mistral.ai/), and navigate to the workspace where you created your API key.
1. Click **Workflows** in the sidebar.
1. Select **hello-world**
1. Launch the workflow using the **Start Workflow** button and the input `{"name": <your_name>}`.
1. Find your execution in the **Executions** tab.
1. Verify the output of the workflow:

```
{
  "result": "Hello, <your_name>! Welcome to Mistral Workflows."
}
```

  </TabItem>
    <TabItem value="workflow-project-command" label="Makefile">

1. In your `my-workflow` directory, open a second terminal window and run the following command:
```bash
make execute workflow=hello-world input='{"name": "World"}'
```

1. In the terminal logs, you will see the following output:
```console
Result: {'result': 'Hello, World! Welcome to Mistral Workflows.'}
```

  </TabItem>
  <TabItem value="python-sdk" label="python">

```python
from mistralai.client import Mistral

client = Mistral(api_key="your_api_key")

execution = client.workflows.execute_workflow(
    workflow_identifier="hello-world",
    input={"name": "Alice"},
)
print(execution.model_dump_json(indent=2))
```

  </TabItem>
  <TabItem value="api" label="curl">

```bash
curl -X POST https://api.mistral.ai/v1/workflows/hello-world/execute \
  -H "Authorization: Bearer $MISTRAL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"input": {"name": "Alice"}}'
```

  </TabItem>
  <TabItem value="output" label="Output">

```json
{

  "end_time": null,
  "result": null,
  "parent_execution_id": null,
  "total_duration_ms": null
}
```

  </TabItem>
</Tabs>

When you're done testing your workflow, press `Ctrl+C` to stop the worker.
