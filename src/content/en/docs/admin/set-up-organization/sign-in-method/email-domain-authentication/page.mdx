# Email domain authentication

:::info
Email domain authentication is available on Team plans and above.
:::

Email domain authentication, also called discoverable domain authentication, controls how users join your Organization. When enabled, any user with an email address that matches a verified domain can join your Organization from <AppLink href="https://admin.mistral.ai/join" app="admin">Join Organization</AppLink>.

<SectionTab as="h1" sectionId="how-it-works">How it works</SectionTab>

- Users with an email address matching a verified domain can join from <AppLink href="https://admin.mistral.ai/join" app="admin">Join Organization</AppLink>.
- Users still **create and manage their own credentials** (email and password). This feature controls Organization membership, not authentication.
- For single sign-on authentication **without separate credentials**, use [SSO](/admin/set-up-organization/sign-in-method/saml-sso) instead (Enterprise plans).

<SectionTab as="h1" sectionId="prerequisites">Prerequisites</SectionTab>

You need to **verify ownership of your domain** before enabling this feature. Domain verification uses a DNS TXT record. See [domain verification](/admin/set-up-organization/verify-domain) for the full setup steps.

<SectionTab as="h1" sectionId="enabling">Enable email domain authentication</SectionTab>

1. Open <AppLink href="https://admin.mistral.ai/organization/access" path={["Admin Panel", "Administration", "Access"]} /> in the Admin Panel.
2. In the **Domain Ownership** section, find the verified domain you want to make discoverable.
3. Turn on **Discoverable** for that domain.

The Admin Panel shows a confirmation message: **Email domain authentication enabled for `domain.com`**.

<SectionTab as="h1" sectionId="disabling">Disable email domain authentication</SectionTab>

You can disable email domain authentication by turning off **Discoverable** for the domain in the **Domain Ownership** section.

- Users with matching domain emails can no longer join automatically from <AppLink href="https://admin.mistral.ai/join" app="admin">Join Organization</AppLink>.
- Existing members aren't affected.
- You'll need to manually invite new users unless [SSO](/admin/set-up-organization/sign-in-method/saml-sso) is configured.

:::caution
If you re-enable the feature later, you'll need to go through the setup process again.
:::
