# Créer votre premier espace de travail

Un espace de travail est un **environnement partagé** pour une équipe ou un cas d'usage au sein de votre Organisation. Il définit les ressources, les accès, l'utilisation et les limites applicables au travail effectué dans cet espace.

Chaque espace de travail peut avoir ses propres :

- **Membres et rôles** : décidez qui peut accéder à l'espace de travail et ce qu'ils peuvent y faire.
- **Clés API** : limitez l'utilisation de l'API à un espace de travail.
- **Ressources** : congruez les ressources Vibe et Studio, telles que les prompts, les Bibliothèques et les Skills, par espace de travail. Vous pouvez également configurer les connexions des Connecteurs et leurs permissions pour chaque espace de travail.

{/* TODO: add link to the Connectors admin config when published */}
- **Utilisation et limites** : suivez la consommation et définissez des plafonds de dépenses par espace de travail.
- **Paramètres produits** : activez ou désactivez les fonctionnalités de Vibe pour un espace de travail spécifique.

Du point de vue d'un utilisateur, changer d'espace de travail modifie le Contexte produit dans lequel il travaille. Consultez [les espaces de travail dans Studio](/admin/workspaces/workspaces-in-studio) et [les espaces de travail dans Vibe](/admin/workspaces/workspaces-in-vibe) pour comprendre comment l'espace sélectionné affecte les ressources dans chaque produit.

:::note
Une Organisation peut compter jusqu'à **500 espaces de travail actifs**. Les noms des espaces de travail doivent être uniques au sein d'une Organisation.
:::

<SectionTab as="h1" sectionId="create-workspace">Créer un espace de travail</SectionTab>

Toute Organisation **commence avec un espace de travail par défaut**. Les Administrateurs de l'Organisation peuvent créer des espaces de travail supplémentaires lorsqu'ils ont besoin de plus de segmentation.

1. Ouvrez <AppLink href="https://admin.mistral.ai/organization/workspaces" path={[ "console Admin", "Administration", "espaces de travail"]} />.
2. Cliquez sur **Nouvel espace de travail**.
3. Choisissez une **icône** pour représenter l'espace de travail.
4. Saisissez un **nom**. C'est l'étiquette conviviale affichée dans les interfaces utilisateur et les exports.
5. Facultativement, saisissez une **description**.
6. Facultativement, ajoutez des **membres** et attribuez des **rôles** d'espace de travail à ces membres. Consultez [Accès à l'espace de travail](#workspace-access) pour comprendre comment les rôles s'appliquent.
7. Cliquez sur **Créer**.

:::tip
Le créateur se voit automatiquement attribuer le rôle **Admin d'espace de travail**.
:::

<SectionTab as="h1" sectionId="workspace-access">Accès à l'espace de travail</SectionTab>

Les membres appartiennent d'abord à l'Organisation, puis obtiennent l'accès à un ou plusieurs espaces de travail. Vous pouvez ajouter des membres directement à un espace de travail ou gérer leur accès via des [Groupes](/admin/identity-access/groups).

Le rôle d'un membre dans un espace de travail contrôle ce qu'il peut faire dans cet espace. Son rôle dans l'Organisation contrôle ce qu'il peut faire dans l'ensemble de l'Organisation. Les deux sont indépendants : par exemple, un Membre de l'Organisation peut être Admin d'espace de travail dans un espace spécifique.

:::info
Les Administrateurs de l'Organisation ont toujours un accès complet à chaque espace de travail de l'Organisation, même lorsqu'ils ne sont pas explicitement ajoutés en tant que membres de cet espace.
:::

Pour la liste complète des rôles dans un espace de travail et de leurs permissions, consultez [Rôles et permissions](/admin/identity-access/roles-permissions).

<SectionTab as="h1" sectionId="archive-workspace">Archiver un espace de travail</SectionTab>

Archivez un espace de travail lorsque les membres n'ont plus besoin d'y accéder, mais que vous souhaitez le conserver dans l'historique de l'Organisation.

1. Ouvrez <AppLink href="https://admin.mistral.ai/organization/workspaces" path={[ "console Admin", "Administration", "espaces de travail"]} />.
2. Trouvez l'espace de travail dans la liste.
3. Cliquez sur l'icône **Archiver**.
4. Confirmez l'action d'archivage.

:::warning
L'archivage d'un espace de travail le rend inaccessible à tous les membres. Cette action est irréversible. L'espace de travail par défaut ne peut pas être archivé.
:::

<SectionTab as="h1" sectionId="next-steps">Étapes suivantes</SectionTab>

Après avoir créé un espace de travail, configurez-le pour Studio et Vibe, puis définissez des limites et passez en revue l'utilisation :

- [Espaces de travail dans Studio](/admin/workspaces/workspaces-in-studio) : comprenez comment la sélection d'un espace de travail affecte les clés API, les ressources développeurs, l'utilisation, les limites et les Connecteurs dans Studio.
- [Espaces de travail dans Vibe](/admin/workspaces/workspaces-in-vibe) : comprenez comment la sélection d'un espace de travail affecte les discussions, les Projets, les outils, les options de fonctionnalités, la rétention, le traçage et les Connecteurs dans Vibe.
- [Utilisation et limites par espace de travail](/admin/workspaces/usage-limits) : définissez des plafonds de dépenses et passez en revue l'utilisation des espaces de travail.
