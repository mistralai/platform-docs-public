# Clés API

Les clés API authentifient vos requêtes auprès de l'API Mistral. Chaque clé est rattachée à un espace de travail spécifique et donne accès à tous les endpoints API disponibles sur votre offre.

<SectionTab as="h1" sectionId="creating-keys">Créer une clé API</SectionTab>

1. <AppLink href="https://console.mistral.ai/api-keys" app="studio">Clés API</AppLink>.
2. Cliquez sur **Créer une nouvelle clé**.
3. Définissez éventuellement un **nom** (par ex. « Pipeline de production ») et une **date d'expiration**.
4. Cliquez sur **Créer une nouvelle clé**.
5. Copiez immédiatement la clé.

:::warning
**La clé complète n'est affichée qu'une seule fois**. Après avoir fermé la boîte de dialogue, vous ne pourrez plus la récupérer. Stockez la clé de manière sécurisée dans un gestionnaire de mots de passe ou un coffre-fort de secrets. Ne la partagez pas et ne la versionnez pas.
:::

<SectionTab as="h1" sectionId="using-keys">Utiliser les clés API</SectionTab>

Transmettez la clé dans l'en-tête `Authorization` :

```bash
curl https://api.mistral.ai/v1/chat/completions \
  -H "Authorization: Bearer $MISTRAL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "mistral-medium-latest", "messages": [{"role": "user", "content": "Hello"}]}'
```

Ou définissez-la comme variable d'environnement pour les SDK :

```bash
export MISTRAL_API_KEY="your-api-key"
```

<SectionTab as="h1" sectionId="workspace-scoping">Portée espace de travail</SectionTab>

Les clés API sont **rattachées à l'espace de travail** dans lequel elles ont été créées. Les ressources créées avec une clé (datasets, jobs batch, etc.) sont visibles par tous les membres de cet espace de travail.

Pour utiliser des clés différentes selon les environnements (développement, staging, production), créez des [espaces de travail](/admin/security-access/organization) distincts.

<SectionTab as="h1" sectionId="key-management">Gestion des clés</SectionTab>

- **Effectuez une rotation des clés** régulièrement. Créez une nouvelle clé, mettez à jour vos applications, puis supprimez l'ancienne.
- **Révoquez immédiatement les clés compromises** en les supprimant depuis la page des clés API.
- **Définissez des dates d'expiration** pour que les clés cessent automatiquement de fonctionner après une date donnée.
- **Suivi de l'utilisation** : toute l'utilisation des clés d'un espace de travail est comptabilisée ensemble et facturée à l'Organisation.

<SectionTab as="h1" sectionId="activation">Activation du paiement</SectionTab>

Les clés API nécessitent l'activation des paiements. Configurez votre facturation sur <AppLink href="https://admin.mistral.ai/organization/billing" app="admin">Abonnements › Facturation</AppLink>. Vous pouvez choisir entre l'offre gratuite Experiment (avec des limites de débit prudentes) et l'offre à l'usage Scale.
