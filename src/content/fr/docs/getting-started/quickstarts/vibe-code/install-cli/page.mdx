# Installer la CLI Vibe et envoyer votre premier prompt

Installez la <AppLink href="/vibe/code/overview">CLI Vibe</AppLink>, enregistrez votre clé API, puis envoyez votre premier prompt directement depuis le terminal.

- **Installation en une ligne :** la CLI est opérationnelle en quelques secondes sur macOS ou Linux.
- **Assistant de configuration :** enregistrez votre clé API au premier lancement.
- **Envoyer un prompt :** laissez Vibe analyser votre projet et répondre dans le terminal.

À la fin de cette procédure, la CLI Vibe sera installée, configurée et répondra à vos prompts dans le terminal.

**Temps estimé :** environ 5 minutes

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

- macOS, Linux ou Windows.
- Python 3.12 ou version ultérieure (pour l'installation manuelle).
- *(Optionnel)* un compte Mistral si vous souhaitez utiliser des modèles hébergés par Mistral. La CLI fonctionne aussi entièrement [hors ligne avec des modèles locaux](/vibe/code/cli/offline-models) ou avec n'importe quelle clé API compatible OpenAI. Consultez [Clés API et profils](/vibe/code/cli/api-keys-profiles) pour le tableau complet des fournisseurs.

:::tip
Sur une autre plateforme ? Consultez les guides pour l’[extension VS Code](/vibe/code/vs-code-extension/install-authenticate) ou [Vibe Code Web](/vibe/code/vibe-code-web/get-started).
:::

<SectionTab as="h2" sectionId="step-1">Étape 1 : installer la CLI</SectionTab>

**Installation en une ligne (macOS/Linux)**

Lancez la commande suivante dans votre terminal :

```bash
curl -LsSf https://mistral.ai/vibe/install.sh | bash
```

Le programme d'installation vérifie la présence de `uv` (l'installateur d’outils Python d’Astral), installe ou met à jour `mistral-vibe` et rend les commandes `vibe` et `vibe-acp` utilisables, à condition que votre variable d’environnement `PATH` soit bien configurée.

**Installation manuelle (macOS/Linux/Windows)**

Si vous préférez une configuration manuelle, ou si vous êtes sous Windows :

1. Vérifiez que Python 3.12 ou plus récent est installé : `python3 --version`.
2. Installez avec votre gestionnaire de paquets préféré :

```bash
# Avec uv (recommandé)
uv tool install mistral-vibe

# Avec pip
pip install mistral-vibe
```

Sous Windows, installez d'abord `uv` via PowerShell, puis lancez `uv tool install mistral-vibe`. Consultez [Installer et configurer](/vibe/code/cli/install-setup) pour le détail des instructions par plateforme.

<SectionTab as="h2" sectionId="step-2">Étape 2 : lancer et configurer</SectionTab>

1. Ouvrez un terminal dans n'importe quel dossier de projet.
2. Lancez :

```bash
vibe
```

![Vibe CLI exécutée dans un terminal](/assets/quickstarts/vibe/home.png)

Au premier lancement, la CLI démarre un **assistant de configuration** pour enregistrer votre clé API. Par défaut, elle ouvre un navigateur pour la connexion à votre compte Mistral ; les identifiants sont stockés localement, vous n’aurez plus à les saisir par la suite.

Pour les fournisseurs hors Mistral ou pour de l'automatisation, collez simplement une clé API. La génération dépend de votre plan :

- **Plan Mistral** (Free, Pro ou supérieur) : <AppLink href="https://chat.mistral.ai/code/extensions" app="studio">Code › Vibe CLI</AppLink>.
- **Accès API à la demande** (mode Free ou plan Scale) : <AppLink href="https://console.mistral.ai/home?profile_dialog=api-keys" app="studio">Clés API</AppLink>.

![Connexion au premier lancement](/assets/quickstarts/vibe/api-key.png)

:::tip
Vous pouvez relancer l'assistant à tout moment avec `vibe --setup`.
:::

:::note
Si vous lancez `vibe` pour la première fois dans un projet non vide contenant un dossier `.vibe/`, la CLI vous demande si vous faites confiance à ce dossier avant de charger une configuration locale. Consultez [Dossiers approuvés](/vibe/code/safety-approvals-permissions#trusted-folders).
:::

<SectionTab as="h2" sectionId="step-3">Étape 3 : envoyer votre premier prompt</SectionTab>

Tapez une demande directement dans le prompt. Par exemple :

> Donnez la liste des fichiers de ce dossier et expliquez le rôle de chacun.

La CLI lit le contexte de votre projet et renvoie les informations pertinentes. Elle peut aussi générer du code, modifier des fichiers et exécuter des commandes shell après votre validation.

![Envoi d’une requête à la CLI Vibe](/assets/quickstarts/vibe/query.png)

Pour exécuter une commande shell via la CLI, préfixez-la par `!`. Par exemple : `!ls` ou `!git status`.

<SectionTab as="h2" sectionId="verify">Vérifier</SectionTab>

La configuration est terminée si :

1. `vibe --version` retourne un numéro de version ;
2. vous êtes connecté (ou votre clé API est enregistrée)
3. la CLI répond à vos prompts de façon contextuelle.
