# Offline / local

Vibe supports local models. Deploy Devstral **on your own infrastructure** and use it **without an internet connection**.

<SectionTab as="h1" sectionId="run-devstral-locally">Run Devstral locally</SectionTab>

We provide weights for Devstral models that you can use to run a local server. Any server with an OpenAI-compatible API works.

:::tip
We recommend using **[Devstral Small 2](https://docs.mistral.ai/models/devstral-2-25-12)** for local usage. [Download the weights](https://huggingface.co/mistralai/Devstral-Small-2-24B-Instruct-2512) on huggingface.
:::

<SectionTab as="h2" variant="secondary" sectionId="pre-requisites">Prerequisites</SectionTab>

Running Devstral locally requires a GPU for best performance:

- **FP8 precision** (~128k context) requires at least an **H100** or **A100 GPU**.
- **4-bit precision** (~32k context) requires at least an **RTX 4090** or **24 GB of VRAM**.

<SectionTab as="h2" variant="secondary" sectionId="deploying">Deploying</SectionTab>

Devstral works with any OpenAI-compatible inference server. We recommend [vLLM](https://github.com/vllm-project/vllm), but you can also use:

- [llama.cpp](https://github.com/ggerganov/llama.cpp)
- [LM Studio](https://lmstudio.ai/)
- [Ollama](https://ollama.com)

To start a server with vLLM:

```bash
vllm serve mistralai/Devstral-Small-2-24B-Instruct-2512 --tool-call-parser mistral --enable-auto-tool-choice --port 8080
```

Depending on your hardware, you may want to add:
- `--tensor-parallel-size`: use multiple GPUs.
- `--dtype`: lower the precision.
- `--max-model-len`: reduce the context length.

<SectionTab as="h2" variant="secondary" sectionId="connect-vibe">Connect Vibe to your server</SectionTab>

Once your server is running, type `/config` in Vibe and select the **"local"** model. Port 8080 is the default.

To switch easily between local and API models, create a reusable preset. See [Change providers and models](introduction/configuration#change-providers).
