Notre modèle d'embeddings excelle dans les tâches de recherche, car il a été entraîné spécifiquement pour la recherche. Les embeddings sont également extrêmement utiles pour implémenter des systèmes de génération augmentée par récupération (RAG), qui utilisent des informations pertinentes extraites d'une base de connaissances pour générer des réponses. À haut niveau, nous transformons une base de connaissances, qu'il s'agisse d'un répertoire local, de fichiers texte ou de wikis internes, en embeddings de texte que nous stockons dans une base de données vectorielle. Ensuite, en fonction de la requête de l'utilisateur, nous récupérons les embeddings les plus similaires, qui représentent les informations pertinentes de la base de connaissances. Enfin, nous transmettons ces embeddings pertinents à un grand modèle de langage pour générer une réponse adaptée à la requête et au contexte de l'utilisateur. Si vous souhaitez en savoir plus sur le fonctionnement des systèmes RAG et comment implémenter un RAG de base, consultez notre [guide](/rag_quickstart) sur ce sujet.
