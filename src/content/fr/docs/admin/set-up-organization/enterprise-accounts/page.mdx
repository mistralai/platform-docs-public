# Comptes Enterprise et Backoffice

Les comptes Enterprise constituent la couche supérieure aux organisations pour les clients Enterprise qui gèrent plusieurs organisations isolées. Utilisez un compte Enterprise lorsque des groupes distincts ont besoin de leurs propres paramètres au niveau organisation, membres, clés API Admin et espaces de travail.

Le Backoffice est l'endroit où les administrateurs BO gèrent la couche des comptes Enterprise. Le panneau Admin reste l'endroit pour administrer une seule organisation.

<SectionTab as="h1" sectionId="when-you-need-an-enterprise-account">Quand utiliser un compte Enterprise</SectionTab>

Les comptes Enterprise sont disponibles dans le plan Enterprise. Ils sont utiles lorsque votre entreprise a besoin d'une séparation plus marquée que celle offerte par les espaces de travail à l'intérieur d'une seule organisation.

Utilisez un compte Enterprise pour regrouper plusieurs organisations lorsque chaque groupe a besoin de ses propres :

- paramètres de l'organisation ;
- membres et modèle d'accès ;
- espaces de travail ;
- clés API Admin ;
- utilisation, facturation ou limites opérationnelles.

Les espaces de travail organisent toujours les équipes, environnements, produits et budgets au sein d'une seule organisation. Les comptes Enterprise organisent plusieurs organisations.

<SectionTab as="h1" sectionId="model">Fonctionnement du modèle</SectionTab>

La hiérarchie d'administration comprend trois couches :

1. **Compte Enterprise** : le compte de niveau supérieur pour les clients Enterprise disposant de plusieurs organisations.
2. **Organisation** : le compte qui contient les membres, les espaces de travail, la facturation, les paramètres d'accès et les stratégies.
3. **Espace de travail** : un environnement partagé au sein d'une organisation, dédié à l'utilisation de produits, aux ressources, aux accès et aux limites.

Le Backoffice gère le compte Enterprise et les organisations qui en dépendent. Le panneau Admin gère une organisation à la fois.

:::note
Un administrateur BO ne remplace pas les rôles d'organisation. Pour administrer une organisation depuis le panneau Admin, l'administrateur BO doit être administrateur de cette organisation.
:::

<SectionTab as="h1" sectionId="open-backoffice">Ouvrir le Backoffice</SectionTab>

Ouvrez <AppLink href="https://backoffice.mistral.ai/">backoffice.mistral.ai</AppLink>.

Un accès administrateur BO est nécessaire pour utiliser le Backoffice. La navigation de gauche inclut :

- **Membres du compte Enterprise** ;
- **Organisations** ;
- **Clés API**.

<SectionTab as="h1" sectionId="enterprise-account-members">Gérer les membres des comptes Enterprise</SectionTab>

Utilisez **Membres du compte Enterprise** pour consulter et gérer les personnes ayant accès au compte Enterprise dans le Backoffice.

La liste des membres affiche :

- nom et e-mail ;
- ID de compte ;
- organisations associées au membre ;
- dernière connexion ;
- actions disponibles.

Vous pouvez ajouter un nouveau membre depuis cette page. Les invitations en attente apparaissent dans la liste avec un statut **Invitation en attente**.

<SectionTab as="h1" sectionId="organizations">Gérer les organisations</SectionTab>

Utilisez **Organisations** pour consulter les organisations dépendant du compte Enterprise. La liste affiche le nom de chaque organisation, l'ID de l'organisation, le nombre de membres et les actions disponibles.

Vous pouvez créer une organisation depuis cette page.

Utilisez **Promouvoir en administrateur** lorsque vous devez administrer l'une des organisations. Cette action promeut l'administrateur BO actuel en administrateur d'organisation pour l'organisation sélectionnée, afin qu'il puisse y accéder depuis le panneau Admin.

<SectionTab as="h1" sectionId="admin-api-keys">Gérer les clés API Admin</SectionTab>

Utilisez **Clés API** pour créer et consulter les clés API Admin des organisations dépendant du compte Enterprise.

Les clés API Admin sont associées à une seule organisation. Une clé peut administrer uniquement l'organisation sélectionnée lors de sa création.

La liste des clés affiche :

- nom de la clé ;
- organisation ;
- valeur masquée de la clé ;
- date de création ;
- date de dernière utilisation ;
- date d'expiration ;
- actions disponibles.

Pour créer une clé API Admin :

1. Ouvrez **Clés API** dans le Backoffice.
2. Cliquez sur **Créez une nouvelle clé**.
3. Saisissez un **Nom**.
4. Sélectionnez l'**Organisation** à laquelle la clé appartient.
5. Définissez éventuellement une date d'**Expiration**.
6. Créez la clé et copiez-la immédiatement.

:::warning
La clé API Admin complète n'est affichée qu'à la création. Si vous la perdez, vous devrez en créer une nouvelle.
:::

<SectionTab as="h1" sectionId="admin-panel-boundary">Ce qui reste dans le panneau Admin</SectionTab>

Utilisez le panneau Admin pour l'administration au niveau organisation :

- paramètres de l'organisation ;
- accès, SSO et vérification de domaine ;
- membres et rôles de l'organisation ;
- espaces de travail et paramètres des espaces de travail ;
- facturation, utilisation et limites ;
- journaux d'audit et contrôles de confidentialité.

Utilisez le Backoffice pour l'administration au niveau du compte Enterprise entre les organisations.
