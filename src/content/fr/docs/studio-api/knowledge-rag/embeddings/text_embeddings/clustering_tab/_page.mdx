Pour cette section, nous allons exploiter les exemples précédents de **Batch et classification** avec le même jeu de données de maladies.

Et si nous ne disposons pas d'étiquettes de maladies ? Une approche pour obtenir des informations à partir des données consiste à utiliser le clustering. Le clustering est une technique d'apprentissage automatique non supervisé qui regroupe des points de données similaires en fonction de leur similarité par rapport à certaines caractéristiques. Dans le contexte des embeddings de texte, nous pouvons utiliser la distance entre chaque embedding comme mesure de similarité, et regrouper les points de données dont les embeddings sont proches les uns des autres dans l'espace de grande dimension.

Puisque nous savons déjà qu'il y a 24 clusters, utilisons l'algorithme K-means avec 24 clusters. Ensuite, nous pouvons inspecter quelques exemples et vérifier si les exemples d'un même cluster sont similaires entre eux. Par exemple, regardez les trois premières lignes du cluster 23. Nous pouvons voir qu'elles sont très similaires en termes de symptômes.

<Tabs groupId="code">
  <TabItem value="python" label="python" default>

```python
from sklearn.cluster import KMeans

model = KMeans(n_clusters=24, max_iter=1000)
model.fit(df['embeddings'].to_list())
df["cluster"] = model.labels_
print(*df[df.cluster==23].text.head(3), sep='\n\n')
```
