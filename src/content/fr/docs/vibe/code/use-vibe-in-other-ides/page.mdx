# Utiliser Vibe dans d’autres IDE

Vibe implémente le **[Agent Client Protocol (ACP)](https://agentclientprotocol.com/)** et est référencé dans le [registre ACP](https://github.com/agentclientprotocol/registry/tree/main/mistral-vibe). Vous pouvez donc exécuter Vibe dans des clients compatibles ACP en dehors du CLI, de l’extension VS Code et de Vibe Code Web.

Cette page explique comment ajouter Vibe à un client compatible ACP, y compris les IDE JetBrains comme IntelliJ IDEA, PyCharm, WebStorm et GoLand. Pour consulter la liste complète des clients, voir l’[annuaire des clients ACP](https://agentclientprotocol.com/get-started/clients).

Dans ces clients, Vibe s’exécute comme un agent dans le chat natif du client. Il utilise le même moteur, la même configuration et la même couche de personnalisation que les interfaces prises en charge, notamment les agents, les compétences et les serveurs MCP.

:::warning
**N’utilisez pas le plugin JetBrains [Mistral Code Enterprise](https://plugins.jetbrains.com/plugin/27493-mistral-code-enterprise).** Il est déprécié, et il n’existe pas d’extension Mistral autonome pour JetBrains. La voie prise en charge consiste à exécuter Vibe via ACP. Pour les interfaces principales, voir [Choisir CLI, VS Code ou session web](/vibe/code/choose-cli-vscode-web-sessions).
:::

<SectionTab as="h2" sectionId="requirements">Avant de commencer</SectionTab>

| Prérequis | Détails |
|---|---|
| **Client compatible ACP** | Un IDE ou un client qui prend en charge les agents ACP, comme un IDE JetBrains avec le plugin AI Assistant. |
| **Compte Mistral** | Requis pour créer ou gérer des clés API. |
| **Plan ou clé API** | Un [plan Mistral](https://mistral.ai/pricing), comme Free, Pro ou supérieur, ou un plan API avec une clé que vous gérez vous-même. Des limites d’utilisation et de débit s’appliquent. |

<SectionTab as="h2" sectionId="general-setup">Ajouter Vibe comme agent ACP</SectionTab>

Les étapes exactes dépendent de votre client, mais le déroulé est le même pour tous les IDE compatibles ACP :

1. **Ajoutez Vibe comme agent ACP** depuis le registre d’agents ou les paramètres de votre client.
2. **Sélectionnez Mistral Vibe** comme agent actif dans le chat du client.
3. **Authentifiez-vous** avec votre clé API Mistral lorsque vous y êtes invité.

Une fois connecté, le chat du client s’exécute avec l’agent Vibe et peut utiliser votre projet ouvert et le contexte de l’éditeur.

<SectionTab as="h2" sectionId="jetbrains">Configurer JetBrains AI Assistant</SectionTab>

JetBrains AI Assistant est un client compatible ACP qui héberge le registre d’agents ACP.

Avant d’ajouter Vibe, vérifiez que vous avez :

- **IDE JetBrains 2025.3.2 ou version ultérieure**. Le registre d’agents ACP est disponible à partir de cette version.
- **Plugin AI Assistant**. Il héberge l’intégration ACP. S’il n’est pas installé, ouvrez l’icône `AI Chat` dans la barre d’outils de droite et cliquez sur `Install Plugin`, ou installez `AI Assistant` depuis `Settings > Plugins`.

Une fois le plugin AI Assistant installé, ajoutez Vibe :

1. Ouvrez `Settings` (`Cmd+,` sur macOS, `Ctrl+Alt+S` sur Windows ou Linux).
2. Allez dans `Tools > AI Assistant > Agents`.
3. Dans `Search agents`, recherchez `Mistral Vibe`.
4. Sur l’entrée `Mistral Vibe`, cliquez sur `Install`. L’IDE télécharge l’agent et tout runtime requis.
5. Ouvrez la fenêtre d’outils `AI Chat`.
6. Choisissez `Mistral Vibe` dans le sélecteur d’agent en bas de la zone de saisie du chat.

<SectionTab as="h2" sectionId="authenticate">Authentifier Vibe</SectionTab>

La première fois que Vibe a besoin d’identifiants, il lance le flux de configuration de l’agent intégré. Fournissez votre clé API Mistral lorsque vous y êtes invité. Vibe stocke les identifiants dans son répertoire personnel (`~/.vibe/`) et les réutilise d’une session à l’autre.

Si vous avez déjà authentifié le [CLI Vibe](/vibe/code/cli/install-setup) ou l’[extension VS Code](/vibe/code/vs-code-extension/install-authenticate) sur la même machine, Vibe réutilise la même configuration et les mêmes identifiants.

<SectionTab as="h2" sectionId="verify">Vérifier que Vibe est prêt</SectionTab>

Votre configuration est prête lorsque :

- `Mistral Vibe` est disponible comme agent dans le chat de votre client.
- Vous pouvez envoyer une invite et recevoir une réponse.
- Vibe peut utiliser le projet actif et le contexte de l’éditeur.

Essayez une première invite simple :

```text
Explique le fichier actuellement ouvert.
```

<SectionTab as="h2" sectionId="shared-config">Utiliser la configuration partagée</SectionTab>

Vibe dans un client ACP partage la même configuration et la même couche de personnalisation que le CLI et l’extension VS Code : agents, compétences, serveurs MCP et profils définis dans `~/.vibe/` et dans les répertoires `.vibe/` propres à chaque projet. Pour en savoir plus, voir [Configuration](/vibe/code/cli/configuration).
