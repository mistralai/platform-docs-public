# Clés API et profils

Vibe Code CLI nécessite une **clé API Mistral** pour appeler les modèles hébergés. Cette page décrit comment créer une clé, l’ajouter au CLI et changer de clé selon les comptes ou fournisseurs.

:::info
**La connexion via navigateur est activée par défaut** lorsque votre configuration cible un modèle avec un fournisseur Mistral. Ce flux gère automatiquement la création et le stockage des identifiants ; aucune clé API n’est nécessaire pour les configurations Mistral typiques. Cette page reste pertinente pour les fournisseurs non-Mistral, l’automatisation, le dépannage ou si vous souhaitez utiliser une clé API personnalisée.
:::

<SectionTab as="h2" sectionId="where-to-create">Où créer votre clé API</SectionTab>

L’emplacement de création de votre clé API dépend de votre plan :

- **Utilisateurs d’un plan Mistral** (Free, Pro ou supérieur, y compris les plans Pro distribués via Google, Apple, Free Mobile ou Orange) : créez la clé depuis [Code › Vibe CLI](https://chat.mistral.ai/code/extensions).
- **Accès API à l’utilisation (Free mode ou plan Scale)** : créez la clé depuis [Clés API](https://console.mistral.ai/home?profile_dialog=api-keys).

:::tip
**Sécurisez vos clés** :

- **Utilisez la clé la moins privilégiée** possible pour la tâche.
- **Faites tourner régulièrement les clés** et révoquez celles non utilisées dans la console.
- **Ne collez jamais de clés API** dans un prompt, des journaux, captures d’écran ou tickets de support.
:::

<SectionTab as="h2" sectionId="give-key">Fournir la clé API au CLI</SectionTab>

Le CLI accepte **trois méthodes** pour fournir la clé, classées de la plus prioritaire à la moins prioritaire.

<SectionTab as="h3" variant="secondary" sectionId="interactive-setup">1. Configuration interactive (recommandée)</SectionTab>

Lors du premier lancement, le CLI lance un assistant de configuration s’il ne trouve pas de clé :

```bash
vibe
```

Vous pouvez aussi relancer la configuration à tout moment :

```bash
vibe --setup
```

L’assistant enregistre la clé dans `~/.vibe/.env` pour les prochaines sessions.

<SectionTab as="h3" variant="secondary" sectionId="env-var">2. Variable d’environnement</SectionTab>

Exportez la clé dans votre shell :

```bash
export MISTRAL_API_KEY="your_mistral_api_key"
```

Les variables d’environnement ont la priorité sur les valeurs enregistrées dans `~/.vibe/.env`.

<SectionTab as="h3" variant="secondary" sectionId="env-file">3. Fichier `.env`</SectionTab>

Vous pouvez également modifier directement `~/.vibe/.env` :

```bash
MISTRAL_API_KEY=your_mistral_api_key
```

Le CLI charge automatiquement `~/.vibe/.env` au démarrage.

:::note
Le fichier `.env` est réservé **aux identifiants**. Pour la configuration générale du CLI, utilisez `config.toml`. Voir [Configuration](/vibe/code/cli/configuration).
:::

<SectionTab as="h2" sectionId="switch-providers">Changer de compte ou de fournisseur</SectionTab>

Le CLI gère plusieurs fournisseurs et modèles via des **presets** dans `config.toml`. Utilisez-les pour, par exemple, basculer entre une clé Mistral pour la production et une clé d’un autre fournisseur compatible OpenAI pour les tests.

Définissez un preset de fournisseur et un preset de modèle :

```toml
[[providers]]
name = "openrouter"
api_base = "https://openrouter.ai/api/v1"
api_key_env_var = "OPENROUTER_API_KEY"
api_style = "openai"
backend = "generic"

[[models]]
name = "mistralai/devstral-2512:free"
provider = "openrouter"
alias = "devstral-openrouter"
temperature = 0.2
input_price = 0.0
output_price = 0.0
```

Définissez le modèle actif :

```toml
active_model = "devstral-openrouter"
```

Puis exportez la clé API correspondante :

```bash
export OPENROUTER_API_KEY="your_openrouter_api_key"
```

Vous pouvez aussi modifier le modèle actif à tout moment dans le CLI avec `/config` ou `/model`.

<SectionTab as="h2" sectionId="plan-billing">Informations sur les plans et la facturation</SectionTab>

Les plans Vibe Pro incluent un **crédit Vibe mensuel**. Une fois celui-ci consommé, la suite dépend d’un interrupteur dans le Panneau Admin sous [Abonnements › abonnement Vibe](https://admin.mistral.ai/subscriptions) :

- **Interrupteur désactivé (valeur par défaut)** : Vibe est désactivé jusqu’à la prochaine période de facturation.
- **Interrupteur activé** : Vibe reste accessible, toute utilisation supplémentaire est facturée à l’utilisation.

Cet interrupteur est **désactivé par défaut** sur toutes les souscriptions et **ne peut pas être activé sur les plans Vibe Pro souscrits via des partenaires** (Google, Apple, Free Mobile, Orange). Sur ces plans, Vibe est désactivé lorsque le crédit inclus est atteint.
