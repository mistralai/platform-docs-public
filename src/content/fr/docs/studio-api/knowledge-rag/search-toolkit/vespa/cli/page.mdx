La CLI `mistral-vespa` gère le cycle de vie des applications Vespa : de la génération des migrations au déploiement et aux tests.

:::tip
Exécutez `uv run mistral-vespa --help` pour obtenir la liste complète des commandes et options disponibles.
:::

<SectionTab as="h2" sectionId="generate-migration">`generate-migration`</SectionTab>

Crée un squelette de migration horodaté.

```bash
uv run mistral-vespa generate-migration <name> [--app-dir <path>]
```

| Argument/Option | Par défaut | Description |
|---|---|---|
| `name` | requis | Nom de la migration, utilisé pour le fichier et la classe de base |
| `--app-dir` | `./vespa_app` | Répertoire racine de l'application Vespa |

<SectionTab as="h2" sectionId="migrate">`migrate`</SectionTab>

Construit l'application à partir des migrations et la déploie. Prise en charge du déploiement sur un seul nœud (localhost) ou sur Kubernetes.

```bash
uv run mistral-vespa migrate [--app-dir <path>] [options]
```

| Option | Par défaut | Description |
|---|---|---|
| `--app-dir` | `./vespa_app` | Répertoire racine de l'application Vespa |
| `--dry-run` | `false` | Prévisualiser les changements sans les appliquer |
| `--config-server` | auto | URL du serveur de configuration (hôte:port). Par défaut : `http://localhost:19071` en local, ou une URL de service in-cluster pour K8s |
| `--query-port` | `8080` | Port de requête côté hôte à interroger après le déploiement |

**Options Kubernetes :**

| Option | Par défaut | Description |
|---|---|---|
| `--k8s-namespace` | — | Namespace Kubernetes (obligatoire avec `--k8s-instance`) |
| `--k8s-instance` | — | Instance Kubernetes (obligatoire avec `--k8s-namespace`) |
| `--k8s-context` | — | Contexte Kubernetes. Nécessite `--k8s-namespace`, `--k8s-instance` et `--config-server` |

**Exemples :**

```bash
# Docker local
uv run mistral-vespa migrate --config-server http://localhost:19071

# Kubernetes (in-cluster, ex : CI)
uv run mistral-vespa migrate \
  --k8s-namespace my-project \
  --k8s-instance vespa

# Kubernetes (station de travail avec kubectl port-forward)
uv run mistral-vespa migrate \
  --k8s-namespace my-project \
  --k8s-instance vespa \
  --k8s-context staging \
  --config-server http://localhost:19071
```

<SectionTab as="h2" sectionId="generate">`generate`</SectionTab>

Écrit les fichiers du package applicatif sur le disque depuis les migrations (pour inspection ou validation CI). Non utilisé pour le déploiement.

```bash
uv run mistral-vespa generate --path <output-dir> [--app-dir <path>]
```

| Option | Par défaut | Description |
|---|---|---|
| `--path` | requis | Répertoire dans lequel écrire les fichiers du package applicatif |
| `--app-dir` | `./vespa_app` | Répertoire racine de l'application Vespa |

**Structure de sortie :**

```
<output-dir>/
├── schemas/
│   └── <document_type>.sd
└── search/
    └── query-profiles/
        ├── <query_profile_name>.xml
        └── types/root.xml
```

<SectionTab as="h2" sectionId="bruno">`bruno`</SectionTab>

Génère les fichiers de test API de Bruno à partir des migrations.

```bash
uv run mistral-vespa bruno --query-url <url> --document-url <url> [--app-dir <path>]
```

| Option | Par défaut | Description |
|---|---|---|
| `--query-url` | requis | URL de l'endpoint de requête Vespa |
| `--document-url` | requis | URL de l'endpoint de document/feed Vespa |
| `--app-dir` | `./vespa_app` | Répertoire racine de l'application Vespa |

<SectionTab as="h2" sectionId="local-up">`local up`</SectionTab>

Lance un conteneur Vespa local via Docker.

```bash
uv run mistral-vespa local up [--query-port <port>] [--config-port <port>] [--name <name>]
```

| Option | Par défaut | Description |
|---|---|---|
| `--query-port` | `8080` | Port hôte mappé sur la requête Vespa |
| `--config-port` | `19071` | Port hôte mappé sur le serveur de configuration Vespa |
| `--name` | `vespa` | Nom du conteneur Docker |

<SectionTab as="h2" sectionId="local-down">`local down`</SectionTab>

Arrête le conteneur Vespa local et supprime les volumes.

```bash
uv run mistral-vespa local down [--name <name>]
```

<SectionTab as="h2" sectionId="local-status">`local status`</SectionTab>

Vérifie si le conteneur Vespa local est en cours d’exécution.

```bash
uv run mistral-vespa local status [--name <name>]
```
