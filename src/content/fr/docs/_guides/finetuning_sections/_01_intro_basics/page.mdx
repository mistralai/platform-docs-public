## Principes de base du fine-tuning

Le fine-tuning permet de personnaliser un modèle de langage de grande taille (LLM) pour un cas d'usage spécifique. En entraînant un LLM pré-entraîné sur un jeu de données étiqueté pour une tâche particulière, le fine-tuning peut améliorer les performances du modèle. Vous pouvez finir d'affiner un grand modèle pour des tâches complexes ou dissemblables, ou fine-tuner un modèle plus petit afin d'atteindre des performances proches de celles d'un modèle plus grand, avec une latence et un coût réduits. Les résultats varient selon le cas d'usage.

Mistral propose une API de fine-tuning dans [Studio](https://console.mistral.ai/) pour les modèles à poids ouverts et commerciaux. Pour des expérimentations nécessitant un contrôle de plus bas niveau, utilisez le dépôt ouvert [mistral-finetune](https://github.com/mistralai/mistral-finetune/). Nous recommandons l'API pour la plupart des travaux de fine-tuning, car elle ne nécessite pas de GPU, prend en charge plusieurs options de configuration et est compatible avec les modèles commerciaux.
