# Créer un Skill dans Studio

Utilisez les <AppLink href="https://console.mistral.ai/build/skills" app="studio">Skill</AppLink> pour regrouper une méthode, une checklist ou une procédure réutilisable qu’un agent peut appliquer lorsqu’une tâche correspond.

Un Skill est différent d’un prompt enregistré. Un prompt stocke un texte ou des instructions réutilisables. Un Skill stocke le moment d’utilisation d’une méthode, la méthode elle-même et des fichiers optionnels comme des exemples, des modèles ou du matériel de référence.

À la fin de la procédure, vous disposerez d’un Skill avec une description claire, des instructions, des paramètres de visibilité et un plan de test.

**Temps estimé** : ~10 minutes

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

- Un compte Studio actif.
- Une tâche répétitive que vous souhaitez standardiser. Les premiers candidats idéaux sont les checklists de relecture, les formats de rapports ou les étapes d’analyse spécifiques à un domaine.

<SectionTab as="h2" sectionId="step-1">Étape 1 : Ouvrir les Skills</SectionTab>

1. Ouvrez <AppLink href="https://console.mistral.ai" app="studio">Studio</AppLink>.
2. Dans la barre latérale de gauche, ouvrez **Construire** > **Skills**.
3. Cliquez sur **Nouvelle tâche**.

<SectionTab as="h2" sectionId="step-2">Étape 2 : Définir quand s’applique le Skill</SectionTab>

Remplissez les détails du Skill. La description est le champ le plus important, car elle indique à l’agent quand charger le Skill.

| Champ | Exemple |
|---|---|
| **Titre** | `relecture-risque-contrat` |
| **Description** | `À utiliser lors de la relecture de contrats clients pour identifier les dates de renouvellement, les clauses de résiliation, les plafonds de responsabilité ou les approbations manquantes.` |

Rédigez la description sous forme de condition :

- **Imprécis** : *« Aide à gérer les contrats. »*
- **Précis** : *« À utiliser lors de la relecture de contrats clients pour identifier les dates de renouvellement, les délais de préavis, les droits de résiliation, les plafonds de responsabilité et les approbations manquantes. »*

<SectionTab as="h2" sectionId="step-3">Étape 3 : Rédiger les instructions</SectionTab>

Dans le champ instructions, décrivez la procédure à suivre par le Skill :

> Lorsqu’un contrat client est fourni :
> 1. Extraire la date de renouvellement, le délai de préavis, les droits de résiliation et le plafond de responsabilité.
> 2. Signaler les approbations manquantes ou les conditions commerciales inhabituelles.
> 3. Retourner un tableau récapitulatif avec **Risque**, **Preuves** et **Action recommandée**.
>
> Ne pas inventer des éléments manquants. Marquez les valeurs inconnues comme `Non trouvé`.

Gardez le Skill axé sur une seule tâche. Si vous avez besoin de plusieurs procédures sans lien entre elles, créez des Skills distincts avec des descriptions différentes.

<SectionTab as="h2" sectionId="step-4">Étape 4 : Ajouter des fichiers</SectionTab>

Joignez des fichiers lorsque le Skill nécessite des exemples ou du matériel de référence :

- Une checklist dans `checklist.md`.
- Un modèle de sortie préféré dans `template.md`.
- Un exemple de contrat relecturé avec des commentaires.

Les fichiers de support sont soumis à une limite de nombre de fichiers. Si vous avez besoin de plus de matériel, divisez la méthode en compétences plus petites ou utilisez des fichiers de référence ciblés explicitement mentionnés dans les instructions. Consultez [Structurer de grandes compétences avec un chargement progressif](https://agentskills.io/skill-creation/best-practices#structure-large-skills-with-progressive-disclosure) pour obtenir des conseils.

<SectionTab as="h2" sectionId="step-5">Étape 5 : enregistrer le Skill</SectionTab>

1. Cliquez sur **Créer un Skill**.
2. Ouvrez le Skill enregistré depuis la liste des `Skills`.
3. Vérifiez que la page des détails affiche le titre, la description, les instructions, les fichiers joints et la première version.

<SectionTab as="h2" sectionId="step-6">Étape 6 : choisir la visibilité et la disponibilité dans Vibe</SectionTab>

Depuis la page des détails du Skill, contrôlez où le Skill est disponible :

1. Cliquez sur `Partager`.
2. Choisissez de garder le Skill privé ou de le partager avec l’ensemble de l’`espace de travail`.
3. Si vous souhaitez utiliser le Skill dans Vibe, cliquez sur `Publier dans Vibe`.

Les Skills privés ne sont visibles que par vous. Les Skills de l’espace de travail peuvent être trouvés et utilisés par les membres de cet espace.

Le partage et la disponibilité dans Vibe sont deux contrôles distincts : `Partager` détermine qui peut accéder au Skill dans Studio, tandis que `Publier dans Vibe` rend le Skill disponible dans Vibe.

<SectionTab as="h2" sectionId="step-7">Étape 7 : tester le Skill</SectionTab>

1. Ouvrez la surface où vous souhaitez utiliser le Skill.
2. Exécutez une tâche réaliste correspondant à la description.
3. Comparez le résultat au format attendu.

Si vous avez cliqué sur `Publier dans Vibe`, ouvrez Vibe Work et exécutez le test.

<SectionTab as="h2" sectionId="step-8">Étape 8 : créer une nouvelle version</SectionTab>

Créez une nouvelle version lorsque vous souhaitez modifier le Skill sans remplacer la version actuelle utilisée par des agents ou des workflows existants. Studio génère la nouvelle version après que vous ayez enregistré les modifications.

1. Ouvrez le Skill depuis la liste des **Skills**.
2. Mettez à jour les instructions ou les fichiers joints.
3. Ajoutez des notes de version expliquant les modifications, comme `Ajoute les vérifications des plafonds de responsabilité` ou `Met à jour le format du tableau récapitulatif`.
4. Cliquez sur **Enregistrer**.
5. Vérifiez que l’historique des versions affiche la nouvelle version.

Les anciennes versions restent disponibles pour consultation et restauration. Testez la nouvelle version avant de la déployer en production.

<SectionTab as="h2" sectionId="verify">Vérification</SectionTab>

Votre Skill est prêt si :

- Il apparaît dans la liste des Skills.
- La description explique clairement quand appliquer le Skill.
- Le résultat respecte la procédure, le format et les contraintes que vous avez définis.
- Les fichiers joints sont pertinents et référencés par les instructions.
- Sa visibilité est correcte : privés pour vous ou partagés avec l’espace de travail.
- Le Skill est publié dans Vibe si vous devez l’utiliser dans Vibe Work.
- L’historique des versions affiche la version initiale et la version mise à jour.
