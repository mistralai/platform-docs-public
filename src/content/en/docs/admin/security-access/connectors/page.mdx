# Manage Connectors

Organization admins manage Connectors from the Admin Console. Use this page to connect organization bots, review Connector details, and control which Connector tools are available to users in Vibe and Studio.

:::info
Connector administration is moving to the Admin Console. Connector setup that previously happened in Vibe will be managed from Admin.
:::

<SectionTab as="h1" sectionId="open-connectors">Open Connectors in Admin</SectionTab>

1. Open <AppLink href="https://admin.mistral.ai/organization/connectors" app="admin">Connectors</AppLink> in the Admin Console.
2. In the Organization section, click `Connectors`.

The page has two main areas:

- `App Connections`: connect organization-level bots.
- `All Connectors`: manage Connectors and their tools for Vibe and Studio.

<SectionTab as="h1" sectionId="app-connections">Connect organization-level bots</SectionTab>

Use `App Connections` to connect services once at the Organization level. Once connected, all members can use them through Mistral without signing in individually.

| App | What it does |
|-----|--------------|
| `GitHub` | Search repositories, review issues, and manage pull requests in GitHub. |
| `Slack` | Search messages, read channels, send messages, and manage canvases in Slack. |

To connect an app, click `Connect to GitHub` or `Connect to Slack` and complete the authorization flow.

<SectionTab as="h1" sectionId="all-connectors">Manage all Connectors</SectionTab>

`All Connectors` lists the Connectors and tools that can be used in Vibe and Studio. Open a Connector to review its setup and control tool access.

Each Connector page includes three tabs:

- `Overview`: review metadata, visibility, authentication, and available content.
- `Credentials`: review or add credentials for the Connector.
- `Permissions`: choose which tools can be used by the model.

<SectionTab as="h2" variant="secondary" sectionId="overview-tab">Review Connector details</SectionTab>

The `Overview` tab shows basic information about the Connector, including:

| Field | Example |
|-------|---------|
| `Visibility` | `Organization` |
| `Authentication` | `OAuth2` |
| `Origin` | `Featured` |
| `Capabilities` | `Read-only` |
| `Connector name` | `atlassian` |
| `Connector ID` | `0198e70f-57b0-77f6-a752-0a7f5ea2da35` |

If you aren't connected, the `Includes` section shows `You're not connected`. Click `Connect` to authenticate and see the tools.

<SectionTab as="h2" variant="secondary" sectionId="credentials-tab">Manage credentials</SectionTab>

The `Credentials` tab lists the credentials available for the Connector. If there are no credentials yet, click `Connect` to start using the Connector.

:::tip
Use organization-level app connections for shared bot access. Use Connector credentials when users or workspaces need a specific account connection.
:::

<SectionTab as="h2" variant="secondary" sectionId="permissions-tab">Set Connector permissions</SectionTab>

The `Permissions` tab controls which tools can be used by the model.

| Permission | Effect |
|------------|--------|
| `Allowed` | All tools are available for users in this Workspace. |
| `Restricted` | Only selected tools are available for users in this Workspace. |
| `Blocked` | The Connector is turned off for users in this Workspace. |

Use `Restricted` when a Connector includes tools that should not be available to every workflow or chat. Use `Blocked` when a Connector should not be used in a Workspace.

<SectionTab as="h1" sectionId="admin-vs-studio">Admin vs. Studio setup</SectionTab>

Configure and govern Connectors in the Admin Console. Use [Studio Connectors](/studio-api/connectors) when you need developer-facing Connector docs, API examples, or the Connector Debugger.

We will update the docs as Connector administration finishes moving from Vibe to Admin.
