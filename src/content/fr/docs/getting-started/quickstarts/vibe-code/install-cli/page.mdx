# Installez le Vibe CLI et envoyez votre premier prompt

Installez le <AppLink href="/vibe/code/overview">Vibe CLI</AppLink>, configurez votre thème et votre clé API, puis envoyez votre premier prompt directement depuis le terminal.

- **Installation en une ligne :** lancez le CLI en quelques secondes sur macOS ou Linux.
- **Sélection du thème :** choisissez le style d’interface de votre CLI interactif.
- **Configuration de la clé API :** connectez le CLI à votre compte Mistral lors du premier lancement.

À la fin, vous aurez installé, configuré et utilisé le Vibe CLI pour envoyer des prompts via votre terminal.

**Temps estimé :** ~5 minutes

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

- macOS, Linux ou Windows.
- Python 3.12 ou version ultérieure (pour l'installation manuelle).
- Un compte Mistral avec une clé API. Le Vibe CLI fonctionne avec tout plan incluant Vibe Code.

:::tip
Sur une autre interface ? Consultez les guides pour [l’extension VS Code](/vibe/code/vs-code-extension/install-authenticate) ou pour [Vibe Code Web](/vibe/code/vibe-code-web/get-started).
:::

<SectionTab as="h2" sectionId="step-1">Étape 1&nbsp;: Installez le CLI</SectionTab>

**Commande unique (macOS/Linux)**

Exécutez ceci dans votre terminal :

```bash
curl -LsSf https://mistral.ai/vibe/install.sh | bash
```

Le programme d’installation vérifie la présence de `uv`, installe ou met à jour `mistral-vibe` et rend les commandes `vibe` et `vibe-acp` disponibles, sous réserve que votre `PATH` soit configuré.

**Installation manuelle (macOS/Linux/Windows)**

Si vous préférez une installation manuelle ou si vous êtes sur Windows :

1. Vérifiez que Python 3.12+ est installé : `python3 --version`.
2. Installez avec le gestionnaire de paquets de votre choix :

```bash
# Avec uv (recommandé)
uv tool install mistral-vibe

# Avec pip
pip install mistral-vibe
```

Pour Windows, installez d'abord `uv` via PowerShell, puis lancez `uv tool install mistral-vibe`. Consultez [Installer et configurer](/vibe/code/cli/install-setup) pour les instructions complètes par plateforme.

<SectionTab as="h2" sectionId="step-2">Étape 2&nbsp;: Démarrage et configuration</SectionTab>

1. Ouvrez un terminal dans n’importe quel dossier de projet.
2. Exécutez&nbsp;:

```bash
vibe
```

![Vibe CLI lancé dans un terminal](/assets/quickstarts/vibe/home.png)

Au premier lancement, le CLI vous invite à configurer deux paramètres :

3. **Choisissez un thème.** Sélectionnez votre style d’interface préféré. Vous pourrez le modifier plus tard avec la commande `/theme`.

![Sélection du thème préféré](/assets/quickstarts/vibe/theme.png)

4. **Saisissez votre clé API.** Générez une clé depuis la page <AppLink href="https://console.mistral.ai/codestral/cli" app="studio">Codestral API keys</AppLink> et collez-la dans le terminal.

![Saisie de la clé API lors du premier démarrage](/assets/quickstarts/vibe/api-key.png)

Le CLI la stocke localement. Inutile de la ressaisir par la suite.

<SectionTab as="h2" sectionId="step-3">Étape 3&nbsp;: Envoyer votre premier prompt</SectionTab>

Saisissez une requête directement dans le champ de prompt. Par exemple&nbsp;:

> Listez les fichiers de ce dossier et expliquez la fonction de chacun.

Le CLI lit le contexte de votre projet et répond avec les informations pertinentes. Il peut également générer du code, modifier les fichiers et exécuter des commandes shell sur validation.

![Envoi d’une requête au Vibe CLI](/assets/quickstarts/vibe/query.png)

Pour exécuter une commande shell directement depuis le CLI, ajoutez un préfixe `!`. Par exemple&nbsp;: `!ls` ou `!git status`.

<SectionTab as="h2" sectionId="verify">Vérification</SectionTab>

La configuration est terminée si&nbsp;:

1. `vibe --version` affiche un numéro de version.
2. Le thème et la clé API sont configurés.
3. Le CLI répond aux prompts avec des réponses adaptées au contexte.
