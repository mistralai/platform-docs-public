# Ajoutez du contexte avec les Bibliothèques

Les bibliothèques sont des **bases de connaissances persistantes** que vous construisez à partir de vos propres documents et pages web.

Joignez une bibliothèque à n'importe quelle tâche et Work **recherche son contenu en temps réel**, fournissant des réponses **issues de vos données** avec des références directes au matériel source.

:::info
Contrairement aux [imports de fichiers](/vibe/work/files-and-canvas), qui ne durent qu'une seule conversation, les bibliothèques sont persistantes et accessibles à chaque session. Un seul import suffit pour réutiliser vos documents partout.
:::

<SectionTab as="h2" sectionId="use-cases">Cas d'usage courants</SectionTab>

Les bibliothèques sont idéales si vous souhaitez retrouver un même ensemble de documents dans plusieurs tâches :

- **Documentation interne** : importez le guide de votre entreprise, les politiques informatiques ou RH et laissez votre équipe poser des questions en langage naturel.
- **Spécifications produit et notes de version** : regroupez les documents techniques dans une bibliothèque ; ainsi, produit, ventes et support disposent d'une source de référence unique.
- **Rapports et recherches** : créez une bibliothèque à partir de rapports sectoriels, livres blancs ou articles académiques pour les interroger sans relire des centaines de pages.
- **Connaissance client ou projet** : pour chaque client ou projet, constituez une bibliothèque avec contrats, briefs et comptes-rendus pour y accéder rapidement.
- **Matériel d'onboarding** : rassemblez vos supports d'onboarding dans une bibliothèque, afin que les nouveaux collaborateurs puissent poser leurs questions à Work en langage naturel.

<SectionTab as="h2" sectionId="creating-a-library">Créer et alimenter une bibliothèque</SectionTab>

1. Ouvrez la page `Bibliothèques` via la barre latérale.
2. Cliquez sur `Nouvelle bibliothèque`.
3. Donnez-lui un nom descriptif. Ce nom est visible par d'autres si vous partagez la bibliothèque : privilégiez un intitulé parlant (par exemple, `Specs produit T1` ou `Onboarding ingénierie`).

<SectionTab as="h3" variant="secondary" sectionId="uploading-documents">Ajouter des documents</SectionTab>

Glissez-déposez des fichiers dans la bibliothèque, ou cliquez sur `Importer` pour sélectionner des fichiers depuis votre appareil. Vous pouvez importer jusqu'à **100 fichiers à la fois**, dont la taille maximale est de **100 Mo par fichier**.

Un indicateur d'état suit la progression du traitement. La plupart des documents standards (quelques Mo) sont traités en quelques secondes à quelques minutes. Les fichiers plus volumineux ou complexes nécessitent plus de temps.

<SectionTab as="h3" variant="secondary" sectionId="adding-web-pages">Ajouter des pages web</SectionTab>

Vous pouvez également ajouter des pages web externes à une bibliothèque :

1. Cliquez sur `Page web` dans la vue Bibliothèque.
2. Saisissez une ou plusieurs URL.
3. Cliquez sur `Indexer`.

:::note
Seul le contenu textuel de la page est indexé. Les pages enfants, images et graphiques ne sont pas inclus. Si vous souhaitez conserver l'aspect visuel intégral, téléchargez la page et importez-la comme fichier.
:::

<SectionTab as="h2" sectionId="supported-formats">Formats pris en charge</SectionTab>

Les bibliothèques acceptent les mêmes formats que les [fichiers joints](/vibe/work/files-and-canvas) :

| Catégorie | Formats |
|----------|---------|
| **Documents** | PDF, Word (`.docx`, `.doc`), PowerPoint (`.pptx`, `.ppt`), ODT, EPUB, RTF |
| **Tableurs** | Excel (`.xlsx`, `.xls`), CSV, ODS, Numbers |
| **Images** | PNG, JPEG, WebP, GIF |
| **Texte et balisage** | TXT, Markdown, RST, LaTeX |
| **Formats de données** | JSON, JSONL, XML, YAML |
| **Fichiers de code** | Python, JavaScript/TypeScript, Java, Go, Rust, C/C++, Ruby, PHP, SQL, etc. |
| **E-mail** | EML, MSG |

<SectionTab as="h2" sectionId="using-libraries">Utiliser des bibliothèques dans les tâches</SectionTab>

<SectionTab as="h3" variant="secondary" sectionId="attaching-a-library">Joindre une bibliothèque à une conversation</SectionTab>

Dans une conversation, cliquez sur le bouton `+`, sélectionnez `Bibliothèques`, puis recherchez la bibliothèque souhaitée ou créez-en une nouvelle. Vous pouvez joindre **plusieurs bibliothèques** à une même conversation.

<SectionTab as="h3" variant="secondary" sectionId="querying-documents">Interroger vos documents</SectionTab>

Une fois attachée, Work recherche dans tous les documents de la bibliothèque pour répondre à la tâche. Posez vos questions naturellement :

- *« Que dit notre politique de remboursement sur les commandes à l'international ? »*
- *« Résume les conclusions clés de l'analyse du marché T2. »*
- *« Trouve toutes les références à la conservation des données dans les documents de conformité. »*

:::tip
Pour interroger un document spécifique au lieu de l'intégralité de la bibliothèque, cliquez sur `+`, sélectionnez `Importer des fichiers`, choisissez votre bibliothèque et le document voulu. Vous pouvez également mentionner le nom du fichier dans votre prompt pour cibler la recherche de Work.
:::

<SectionTab as="h3" variant="secondary" sectionId="verifying-sources">Vérifier les sources</SectionTab>

Chaque réponse comprend des **notes de bas de page numérotées** renvoyant vers le document source. Cliquez sur une note pour accéder directement au passage concerné. Pour voir toutes les références utilisées dans une réponse, cliquez sur le bouton `Sources` en bas du message.

Cela facilite la vérification des réponses dans vos documents originaux avant de les utiliser dans des rapports, e-mails ou présentations.

<SectionTab as="h2" sectionId="sharing">Partage et droits d'accès</SectionTab>

Par défaut, une nouvelle bibliothèque est **privée** (vous seul pouvez la voir et l'utiliser).

<SectionTab as="h3" variant="secondary" sectionId="sharing-specific-people">Partager avec des personnes spécifiques</SectionTab>

1. Ouvrez la page `Bibliothèques`.
2. Sélectionnez la bibliothèque à partager.
3. Cliquez sur `Partager` pour ouvrir la fenêtre de gestion des accès.
4. Recherchez des utilisateurs par nom ou adresse e-mail et attribuez-leur un niveau d'accès :
   - **Collaborateur** : peut utiliser la bibliothèque, ajouter ou supprimer des documents et modifier les paramètres.
   - **Lecteur** : peut utiliser la bibliothèque dans ses propres conversations, mais ne peut pas la modifier.

<SectionTab as="h3" variant="secondary" sectionId="sharing-organization">Partager avec l’ensemble de votre organisation</SectionTab>

Dans la même fenêtre, activez `Organisation entière` et choisissez si tout le monde aura un accès de collaborateur ou de lecteur.

:::note
Les destinataires devront peut-être actualiser la page pour voir les bibliothèques nouvellement partagées s'afficher dans leur liste.
:::

<SectionTab as="h2" sectionId="managing-libraries">Gérer les bibliothèques</SectionTab>

<SectionTab as="h3" variant="secondary" sectionId="viewing-documents">Afficher et télécharger des documents</SectionTab>

Ouvrez une bibliothèque et cliquez sur n'importe quel document de la liste. Le panneau de droite affiche le contenu traité et un résumé généré automatiquement. Cliquez sur `Télécharger` dans le panneau ou passez par le menu trois points pour enregistrer le fichier original.

<SectionTab as="h3" variant="secondary" sectionId="renaming-library">Renommer une bibliothèque</SectionTab>

Renommez une bibliothèque depuis ses paramètres. Le nouveau nom est visible par tous les utilisateurs ayant accès (il peut être nécessaire de rafraîchir l'interface).

<SectionTab as="h3" variant="secondary" sectionId="deleting-library">Supprimer une bibliothèque</SectionTab>

Supprimez une bibliothèque depuis la page `Bibliothèques`. Cette action la fait disparaître définitivement pour tous, avec l'ensemble des documents importés.

:::caution
La suppression d'une bibliothèque est définitive et irréversible. Tous les documents correspondants sont supprimés de nos systèmes.
:::

<SectionTab as="h2" sectionId="limits">Limites et traitement</SectionTab>

- **Aucune limite** sur le nombre de bibliothèques que vous pouvez créer.
- Jusqu'à **100 fichiers** importables à la fois par bibliothèque.
- Taille maximale d'un fichier : **100 Mo**.
- Chaque document consomme une capacité de traitement selon sa taille. Votre quota mensuel est réinitialisé automatiquement et partagé entre toutes vos bibliothèques.

:::info
Si vous atteignez votre limite de traitement mensuelle, les nouveaux imports sont suspendus jusqu'au prochain renouvellement de quota. Les bibliothèques existantes et leur contenu restent disponibles.
:::

<SectionTab as="h2" sectionId="data-privacy">Données et confidentialité</SectionTab>

- Les documents sont stockés de façon sécurisée dans notre infrastructure cloud européenne.
- Le stockage est effectif tant que la bibliothèque existe dans un compte actif. La suppression d'une bibliothèque ou de votre compte déclenche l'effacement permanent de nos systèmes.
- Les politiques d'entraînement varient selon votre plan. Vous pouvez [vous opposer](https://help.mistral.ai/en/articles/455207-can-i-opt-out-of-my-input-or-output-data-being-used-for-training) à l'utilisation de vos données à tout moment.

Pour plus d'informations, consultez notre [politique de confidentialité](https://mistral.ai/terms#privacy-policy) et le [trust center](https://trust.mistral.ai/).
