# Choisissez Chat, Work ou Code

Aperçu de chaque mode Vibe : quand l’utiliser, ce qu’il permet, et comment démarrer.

<SectionTab as="h2" sectionId="work">Work</SectionTab>

**Choisissez Work lorsque vous souhaitez que l’agent réalise le travail, et pas seulement réponde à une question.** Disponible dans les applications web et mobile Vibe avec une interface chat, Work est le mode adapté quand votre tâche implique plusieurs étapes, diverses sources ou des actions dans des outils connectés, et si vous souhaitez valider le résultat avant de l’utiliser.

Exemples de tâches :

- *« Faites-moi un récapitulatif de ce qui s’est passé sur le projet de lancement Q4 cette semaine. »*
- *« Synthétisez ce contrat de 80 pages et signalez les risques de renouvellement. »*
- *« Rédigez des réponses à mes emails non lus de l’équipe juridique. »*
- *« Analysez les 5 principaux concurrents de notre secteur et créez une synthèse d’une page. »*
- *« Trouvez les derniers retours clients dans Notion, classez-les par thème et préparez un support de présentation. »*

Avec Work, vous bénéficiez de :

- **Exécution de tâches via un nombre illimité de [Connecteurs](/vibe/work/connectors)**, dont Gmail, Outlook, Slack, Notion, Linear, GitHub, Atlassian, Google Drive, SharePoint et Stripe.
- Suivi en temps réel de l’avancement et des appels d’outils, avec questions de clarification en cas de consigne ambiguë.
- Validation demandée avant chaque action sensible (envoi d’email, modification ou suppression de données).
- [Skills](/vibe/work/skills) et [Projects](/vibe/work/projects) pour faciliter l’automatisation de tâches répétées.
- [Planification de tâches](/vibe/work/scheduled-tasks) pour exécuter automatiquement une consigne, une fois ou à une fréquence régulière (quotidienne, hebdomadaire, mensuelle ou annuelle).

<SectionTab as="h2" sectionId="code">Code</SectionTab>

**Choisissez Code lorsque la tâche dépend d’un codebase, d’un terminal, d’un IDE ou d’une session de code.** Disponible en CLI, extension VS Code ou sessions distantes via le web, Code est le mode développeur : Vibe lit les fichiers, édite le code, lance des commandes et ouvre des pull requests.

Exemples de tâches :

- *« Expliquez le fonctionnement de l’authentification dans ce repo. »*
- *« Refactorisez `src/api/handlers/` pour utiliser le nouveau modèle de gestion d’erreurs. »*
- *« Ajoutez des tests unitaires à la fonction `parseInvoice`. »*
- *« Lancez la suite de tests, corrigez les échecs et ouvrez une PR. »*
- *« Reprenez la session démarrée sur la CLI depuis le web. »*

Avec Code, vous bénéficiez de :

- **CLI** : interface terminal native, scriptable, idéale pour les repos locaux et l’automatisation.
- **Extension VS Code** : assistance intégrée, édition de fichier et chat dans l’éditeur.
- **Sessions à distance** : sandbox cloud accessible depuis n’importe quel environnement (web, mobile, desktop, IDE, CLI) pour travailler sur un repo GitHub.

<SectionTab as="h2" sectionId="chat">Chat (migration depuis Le Chat)</SectionTab>

**Choisissez Chat pour une conversation rapide au tour par tour, ou si vous dépendez d’une fonctionnalité héritée.** Accessible dans les applications web et mobile Vibe via l’onglet Chat, ce mode conserve l’expérience tour à tour ainsi que les fonctions de Le Chat, de sorte que vos habitudes continuent de fonctionner.

Chacune de ces fonctionnalités est repensée dans Work. Restez sur Chat si vous dépendez du comportement actuel ; basculez vers Work lorsque les nouveautés répondent mieux à vos besoins.


**Fonctionnalités de Chat et équivalents Work :**
| Fonctionnalité Chat | Équivalent Work |
|---|---|
| [Agents](/vibe/chat-legacy/agents) : assistants spécialisés avec leurs propres outils et instructions. | [Skills](/vibe/work/skills) avec un autre modèle de configuration. |
| [Think mode](/vibe/chat-legacy/think-mode) : raisonnement étendu pour questions nécessitant une analyse en profondeur. | Niveau de raisonnement choisi automatiquement par le modèle. |
| [Deep Research](/vibe/chat-legacy/deep-research) : recherche multi-sources avec rapport structuré et référencé. | Bientôt disponible sous forme de [Skill](/vibe/work/skills). |
| [Code Interpreter](/vibe/chat-legacy/code-interpreter) : Python en environnement isolé pour graphiques et analyses de données. | Environnement TypeScript intégré pour la majorité des traitements de données et de visualisation. |
| [Memories](/vibe/chat-legacy/memories) : faits et préférences mémorisés à long terme dans les chats. | Nouveau système de mémoire, limité aux projets. |

:::note
Si votre équipe ou organisation a construit des workflows autour des Agents Le Chat, ils restent utilisables dans Chat. Des migrations vers des Skills, Sub-Agents ou agents personnalisés sont prévues ultérieurement.
:::
