# Déploiements

Un **déploiement** est un groupe nommé de [workers](/studio-api/workflows/getting-started/core_concepts/workers) qui possède un ensemble de définitions de workflow et reçoit toutes les exécutions associées. Les workers rejoignent un déploiement en définissant la variable d'environnement `DEPLOYMENT_NAME` au démarrage. Consultez [Workers > Démarrer un worker](/studio-api/workflows/getting-started/core_concepts/workers#starting-a-worker) pour connaître la commande de lancement.

Tous les workers exécutés avec le même `DEPLOYMENT_NAME` constituent un seul et même déploiement. Ils partagent les mêmes définitions de workflow et reçoivent collectivement toutes les exécutions qui leur sont routées.

:::tip
Définissez `DEPLOYMENT_NAME` par environnement (par exemple, `invoice-service-staging` ou `invoice-service-prod`) afin que les workers de préproduction et de production ne se partagent jamais les mêmes exécutions.
:::

<SectionTab as="h1" sectionId="why-deployments">Pourquoi utiliser des déploiements ?</SectionTab>

Les déploiements vous offrent quatre fonctionnalités clés :

- **Isolement des workers** : Dans un même espace de travail, chaque déploiement reçoit exclusivement les exécutions de ses propres workflows. Plusieurs équipes peuvent faire tourner des workers simultanément sans interférer les unes avec les autres.
- **Routage automatique** : Lorsqu’un seul déploiement actif possède un workflow, les exécutions lui sont automatiquement routées. Aucune configuration manuelle requise.
- **Scalabilité horizontale** : Exécutez plusieurs workers sous le même `DEPLOYMENT_NAME` pour augmenter le débit. La plateforme répartit automatiquement les tâches entre eux.
- **Suivi du cycle de vie** : Un déploiement est considéré comme actif tant qu’au moins un de ses workers a envoyé un [heartbeat](/studio-api/workflows/getting-started/core_concepts/activities#long-running-activities-heartbeats) dans la fenêtre de liveness. Les déploiements inactifs sont exclus du routage.
- **Contrôle d’accès** : Limitez l’enregistrement de workflow à certaines clés API avec les [déploiements renforcés](/studio-api/workflows/managing-workflows-in-production/hardened_deployments).

:::tip
Pour une présentation complète couvrant le routage des exécutions, la détection des conflits et l’API Deployments, voir [Gestion des déploiements](/studio-api/workflows/managing-workflows-in-production/deployments).
:::
