Notre modèle d'embeddings excelle dans les tâches de recherche, car il est entraîné spécifiquement pour cela. Les embeddings sont également extrêmement utiles pour implémenter des systèmes de génération augmentée par récupération (RAG), qui utilisent des informations pertinentes extraites d'une base de connaissances pour générer des réponses. De manière générale, nous encodons une base de connaissances — qu'il s'agisse d'un répertoire local, de fichiers texte ou de wikis internes — sous forme d'embeddings de texte que nous stockons dans une base de données vectorielle. Ensuite, en fonction de la requête de l'utilisateur, nous récupérons les embeddings les plus similaires, qui représentent les informations pertinentes de la base de connaissances. Enfin, nous transmettons ces embeddings pertinents à un grand modèle de langage pour générer une réponse adaptée à la requête et au contexte de l'utilisateur. Si vous souhaitez en savoir plus sur le fonctionnement des systèmes RAG et comment implémenter un RAG basique, consultez notre [guide](rag_quickstart) sur ce sujet.
