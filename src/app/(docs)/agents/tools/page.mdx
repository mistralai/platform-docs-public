# Tools

Agents can use tools to interact with the external world. These can be APIs, databases, or other services, increasing the capabilities of your agent and extending its functionality beyond its own knowledge base and fixed environment.

<SectionTab as="h1" sectionId="tool-types">Tool types</SectionTab>

We provide a variety of different **types** of tools agents can use.
|Type|Description|
|---|---|
|[Built-in](tools/built-in)| These are tools we provide out of the box. They can be called at any point and all execution happens in our internal environment and cloud.|
|[Functions](tools/function_calling)| Custom local tools. These are usually functions defined in your local environment or code that can be called by the agent. Execution happens in your local environment.|
|[MCP servers](tools/mcp)| Following the Model Context Protocol, we provide a compatible SDK that lets you create or use tools from servers that can be called by the agent. Execution happens in your local environment and/or the cloud provider hosting the MCP server.|

<SectionTab as="h2" variant="secondary" sectionId="built-in">Built-in</SectionTab>

Below is a list of the built-in tools currently available.
|Type|Description|
|---|---|
|[Web Search](tools/built-in/websearch)| Searches the web for information. Useful for finding up-to-date information or information that isn't in the agent's knowledge.|
|[Code Interpreter](tools/built-in/code_interpreter)| Runs code and generates plots. Useful for data analysis, visualization, or running code in a sandboxed environment.|
|[Image Generation](tools/built-in/image_generation)| Generates images based on a prompt or description.|
|[Document Library](tools/built-in/document_library)| Provides access to read and search through a large collection of documents. Useful for finding information in documents or PDFs and performing RAG (Retrieval Augmented Generation) to answer questions based on specific information.|
