# Premiers pas avec Work

**Vibe Work** est le mode productivité de Vibe, conçu pour déléguer des tâches complexes et multi-étapes à travers vos applications et outils.

Décrivez le résultat souhaité **en langage naturel** : Work rassemble le contexte, découpe la tâche en étapes, appelle les bons outils, et demande votre validation avant toute action sensible.

Ce guide vous montre comment démarrer une tâche dans Work, lui fournir le bon contexte, suivre sa progression et vérifier le résultat.

<SectionTab as="h2" sectionId="open-work">Ouvrir Work</SectionTab>

1. Ouvrez <AppLink href="https://chat.mistral.ai/work" app="vibe">Work</AppLink>.
2. Si la barre latérale est fermée, cliquez sur l’icône du tiroir ou appuyez sur `Cmd+Shift+B` sous macOS ou `Ctrl+Shift+B` sous Windows et Linux.
3. Dans la barre latérale, sélectionnez **Work** dans le sélecteur de mode.
4. Démarrez votre tâche.

Vibe retient l’onglet sélectionné lors de votre dernière utilisation. Si **Chat** ou **Code** s’ouvre par défaut, choisissez **Work** dans la barre latérale avant de lancer la tâche.

<SectionTab as="h2" sectionId="start-a-task">Démarrer une tâche</SectionTab>

Commencez par indiquer le résultat souhaité. Work analyse votre demande, la découpe en sous-étapes, et exécute les outils nécessaires le cas échéant.

:::info
**Vous gardez la main : **Work affiche la progression et demande une confirmation avant toute action sensible. Voir [Sécurité et validations](/vibe/work/safety-and-approvals) pour plus d’informations.
:::

Un bon premier prompt précise :

- le résultat attendu
- les sources ou outils à prendre en compte
- le public cible
- les contraintes éventuelles : longueur, ton, format, délai…

Quelques exemples de prompts :

- `Fais une recherche sur ce sujet et crée une synthèse d'une page à partager à mon équipe.`
- `Résume ce PDF et extrait les responsables, échéances et actions à mener.`
- `Compare ces deux documents et mets en évidence les différences principales.`
- `Rédige une réponse préliminaire avec les informations ci-jointes.`

<SectionTab as="h2" sectionId="give-work-context">Donner le bon contexte à Work</SectionTab>

Work sélectionne automatiquement les outils et le contexte adaptés à partir de votre prompt. Vous pouvez aussi lui indiquer une source précise si vous souhaitez tout maîtriser.

Work s’appuie sur les capacités suivantes :

- [Connecteurs](/vibe/work/connectors) : connectez vos outils (email, agenda, Slack, Notion, GitHub, Google Drive, SharePoint…) afin que Work puisse exploiter les données externes autorisées.
- [Bibliothèques](/vibe/work/libraries) : accédez à des collections de documents déjà téléchargés et indexés.
- [Skills](/vibe/work/skills) : appliquez des méthodes répétables, listes de contrôle ou modèles à une tâche.
- [Fichiers et Canvas](/vibe/work/files-and-canvas) : téléchargez des documents, tableaux, présentations, PDF ou images pour que Work puisse les lire, résumer, extraire ou transformer en livrable exploitable.
- [Recherche web et Ouverture d’URL](/vibe/work/web-search-open-url) : exploitez des informations publiques ou demandez à Work de consulter une page web précise.

:::note
Work peut vous demander de connecter ou d’autoriser un outil manquant au cours d’une tâche. Les réglages de votre organisation peuvent également limiter les outils disponibles.
:::

<SectionTab as="h2" sectionId="follow-progress">Suivre les étapes et la progression</SectionTab>

Pour les tâches longues, Work affiche en temps réel un **panneau des étapes** à droite, et peut vous poser des **questions de précision** si votre demande n’est pas claire. Il affiche aussi la progression, l’utilisation des outils et des résultats intermédiaires au fil du traitement.

Utilisez ces points de passage pour garder la main :

- Suivez les étapes à mesure qu’elles apparaissent pour voir ce que fait Work.
- Validez ou refusez les actions sensibles lorsqu’on vous le demande.
- Arrêtez la tâche avec le bouton **stop** (carré noir) si Work prend une mauvaise direction.
- Redirigez avec un nouveau message si Work choisit la mauvaise source ou méthode.

:::note
Work requiert votre validation avant chaque action sensible comme l’envoi d’un email, la publication d’un message, la création d’événement dans l’agenda, la suppression d’un ticket ou la modification de données dans des outils externes.

Pour plus de détails, consultez [Sécurité et validations](/vibe/work/safety-and-approvals).
:::

<SectionTab as="h2" sectionId="review-result">Vérifier le résultat avant de l’utiliser</SectionTab>

Considérez les productions de Work comme des brouillons tant que vous ne les avez pas vérifiées. Avant d’utiliser ou de partager un résultat :

- Lisez la synthèse finale.
- Relisez le contenu généré dans [Canvas](/vibe/work/files-and-canvas) ou dans les fichiers.
- Vérifiez les faits, tableaux, dates, noms, responsables et sources citées.
- Contrôlez les informations extraites par rapport au fichier ou à la source originelle si l’exactitude est cruciale.
- Demandez à Work de corriger la production si le public, le ton, la structure ou les faits sont inadaptés.
- N’approuvez, modifiez, partagez ou réutilisez le résultat qu’après relecture.


<SectionTab as="h2" sectionId="repeated-work">Faciliter le travail récurrent</SectionTab>

Après vos premières tâches, utilisez les réglages Vibe et le contexte réutilisable pour éviter de répéter toujours les mêmes instructions :

| Cas d’usage | Fonction |
|---|---|
| Préférences stables (ton, format, langue, contraintes récurrentes) | [Instructions personnalisées](/vibe/work/custom-instructions) |
| Tâches apparentées autour de la même équipe, client, initiative ou thématique | [Projets](/vibe/work/projects) |
| Procédures répétables et consignes spécifiques à une tâche | [Skills](/vibe/work/skills) |
| Processus prédéfinis (si votre espace de travail les propose) | [Workflows](/vibe/work/workflows) |
| Programmation d’un prompt à une date précise (ponctuel, quotidien, hebdo, mensuel, annuel) | [Planifier des tâches](/vibe/work/scheduled-tasks) |
