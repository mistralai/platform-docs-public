Les pénalités de présence et de fréquence sont des paramètres qui contrôlent la répétition, permettant de réguler la diversité du texte généré.

<SectionTab as="h2" variant="secondary" sectionId="presence-penalty">Pénalité de présence</SectionTab>

La **pénalité de présence** détermine dans quelle mesure le modèle pénalise la répétition de mots ou d'expressions. Elle encourage le modèle à utiliser une plus grande variété de mots et d'expressions, rendant la sortie plus diverse et créative.

- **Plage** : [-2, 2]
- **Valeur par défaut** : 0

Une pénalité de présence plus élevée encourage le modèle à éviter de répéter des mots ou des expressions déjà apparus dans la sortie, garantissant un texte plus varié et créatif.

La pénalité de présence est spécifiquement un **ajustement ponctuel** appliqué à tous les tokens utilisés au moins une fois. Elle réduit la probabilité de répéter tout token déjà apparu. Cela encourage le modèle à utiliser une gamme diverse de tokens, favorisant la créativité et la variété dans la sortie.

<SectionTab as="h2" variant="secondary" sectionId="frequency-penalty">Pénalité de fréquence</SectionTab>

La **pénalité de fréquence** est un paramètre qui pénalise la répétition de mots en fonction de leur fréquence dans le texte généré. Elle contribue à promouvoir la diversité et à réduire la répétition dans la sortie.

- **Plage** : [-2, 2]
- **Valeur par défaut** : 0

Une pénalité de fréquence plus élevée dissuade le modèle de répéter des mots déjà apparus fréquemment dans la sortie. Cela garantit que le texte généré est plus varié et moins répétitif.

La pénalité de fréquence est spécifiquement une valeur qui augmente avec la fréquence d'apparition d'un token dans le texte généré, **une pénalité cumulative** : plus le token est échantillonné, plus la pénalité est élevée. Elle réduit la probabilité de répéter tout token déjà apparu fréquemment. Cela garantit que le texte généré est plus varié et moins répétitif.

<SectionTab as="h2" variant="secondary" sectionId="differences-between-presence-penalty-and-frequency-penalty">Différences entre les pénalités de présence et de fréquence</SectionTab>

- **Pénalité de présence** : il s'agit d'une contribution additive ponctuelle qui s'applique à tous les tokens échantillonnés au moins une fois. Elle encourage le modèle à inclure une gamme diverse de tokens dans le texte généré.
- **Pénalité de fréquence** : il s'agit d'une contribution proportionnelle à la fréquence d'échantillonnage d'un token particulier. Elle dissuade le modèle de répéter trop fréquemment les mêmes mots ou expressions dans le texte généré.

Les deux paramètres peuvent être ajustés pour façonner la qualité et la diversité du texte généré. Les valeurs optimales pour ces paramètres peuvent différer selon la tâche spécifique et le résultat souhaité.

Les pénalités sont un paramètre sensible qui peut avoir un impact significatif sur les requêtes à contexte long et à sortie longue, aussi bien négativement que positivement. Elles peuvent également aider à éviter les boucles hautement répétitives dans lesquelles le modèle pourrait autrement tomber, ce qui en fait un paramètre précieux.
