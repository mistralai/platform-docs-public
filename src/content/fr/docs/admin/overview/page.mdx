# Vue d’ensemble Admin

Le <AppLink href="https://admin.mistral.ai" app="admin">tableau de bord Admin</AppLink> est la surface de contrôle de votre Organisation. Il rassemble la configuration de l’Organisation, la gestion des accès, les espaces de travail, la facturation, les limites d’utilisation, les contrôles de confidentialité, les journaux d’audit et l’automatisation via l’API Admin.

<SectionTab as="h2" sectionId="open-admin">Ouvrir Admin</SectionTab>

Vous pouvez ouvrir Admin de deux façons :

- Aller directement au <AppLink href="https://admin.mistral.ai" app="admin">tableau de bord Admin</AppLink>.
- Depuis Vibe ou Studio, ouvrez le menu principal et sélectionnez `Admin`.

:::info
Vous n’avez pas besoin de vous reconnecter. Les sessions sont partagées entre Vibe, Studio et le tableau de bord Admin.
:::

Utilisez cette page pour comprendre le modèle Admin et l’ordre de configuration recommandé.

<SectionTab as="h2" sectionId="set-up-your-organization-first">Configurer d’abord votre Organisation</SectionTab>

Si vous configurez Admin pour la première fois, commencez par l’Organisation. Les paramètres au niveau de l’Organisation concernent chaque membre et chaque espace de travail, donc configurez-les avant d’inviter une grande équipe.

1. Ouvrir le <AppLink href="https://admin.mistral.ai" app="admin">tableau de bord Admin</AppLink>.
2. [Créer ou configurer votre Organisation](/admin/set-up-organization/create-organization).
3. [Vérifier votre domaine](/admin/set-up-organization/verify-domain) et choisir une méthode de connexion.
4. [Inviter des membres, attribuer des rôles et créer des groupes](/admin/identity-access/user-management).
5. [Créer des espaces de travail](/admin/workspaces/organization-vs-workspaces) pour les équipes, les produits ou les environnements.
6. Configurer l’accès aux espaces de travail, les préférences de fonctionnalités, les clés API, le suivi d’utilisation et les limites.
7. [Consulter la facturation et l’utilisation](/admin/billing-usage/subscriptions), puis définir des limites si nécessaire.
8. [Automatiser l’administration](/admin/admin-api/overview) pour le provisionnement, le reporting d’utilisation ou la gestion des clés via l’API.
9. [Surveiller l’activité](/admin/monitor-comply/audit-logs) avec les journaux d’audit et les contrôles de confidentialité.

<SectionTab as="h2" sectionId="how-admin-fits-together">Comment Admin s’articule</SectionTab>

Admin fonctionne avec Vibe et Studio. Vibe permet d’utiliser Mistral pour le travail et le code, et Studio permet aux développeurs de construire avec les API Mistral. Admin se place au-dessus des deux : il contrôle l’Organisation, les membres, les espaces de travail, la facturation, l’utilisation et les politiques d’accès qui structurent l’usage de Vibe et Studio.

Le modèle Admin repose sur trois niveaux principaux : l’organisation, les membres et les espaces de travail. L’organisation est le compte de niveau supérieur pour votre entreprise. Les membres appartiennent à l’organisation, et les espaces de travail permettent de séparer l’utilisation des produits, les autorisations et les limites au sein de cette organisation.

<SectionTab as="h3" variant="secondary" sectionId="organization-level-settings">Paramètres au niveau de l’organisation</SectionTab>

Les paramètres au niveau de l’organisation s’appliquent à l’ensemble de l’organisation. Ils incluent la vérification de domaine, les méthodes de connexion comme l’authentification unique SAML, les abonnements, la facturation, les journaux d’audit et les contrôles de confidentialité.

Configurez ces paramètres avant d’inviter une grande équipe, car ils concernent chaque membre et chaque espace de travail. Pour les étapes de configuration, voir [Configurer votre organisation](/admin/set-up-organization/create-organization).

<SectionTab as="h3" variant="secondary" sectionId="members-roles-and-groups">Membres, rôles et groupes</SectionTab>

Les membres sont les personnes de votre organisation. Les rôles définissent ce qu’un membre peut voir ou modifier. Les groupes vous permettent de gérer l’accès pour plusieurs membres en une seule fois.

Utilisez le _contrôle d’accès basé sur les rôles_ (_RBAC_) pour accorder uniquement les autorisations nécessaires à chaque personne. Les groupes vous aident à gérer l’accès de plusieurs membres à la fois. La prise en charge de _System for Cross-domain Identity Management_ (_SCIM_) arrive bientôt pour automatiser l’onboarding et l’offboarding.

<SectionTab as="h3" variant="secondary" sectionId="workspaces">Espaces de travail</SectionTab>

Les espaces de travail permettent de séparer les équipes, les produits, les environnements ou les unités commerciales au sein d’une même organisation. Chaque espace de travail peut avoir ses propres membres, son utilisation et ses limites.

Utilisez les espaces de travail lorsque les équipes ont besoin de budgets distincts, de suivi d’utilisation, de limites d’accès ou de zones produit dédiées. Pour le modèle conceptuel, voir [Organisation vs. espaces de travail](/admin/workspaces/organization-vs-workspaces).

<SectionTab as="h2" sectionId="where-settings-apply">Où s’appliquent les paramètres</SectionTab>

Avant de modifier un paramètre, identifiez s’il concerne l’ensemble de l’organisation, un espace de travail spécifique ou un workflow d’automatisation.

| Niveau | S’applique à | Tâches courantes |
| --- | --- | --- |
| Organisation | Tous les membres et tous les espaces de travail de l’organisation | Vérification de domaine, SAML SSO, rôles au niveau de l’organisation, facturation, journaux d’audit et contrôles de confidentialité |
| Espace de travail | Un seul espace de travail | Membres de l’espace de travail, clés API, suivi d’utilisation, limites d’utilisation et accès spécifiques au produit |
| API Admin | Administration programmatique pour une organisation | Gérer les utilisateurs, les groupes, les rôles, les espaces de travail, l’approvisionnement et les métriques d’utilisation |

<SectionTab as="h2" sectionId="which-surface-to-use">Quelle interface utiliser</SectionTab>

Utilisez l’interface adaptée pour chaque tâche administrative.

| Interface | Utilisation |
| --- | --- |
| <AppLink href="https://admin.mistral.ai" app="admin">Tableau de bord Admin</AppLink> | Configuration de l’organisation, gestion des membres, des rôles, des espaces de travail, de la facturation, de l’utilisation, des journaux d’audit et des contrôles de confidentialité |
| <AppLink href="https://console.mistral.ai" app="studio">Studio</AppLink> | Conception, tests des modèles, gestion des ressources pour développeurs et travail avec les ressources API |
| [API Admin](/admin/admin-api/overview) | Automatisation des workflows pour les utilisateurs, les groupes, les rôles, les espaces de travail, l’approvisionnement et l’utilisation |
