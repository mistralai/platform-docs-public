<SectionTab as="h1" sectionId="manage-and-deploy-vespa-applications">Gérer et déployer des applications Vespa</SectionTab>

Créez et exploitez des applications Vespa avec le plugin Search Toolkit. Définissez des schémas via des migrations Python, configurez les profils de ranking et de requête, déployez avec la CLI `mistral-vespa` et gérez les clusters de production.

<SectionTab as="h2" sectionId="getting-started">Premiers pas</SectionTab>

Suivez ces guides pour créer et déployer une application Vespa :

| Guide | Objectif |
|---|---|
| **[Anatomie d'une application](vespa/anatomy)** | Comprendre les packages d'applications, schémas, champs, profils de ranking et de requête, migrations |
| **[Gérer les schémas](vespa/migrations)** | Créer et faire évoluer des schémas avec des migrations Python |
| **[Gérer le ranking](vespa/query-profiles)** | Configurer le ranking à l'exécution sans modifier le schéma |
| **[Développement local](vespa/local-development)** | Boucle complète en local : création, déploiement, alimentation, requêtes, itérations |
| **[Déployer et exploiter](vespa/operations)** | Configuration en production, vérifications d'état, supervision |
| **[Référence CLI](vespa/cli)** | Référence des commandes mistral-vespa |

<SectionTab as="h2" sectionId="using-vespa-as-a-search-backend">Utiliser Vespa comme backend de recherche</SectionTab>

Pour utiliser Vespa comme backend de recherche dans vos pipelines Search Toolkit, voir : **[Index de recherche](search-index)**.

<SectionTab as="h2" sectionId="installation">Installation</SectionTab>

```bash
uv add "mistralai-search-toolkit[vespa]"
```
