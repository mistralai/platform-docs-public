# Clés API

Les clés API authentifient vos requêtes vers l'API Mistral. Chaque clé est limitée à un espace de travail spécifique et donne accès à tous les endpoints API disponibles dans votre plan.

<SectionTab as="h1" sectionId="creating-keys">Créer une clé API</SectionTab>

1. <AppLink href="https://console.mistral.ai/api-keys" app="studio">Clés API</AppLink>.
2. Cliquez sur **Créez une nouvelle clé**.
3. Vous pouvez aussi définir un **nom** (par exemple, "Production pipeline") et une **date d'expiration**.
4. Cliquez sur **Créez une nouvelle clé**.
5. Copiez la clé immédiatement.

:::warning
**La clé complète n'est affichée qu'une seule fois**. Après avoir fermé la boîte de dialogue, vous ne pourrez pas la récupérer. Stockez la clé en toute sécurité dans un gestionnaire de mots de passe ou un coffre-fort de secrets. Ne la partagez pas et ne la committez pas dans le contrôle de version.
:::

<SectionTab as="h1" sectionId="using-keys">Utiliser les clés API</SectionTab>

Transmettez la clé dans l'en-tête `Authorization` :

```bash
curl https://api.mistral.ai/v1/chat/completions \
  -H "Authorization: Bearer $MISTRAL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "mistral-medium-latest", "messages": [{"role": "user", "content": "Hello"}]}'
```

Ou définissez-la comme variable d'environnement pour les SDK :

```bash
export MISTRAL_API_KEY="your-api-key"
```

<SectionTab as="h1" sectionId="workspace-scoping">Portée de l'espace de travail</SectionTab>

Les clés API sont **limitées à l'espace de travail** dans lequel elles ont été créées. Les ressources créées avec une clé (comme les jeux de données, les batch jobs, etc.) sont visibles par tous les membres de cet espace de travail.

Pour utiliser des clés différentes selon les environnements (développement, préproduction, production), créez des [espaces de travail](/admin/security-access/organization) distincts.

<SectionTab as="h1" sectionId="key-management">Gestion des clés</SectionTab>

- **Faites tourner les clés** régulièrement. Créez une nouvelle clé, mettez à jour vos applications, puis supprimez l'ancienne.
- **Révoquez immédiatement les clés compromises** en les supprimant de la page des clés API.
- **Définissez des dates d'expiration** pour que les clés cessent automatiquement de fonctionner après une date donnée.
- **Suivi de l'utilisation** : toute l'utilisation des clés dans un espace de travail est suivie ensemble et facturée à l'organisation.

<SectionTab as="h1" sectionId="plans">Plans</SectionTab>

Les clés API fonctionnent par défaut en **mode Gratuit**, avec des limites d'utilisation et de débit adaptées à l'évaluation et au prototypage. Pour augmenter vos limites avec la facturation à l'usage, passez au **plan Scale** dans <AppLink href="https://admin.mistral.ai/organization/billing" app="admin">Subscriptions › Billing</AppLink>.
