# Sorties structurées

Lorsque vous utilisez des LLM comme agents ou étapes d'un processus, d'une chaîne ou d'un pipeline complexe, il est souvent nécessaire que les sorties respectent un format structuré spécifique. JSON est le format le plus couramment utilisé à cette fin.

:::tip
Avant de continuer, nous vous recommandons de lire la documentation [Complétions de conversation](/studio-api/conversations/chat-completion) pour en savoir plus sur l'API de complétions de conversation et son utilisation.
:::

Nous proposons une méthode fiable pour obtenir une sortie structurée dans le format souhaité.

Notre système inclut un mode intégré pour la sortie JSON, ainsi que la possibilité d'utiliser des sorties structurées personnalisées.

:::warning
Pour le mode JSON, il est essentiel d'indiquer explicitement au modèle dans votre prompt de produire du JSON et de spécifier le format souhaité.

Les sorties structurées personnalisées sont plus fiables et sont recommandées dans la mesure du possible. Cependant, il reste conseillé d'itérer sur vos prompts.  
Utilisez le mode JSON lorsque vous avez besoin de plus de flexibilité dans la sortie tout en conservant une structure JSON, et personnalisez-le si vous souhaitez imposer un format plus clair pour améliorer la fiabilité.
:::
