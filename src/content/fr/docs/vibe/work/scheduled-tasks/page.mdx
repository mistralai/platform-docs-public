# Planifier des tâches

:::note
**Aperçu.** Les tâches planifiées sont disponibles en version bêta. L’interface, la nomenclature et les options peuvent évoluer.
:::

Les tâches planifiées vous permettent d’**exécuter un prompt automatiquement** à une date future ou selon une planification récurrente. Work traite le prompt à l’heure prévue, l’exécute comme toute autre tâche et vous notifie lorsque le résultat est prêt.

Utilisez-les pour toute action répétitive : un suivi concurrentiel hebdomadaire, un tri quotidien de votre boîte de réception, un récapitulatif du lundi matin de vos e-mails non lus et messages Slack, ou un rappel ponctuel pour rédiger un suivi le mardi suivant.

:::info
Les tâches planifiées ne fonctionnent **qu’en mode Work** et s’appuient sur l’infrastructure [Workflows](/vibe/work/workflows). Elles bénéficient de toutes les capacités Work : [Compétences](/vibe/work/skills), [Connecteurs](/vibe/work/connectors) intégrés, [Recherche web](/vibe/work/web-search-open-url), [Bibliothèques](/vibe/work/libraries), et [Projets](/vibe/work/projects). Les déclencheurs sont pour l’instant **basés sur le temps** ; d’autres types pourront suivre.
:::

<SectionTab as="h2" sectionId="use-cases">Cas d’usage courants</SectionTab>

Les tâches planifiées sont idéales lorsque **la même question** revient à intervalles réguliers :

- **Résumé hebdomadaire** : chaque lundi matin, résumez vos e-mails non lus et messages Slack de la semaine précédente.
- **Bilan des versions** : chaque vendredi, générez un récapitulatif hebdomadaire des versions à partir de Linear, Notion et Slack.
- **Briefing de réunion** : tous les jours à 9h, préparez un briefing pour votre première réunion de la journée.
- **Veille technologique et concurrentielle** : chaque lundi, résumez les annonces de la semaine précédente d’une liste de concurrents.
- **Rappels ponctuels** : un seul prompt planifié pour le mardi suivant afin de rédiger un e-mail de suivi en vous basant sur vos notes du jour.

<SectionTab as="h2" sectionId="creating-a-schedule">Créer une tâche planifiée</SectionTab>

Vous pouvez créer une tâche planifiée de deux manières.

<SectionTab as="h3" variant="secondary" sectionId="from-tasks-page">Depuis la page Tâches</SectionTab>

1. Dans la barre latérale, cliquez sur `Planifiées`.
2. Cliquez sur `Nouvelle tâche`.
3. Écrivez le prompt que vous souhaitez que Work exécute.
4. Cliquez sur `Planifier` et choisissez une fréquence (voir [ci-dessous](#scheduling-options)).
5. Définissez l’heure de la journée.
6. Cliquez sur `Planifier` pour confirmer, ou `Annuler` pour abandonner.

Une carte de confirmation résume la planification en toutes lettres (par exemple *« Tous les lundis à 9h00 »*) afin que vous puissiez vérifier avant de l’activer.

<SectionTab as="h3" variant="secondary" sectionId="from-a-chat">Depuis un chat</SectionTab>

Vous pouvez également demander à Work de planifier le prompt actuel directement dans la conversation. Par exemple :

- `Planifier ceci pour qu’il s’exécute tous les lundis à 9h.`
- `Exécuter ceci tous les matins de semaine.`
- `Rappelle-moi avec ce prompt mardi prochain à 14h00.`

Work propose une planification, vous confirmez, puis la tâche est ajoutée à votre page Tâches.

La tâche planifiée utilise **la même surface Work** qu’un prompt manuel : Compétences, Connecteurs, recherche Web, Bibliothèques et Projets sont tous disponibles nativement. Vous pouvez également voir des **planifications suggérées** basées sur vos conversations récentes et outils connectés.

<SectionTab as="h2" sectionId="scheduling-options">Options de planification</SectionTab>

Cinq fréquences sont disponibles :

| Fréquence | Votre choix |
|---|---|
| `Une fois` | Une date et heure uniques dans le calendrier. |
| `Quotidien` | Une heure de la journée. S’exécute chaque jour à cette heure. |
| `Hebdomadaire` | Un ou plusieurs jours de la semaine, plus une heure. |
| `Mensuel` | Un jour du mois, plus une heure. |
| `Annuel` | Un jour et un mois (une date par an), plus une heure. |

<SectionTab as="h2" sectionId="tools-and-approvals">Outils, Connecteurs et autorisations</SectionTab>

Une tâche planifiée s’exécute sans surveillance : **pensez aux autorisations à l’avance**.

Lorsque le prompt déclenche une action sensible via un Connecteur (envoi d’un e-mail, publication d’un message, modification d’un fichier), Work demande normalement confirmation. Pour qu’une tâche planifiée s’achève sans votre présence, autorisez à l’avance les actions concernées avec « Toujours autoriser » pour ce Connecteur. Voir [Sécurité et approbations](/vibe/work/safety-and-approvals#approvals) et [Autorisations par fonction du Connecteur](/vibe/work/safety-and-approvals#connector-functions).

:::tip
Utilisez des prompts **en lecture seule** (résumer, briefer, surveiller) pour les exécutions sans surveillance, et réservez les **actions en écriture** (envoyer, publier, modifier) aux prompts que vous passerez en revue avant de les préautoriser.
:::

<SectionTab as="h2" sectionId="notifications">Notifications et résultats</SectionTab>

Lorsqu’une tâche planifiée est terminée, le résultat apparaît dans la barre latérale avec un **point de notification non llu**, comme pour toute autre conversation. Ouvrez-le pour :

- Consulter la sortie.
- Poser des questions de suivi.
- Accéder au **chat associé** si vous souhaitez peaufiner le prompt pour la prochaine exécution.

<SectionTab as="h2" sectionId="managing-schedules">Gérer vos planifications</SectionTab>

Depuis la page `Planifiées`, vous pouvez modifier une tâche planifiée à tout moment :

- **Modifier** : mettre à jour le prompt, la fréquence ou la date de prochaine exécution.
- **Pause** : arrêter les exécutions futures sans supprimer la planification. Reprenez-la plus tard.
- **Supprimer** : retirer définitivement la planification. Les résultats passés restent dans l’historique de conversation.

<SectionTab as="h2" sectionId="limits">Limites</SectionTab>

Le nombre de tâches planifiées que vous pouvez créer en même temps dépend de votre **plan**. Si vous atteignez la limite, mettez en pause ou supprimez une tâche existante pour libérer une place, ou passez à un plan supérieur. Voir [tarification](https://mistral.ai/pricing) pour les chiffres actuels.
