# Rechercher sur le web

Work peut parcourir Internet en temps réel et lire des pages web spécifiques dans le cadre d’une tâche. **Recherche web** répond à vos questions avec des informations à jour et sourcées. **Ouvrir une URL** permet à Work de lire et d’analyser une page à laquelle vous avez déjà le lien.

:::info
**Date limite d’entraînement** : chaque modèle de langage est entraîné sur des données collectées jusqu’à une date fixe. Après cette date, le modèle ne dispose d’aucune information intégrée sur les nouveaux événements, sorties ou faits. Les fonctions de recherche web et d’ouverture d’URL permettent à Work de dépasser cette limite et de fournir des réponses avec des **sources vérifiables et à jour**.
:::

:::note
Pour toute recherche approfondie, multi-sources, avec rapports structurés cités et export PDF, utilisez [Deep Research](/vibe/chat-legacy/deep-research) dans Chat. Deep Research n’est pas disponible dans Work.
:::

<SectionTab as="h2" sectionId="web-search">Recherche web</SectionTab>

La recherche web permet à Work d’aller chercher des informations récentes sur l’ensemble du web pour répondre à votre demande.

<SectionTab as="h3" variant="secondary" sectionId="web-search-activation">Activation</SectionTab>

1. Cliquez sur l’icône `+` ou tapez `/` dans la fenêtre de chat.
2. Sélectionnez `Outils` puis activez `Recherche web`.

Si vous posez une question qui nécessite des informations à jour, Work **recherche sur le web** et intègre les résultats dans sa réponse. Posez votre question naturellement.

<SectionTab as="h3" variant="secondary" sectionId="reading-results">Lecture des résultats</SectionTab>

Lorsque Work utilise la recherche web pour répondre à votre question, voici ce que vous remarquerez :

- Une **icône globe** à côté de la réponse, confirmant que des données web ont été utilisées.
- Des **liens intégrés** renvoyant vers les sources d’origine.
- Un bouton `Sources` en bas de la réponse, qui ouvre un panneau récapitulant toutes les références.

Chaque **réponse issue du web fournit ses références** pour vous laisser vérifier vous-même la source d’information.

<SectionTab as="h3" variant="secondary" sectionId="news-search">Recherche d’actualités</SectionTab>

Pour les requêtes liées à l’actualité, Work s’appuie sur des partenaires professionnels comme [Agence France-Presse](https://www.afp.com/en) (AFP) et [Associated Press](https://apnews.com/) (AP), afin de garantir des résultats provenant de **sources vérifiées, validées éditorialement** que vous pouvez citer en toute confiance.

Lorsque des résultats d’actualité sont affichés, une **icône d’actualité** apparaît à côté de l’icône globe. Les sources renvoient directement au reportage de l’agence, pour vous fournir un contenu fiable lors de vos briefings ou mises à jour internes.

:::tip
Pour une veille sectorielle express, essayez des requêtes comme *« Dernières actualités réglementaires dans [votre secteur] »* ou *« Nouvelles levées de fonds dans [secteur] »*.
:::

<SectionTab as="h2" sectionId="open-url">Ouvrir une URL</SectionTab>

Ouvrir une URL permet **d’intégrer directement du contenu web dans une conversation Work**. Plutôt que de copier-coller le texte d’une page web, collez l’URL et laissez Work récupérer, lire et utiliser le contenu de la page comme contexte.

<SectionTab as="h3" variant="secondary" sectionId="open-url-how-to-use">Comment l’utiliser</SectionTab>

1. Vérifiez que la **Recherche web** est activée :
    + Cliquez sur l’icône `+` ou tapez `/` dans la fenêtre de chat.
    + Sélectionnez `Outils`.
    + Activez `Recherche web`. Sans cela, Work ne pourra pas parcourir le web.
2. Collez une URL dans la zone de saisie, accompagnée de votre question ou instruction.
3. Envoyez le message. Work récupère la page et utilise son contenu pour répondre.

Vous verrez une icône lien et une mention `Page ouverte` dans la réponse, confirmant que Work a bien lu le contenu de la page.

:::tip
Ouvrir une URL fonctionne aussi avec des liens vers des fichiers en ligne (PDF, documents). Work récupère le fichier et répond avec le contexte issu de ce document.
:::

<SectionTab as="h3" variant="secondary" sectionId="multiple-urls">Travailler avec plusieurs URL</SectionTab>

Vous pouvez coller **plusieurs URL dans une même conversation**. Work conserve le contenu de chaque page en contexte, ce qui vous permet de poser des questions de suivi ou de faire des comparaisons entre elles.

Par exemple, collez deux pages produits de concurrents et demandez : *« Compare les fonctionnalités décrites sur ces deux pages »*. Work utilise les deux contenus en contexte pour générer une comparaison détaillée.

<SectionTab as="h3" variant="secondary" sectionId="open-url-use-cases">Cas d’usage courants</SectionTab>

- **Analyse de la concurrence** : collez la page produit d’un concurrent et demandez à Work d’en extraire les fonctionnalités, tarifs ou positionnement.
- **Relecture de documentation** : partagez une page de documentation et demandez un résumé, ou vérifiez-la par rapport à vos propres spécifications.
- **Résumé d’article** : collez un long article pour en obtenir les points clés.
- **Préparation de réunion** : transmettez un ordre du jour ou un brief en ligne et demandez à Work d’identifier les points importants.

<SectionTab as="h2" sectionId="choosing-the-right-tool">Choisir le bon outil</SectionTab>

Choisissez l’option adaptée à votre situation :

| Vous voulez… | Utiliser |
|----------------|-----|
| Obtenir une réponse rapide et à jour, sourcée | **Recherche web** |
| Analyser une page précise dont vous avez le lien | **Ouvrir une URL** |
| Obtenir un rapport structuré avec sources multiples | **[Deep Research](/vibe/chat-legacy/deep-research)** (uniquement dans Chat legacy) |
| Rechercher dans des documents internes fiables | **[Bibliothèques](/vibe/work/libraries)** |
| Récupérer des données en direct depuis un outil connecté (Gmail, Drive, Notion…) | **[Connecteurs](/vibe/work/connectors)** |

<SectionTab as="h2" sectionId="limitations">Limitations</SectionTab>

- Ouvrir une URL récupère uniquement la page indiquée par votre URL. Il n’explore pas l’ensemble du site et ne suit pas les liens.
- Les pages nécessitant une authentification ou un abonnement ne peuvent pas être consultées. Téléchargez le contenu puis [importez-le en tant que fichier](/vibe/work/files-and-canvas) à la place.
- Certains sites web très interactifs peuvent ne pas charger totalement, ce qui peut entraîner une récupération incomplète du contenu.
- La recherche web peut faire remonter des informations obsolètes ou incomplètes selon ce que la page source expose.
