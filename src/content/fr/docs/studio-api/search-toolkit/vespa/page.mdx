<SectionTab as="h1" sectionId="manage-and-deploy-vespa-applications">Gérer et déployer des applications Vespa</SectionTab>

Construisez et exploitez des applications Vespa avec le plugin Search Toolkit. Définissez les schémas via des migrations Python, configurez le classement et les profils de requête, déployez avec le CLI `mistral-vespa` et gérez les clusters de production.

<SectionTab as="h2" sectionId="getting-started">Pour commencer</SectionTab>

Suivez ces guides pour construire et déployer une application Vespa :

| Guide | Objectif |
|---|---|
| **[Anatomie d'une application](vespa/anatomy)** | Comprendre les packages d'application, les schémas, les champs, les profils de classement, les profils de requête et les migrations |
| **[Gérer le schéma](vespa/migrations)** | Créer et faire évoluer des schémas à l'aide de migrations Python |
| **[Gérer le classement](vespa/query-profiles)** | Configurer le classement au moment de la requête, sans modifier le schéma |
| **[Développement local](vespa/local-development)** | Cycle complet de développement local : créer, déployer, alimenter, interroger, itérer |
| **[Déployer et exploiter](vespa/operations)** | Configuration de production, vérifications de statut, surveillances |
| **[Référence CLI](vespa/cli)** | Référence des commandes `mistral-vespa` |

<SectionTab as="h2" sectionId="using-vespa-as-a-search-backend">Utiliser Vespa comme backend de recherche</SectionTab>

Pour utiliser Vespa comme backend de recherche dans vos pipelines Search Toolkit, consultez [mode "Recherche"](/studio-api/search-toolkit/search-index).

<SectionTab as="h2" sectionId="installation">Installation</SectionTab>

```bash
uv add "mistralai-search-toolkit[vespa]"
```
