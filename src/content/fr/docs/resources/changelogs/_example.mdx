6 novembre 2024
- Nous avons publié l'API de modération et l'API batch.
- Nous avons introduit trois nouveaux paramètres :
    - `presence_penalty` : pénalise la répétition de mots ou de phrases
    - `frequency_penalty` : pénalise la répétition de mots en fonction de leur fréquence dans le texte généré
    - `n` : nombre de complétions à retourner pour chaque requête, les tokens d'entrée ne sont facturés qu'une seule fois.

13 septembre 2024
- Dans Le Chat, nous avons ajouté une protection contre une méthode de prompt obfusqué pouvant conduire à une exfiltration de données, signalée par les chercheurs [Xiaohan Fu](https://xhfu.me/) et Earlence Fernandes. L'attaque nécessitait que les utilisateurs copient et collent volontairement des prompts malveillants et fournissent des données personnelles au modèle. Aucun utilisateur n'a été impacté et aucune donnée n'a été exfiltrée.

11 janvier 2024
- Nous avons renforcé la rigueur de l'API. Auparavant, l'API ignorait silencieusement les paramètres non supportés dans les requêtes, mais elle applique désormais strictement la validité de tous les paramètres. Si vous avez des paramètres non supportés dans votre requête, vous verrez le message d'erreur "Extra inputs are not permitted".
- Une version précédente de la [documentation sur la sécurité](/studio-api/safety-moderation) faisait incorrectement référence au paramètre API comme `safe_mode` au lieu de `safe_prompt`. Nous avons corrigé cela dans la documentation.

16 janvier 2024
- Nous avons ajouté les informations d'utilisation de tokens dans les requêtes en streaming. Vous pouvez les trouver dans le dernier chunk retourné.
