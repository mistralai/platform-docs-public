# Présentation de la plateforme

Mistral AI propose trois produits. Choisissez celui qui correspond à votre usage.

- **Vibe** : l'agent tout-en-un pour la productivité et le développement. Utilisez-le sur le web, sur mobile ou directement dans votre terminal ou éditeur.
- **Studio** : la console développeur et l'API Mistral. Générez des clés, prototypez dans le Playground, créez des workflows robustes et exploitez des modèles texte, audio et OCR via un seul SDK.
- **Admin** : la console de pilotage pour la configuration de l'organisation, la facturation, le SSO, les espaces de travail et la gestion des accès.

:::tip
Pour comparer les fonctionnalités et plans de chaque produit, consultez la [page des tarifs](https://mistral.ai/pricing).
:::

<InterfaceSection id="vibe" title="Vibe">

Vibe est l'agent unifié de Mistral. Il fonctionne selon trois modes, chacun adapté à un type de tâche différent :

- **Travail** : mode productivité dans les applications web et mobile de Vibe. Décrivez le résultat attendu, le mode Travail sélectionne les outils, exécute les étapes et vous présente une liste de tâches en temps réel. Utilisez-le pour la recherche, l'analyse documentaire, l'exploration de données et les tâches programmées.
- **Code** : mode développeur disponible en CLI, comme extension VS Code ou via des sessions Vibe Code Web à distance. Vibe Code lit vos fichiers, édite le code, exécute des commandes et ouvre des pull requests sous votre supervision.
- **Chat** : mode conversationnel par tours, pour les échanges rapides et les fonctionnalités héritées de Le Chat (Agents, mode "Réflexion", Recherche, interpréteur de code, Souvenirs).

<SectionTab as="h2" sectionId="common-questions">Questions fréquentes</SectionTab>

<Faq type="multiple">
  <FaqItem question="Quelle est la différence entre Vibe et Studio ?">

Vibe est le produit destiné aux utilisateurs finaux. Vous l'utilisez pour accomplir vos tâches, que ce soit dans l'interface de conversation, dans votre éditeur ou votre terminal. Studio est la console développeur. Vous l'utilisez pour générer des clés API, tester des prompts, effectuer des évaluations et mettre en production des applications reposant sur l'API Mistral. La plupart des organisations combinent les deux : les métiers utilisent Vibe, les développeurs passent par Studio.

  </FaqItem>
  <FaqItem question="Quelle est la différence entre Vibe Work et Vibe Code ?">

Vibe Work fonctionne dans les applications web et mobile de Vibe avec une interface de conversation. Utilisez-le pour vos activités de productivité : recherche, analyse documentaire, tâches programmées, Connecteurs, Skills, Projets.

Vibe Code fonctionne en CLI, via une extension VS Code ou en session web distante. Adoptez-le dès que vos tâches dépendent d'une base de code : lecture de fichiers, édition du code, exécution de commandes, ouverture de pull requests.

Les deux sont des modes du même agent. Consultez [Choisir Chat, Travail ou Code](/vibe/choose-chat-work-code) pour une comparaison détaillée.

  </FaqItem>
  <FaqItem question="J'utilisais Le Chat. Qu'est-ce qui change pour moi ?">

Le Chat devient Vibe. Votre compte, votre plan, votre historique et vos paramètres sont conservés. L'onglet Chat dans Vibe maintient l'expérience au tour par tour à laquelle vous êtes habitué, ainsi que les fonctionnalités héritées (Agents, mode "Réflexion", Recherche, interpréteur de code, Souvenirs). Les nouveaux usages passent par le mode **Travail**, qui ajoute des outils, des Connecteurs, des Skills et une liste de tâches dynamique. Consultez la [présentation de Vibe](/vibe/overview) pour tous les détails sur la migration.

  </FaqItem>
  <FaqItem question="Qui doit avoir le rôle Facturation ?">

Uniquement la personne chargée des opérations financières, généralement un membre de l'équipe finance ou un office manager. Le rôle Facturation donne l'accès aux abonnements, factures et moyens de paiement sans exposer la configuration ou la gestion des utilisateurs de l'organisation. La plupart des membres de l'équipe doivent être Admin ou Membre.

  </FaqItem>
  <FaqItem question="Les agents Chat et les Skills de Vibe Work sont-ils identiques ?">

[Les Agents de Chat](/vibe/chat-legacy/agents) restent disponibles dans Vibe Chat comme fonctionnalité héritée de Le Chat. Dans Vibe Work, le concept équivalent s'appelle [Skills](/vibe/work/skills) : des méthodes réutilisables configurées une seule fois et utilisables dans toutes vos tâches. Pour les nouveaux usages, privilégiez Skills. Les Agents restent disponibles pour ceux qui avaient construit des workflows spécifiques avant le changement de marque.

  </FaqItem>
</Faq>
