### Préparer le jeu de données

Dans cet exemple, nous utilisons le [jeu de données ultrachat_200k](https://huggingface.co/datasets/HuggingFaceH4/ultrachat_200k). Nous chargeons une partie des données dans des Pandas Dataframes, divisons les données en ensembles d'entraînement et de validation, puis sauvegardons les données au format `jsonl` requis pour l'affinage.

```py
import pandas as pd
df = pd.read_parquet('https://huggingface.co/datasets/HuggingFaceH4/ultrachat_200k/resolve/main/data/test_gen-00000-of-00001-3d4cd8309148a71f.parquet')

df_train=df.sample(frac=0.995,random_state=200)
df_eval=df.drop(df_train.index)

df_train.to_json("ultrachat_chunk_train.jsonl", orient="records", lines=True)
df_eval.to_json("ultrachat_chunk_eval.jsonl", orient="records", lines=True)
```

### Reformater le jeu de données

Si vous téléchargez ce fichier `ultrachat_chunk_train.jsonl` sur l'API Mistral, vous pourriez rencontrer un message d'erreur « Invalid file format » en raison de problèmes de formatage des données. Pour reformater les données au bon format, vous pouvez télécharger le script reformat_data.py et l'utiliser pour valider et reformater les données d'entraînement et d'évaluation :

```bash
# télécharger le script de validation et de reformatage
wget https://raw.githubusercontent.com/mistralai/mistral-finetune/main/utils/reformat_data.py

# valider et reformater les données d'entraînement
python reformat_data.py ultrachat_chunk_train.jsonl

# valider et reformater les données d'évaluation
python reformat_data.py ultrachat_chunk_eval.jsonl
```

:::important[ ]
Ce script `reformat_data.py` est conçu pour les données UltraChat et peut ne pas être universel pour d'autres jeux de données. Modifiez ce script et reformatez vos données en conséquence.
:::

### Créer un job d'affinage

Ensuite, nous pouvons créer un job d'affinage :

<Tabs>
  <TabItem value="python" label="python" default>

```python
# créer une tâche d'affinage
created_jobs = client.fine_tuning.jobs.create(
    model="open-mistral-7b",
    training_files=[{"file_id": ultrachat_chunk_train.id, "weight": 1}],
    validation_files=[ultrachat_chunk_eval.id],
    hyperparameters={
        "training_steps": 10,
        "learning_rate":0.0001
    },
    auto_start=False
)

# démarrer une tâche d'affinage
client.fine_tuning.jobs.start(job_id = created_jobs.id)
```

### Analyser et évaluer le modèle affiné

Lorsque nous récupérons un modèle, nous obtenons les métriques suivantes tous les 10 % de la progression avec un minimum de 10 étapes entre chaque mesure :

- Perte d'entraînement : l'erreur du modèle sur les données d'entraînement, indiquant dans quelle mesure le modèle apprend à partir de l'ensemble d'entraînement.
- Perte de validation : l'erreur du modèle sur les données de validation, fournissant un aperçu de la capacité du modèle à généraliser sur des données non vues.
- Précision des tokens de validation : le pourcentage de tokens dans l'ensemble de validation qui sont correctement prédits par le modèle.

La perte de validation et la précision des tokens de validation servent tous deux d'indicateurs essentiels de la performance globale du modèle, aidant à évaluer sa capacité à généraliser et à effectuer des prédictions précises sur de nouvelles données.

### Intégration avec Weights and Biases

Nous proposons également une intégration avec Weights & Biases (W&B) pour surveiller et suivre diverses métriques et statistiques associées à nos travaux d'affinage. Pour activer l'intégration avec W&B, vous devez créer un compte W&B et ajouter vos informations W&B dans la section « integrations » de la requête de création de travail :

```python
client.fine_tuning.jobs.create(
    model="open-mistral-7b",
    training_files=[{"file_id": ultrachat_chunk_train.id, "weight": 1}],
    validation_files=[ultrachat_chunk_eval.id],
    hyperparameters={"training_steps": 10, "learning_rate": 0.0001},
    integrations=[
        {
            "project": "<value>",
            "api_key": "<value>",
        }
    ]
)
```
