# Changer d’organisation ou d’espace de travail

Votre compte Mistral est rattaché à une ou plusieurs **organisations**, et chaque organisation contient un ou plusieurs **espaces de travail**. Un espace de travail est un espace cloisonné : utilisez-le pour séparer les ressources entre départements, projets ou environnements. Ce n’est pas un conteneur de gestion d’équipe.

:::info
Les espaces de travail sont partagés entre Vibe, Studio et le panneau Admin : la même entité espace de travail est visible dans les trois interfaces. Dans Vibe Work, changer d’espace de travail modifie le périmètre sur lequel vous travaillez. Consultez la page [Organisations et espaces de travail](/admin/security-access/organization) pour la vue admin.
:::

<SectionTab as="h2" sectionId="switcher">Changer via la barre latérale</SectionTab>

Cliquez sur le **nom de votre organisation** dans le coin inférieur gauche de l’application Vibe, à côté de vos initiales. Le menu contextuel affiche :

- Votre **espace de travail** actuel (nommé **Espace de travail par défaut** si l’organisation n’en contient qu’un).
- Les autres espaces de travail de la même organisation.
- Les autres organisations auxquelles vous appartenez.

Sélectionnez un espace de travail pour basculer directement. Sélectionnez une organisation pour arriver sur son espace de travail par défaut.

:::note
Si votre compte n’a accès qu’à un seul espace de travail, le sélecteur est masqué. Rien à faire.
:::

:::warning
Changer d’espace de travail bascule **tous les onglets ouverts** (une session par utilisateur). Pour travailler dans deux espaces de travail en même temps, ouvrez des profils de navigateur séparés.
:::

<SectionTab as="h2" sectionId="scoped">Tout est limité par espace de travail</SectionTab>

Quand vous changez d’espace de travail, votre vue dans Vibe Work est réinitialisée. Les ressources suivantes restent là où vous les avez créées et **ne sont pas partagées entre espaces de travail** :

- L’historique de chat
- [Projets](/vibe/work/projects)
- [Compétences](/vibe/work/skills) (personnelles ou publiées sur l’espace de travail)
- [Tâches planifiées](/vibe/work/scheduled-tasks)
- [Instructions personnalisées](/vibe/work/custom-instructions)
- Authentifications de connecteurs

Si vous avez besoin d’une ressource dans un autre espace de travail, vous devez la recréer dans l’espace concerné.

:::info
[Les bibliothèques](/vibe/work/libraries) sont également limitées par espace de travail par défaut, mais l’admin de votre organisation peut activer le partage au niveau de l’organisation afin qu’une bibliothèque soit accessible dans tous les espaces de travail de l’organisation.
:::

<SectionTab as="h2" sectionId="admin-config">Paramètres admin au niveau de l’espace de travail</SectionTab>

Certains comportements de Vibe Work sont configurés par espace de travail par les admins de l’organisation :

- **Outils disponibles** : quels outils et connecteurs Vibe Work sont activés.
- **Accès et rôles utilisateurs** : qui peut rejoindre l’espace de travail et à quel niveau de rôle.
- **Suivi de l’utilisation** : métriques d’usage par espace de travail.

Si une fonctionnalité manque ou fonctionne différemment selon l’espace de travail, c’est généralement pour cette raison. Contactez l’admin de votre organisation pour ajuster les paramètres, ou consultez la page [Organisations et espaces de travail](/admin/security-access/organization) pour les détails de configuration.

:::tip
Un partage plus granulaire de certaines ressources avec des utilisateurs ou groupes spécifiques, sans changer d’espace de travail, est prévu prochainement.
:::
