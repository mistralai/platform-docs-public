**Top P** est un paramètre qui limite les tokens considérés par un modèle de langage en fonction d'un seuil de probabilité. Il permet de se concentrer sur les tokens les plus probables, améliorant ainsi la qualité de sortie.

<SectionTab as="h2" variant="secondary" sectionId="visualization">Visualisation</SectionTab>

Pour ces exemples, nous définissons d'abord la Temperature, puis appliquons un Top P de 50 %. Notez qu'une Temperature de 0 est déterministe, rendant Top P non pertinent dans ce cas.

Le processus est le suivant :

1. Appliquer la Temperature.
2. Utiliser Top P (0,5) pour ne conserver que les tokens les plus probables.
3. Ajuster les probabilités des tokens restants.

Nous allons visualiser la distribution de probabilité des tokens pour différentes valeurs de température pour la question :

- « Quelle est la meilleure créature mythique ? Répondez par un seul mot. »

Top P garantit que seuls les tokens de haute qualité sont considérés, maintenant ainsi la qualité de sortie en excluant les tokens peu probables. Il est difficile d'équilibrer Temperature et Top P, il est donc recommandé de fixer l'un et d'ajuster l'autre. Néanmoins, vous devriez expérimenter pour trouver les meilleurs réglages pour votre cas d'usage !

<SectionTab as="h2" variant="secondary" sectionId="to-summarize">En résumé</SectionTab>

1. **Rôle de Top P** : Top P limite les tokens considérés en fonction d'un seuil de probabilité, en se concentrant sur les tokens les plus probables pour améliorer la qualité de sortie.
2. **Interaction avec Temperature** : Top P est appliqué après Temperature.
3. **Impact sur les sorties** : Top P évite de considérer les tokens très peu probables, maintenant ainsi la qualité et la cohérence de la sortie.
4. **Équilibre entre Temperature et Top P** : Il est difficile d'équilibrer les deux. Commencez par fixer un paramètre et ajustez l'autre, expérimentez pour trouver les réglages optimaux.

<SectionTab as="h2" variant="secondary" sectionId="example">Exemple</SectionTab>

Voici un exemple d'utilisation du paramètre `Top P` avec notre client Python :

<Tabs groupId="sdk-version">
    <TabItem value="v1" label="V1" default>

```py
import os
from mistralai.client import Mistral

api_key = os.environ["MISTRAL_API_KEY"]
model = "ministral-3b-latest"

client = Mistral(api_key=api_key)

chat_response = client.chat.complete(
    model=model,
    messages=[
        {
            "role": "user",
            "content": "What is the best mythical creature? Answer with a single word.",
        },
    ],
    temperature=1,
    top_p=0.5,
    n=10
)

for i, choice in enumerate(chat_response.choices):
    print(choice.message.content)
```
