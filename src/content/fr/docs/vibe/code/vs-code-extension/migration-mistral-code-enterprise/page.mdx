# Migration depuis Mistral Code Enterprise

Cette page explique ce à quoi vous attendre lors du passage de l’**ancienne extension Mistral Code** à l’extension **Mistral Vibe** pour VS Code.

:::warning
**Mistral Code est en fin de vie.** Nous ne prévoyons pas d’outil de migration complexe : Mistral Code s’appuyait sur plusieurs modèles spécialisés, alors que Vibe n’en requiert qu’un seul. Pour la plupart des équipes, migrer revient à **désinstaller l’ancienne extension, installer Vibe, s’authentifier**. C’est tout.
:::

<SectionTab as="h2" sectionId="key-differences">Principales différences</SectionTab>

- **Un seul modèle, plus de choix multiples.** Vibe utilise un unique agent soutenu par un seul modèle, ce qui supprime la sélection de modèles auparavant nécessaire avec Mistral Code.
- **Session partagée entre le CLI et l’extension.** Votre configuration, vos agents, Skills et session active sont partagés entre le CLI Vibe et l’extension Vibe pour VS Code.
- **Nouveau modèle de configuration.** La configuration de Vibe se trouve dans `~/.vibe/` (et `.vibe/` par projet), et non plus dans les anciens chemins de config de Mistral Code.
- **Nouveau système d’authentification.** Vibe et Mistral Code n’utilisent pas le même processus d’authentification ni la même structure de plan. Les clients existants de Mistral Code Enterprise doivent vérifier auprès de leur équipe de compte si leur plan actuel couvre l’usage de Vibe, ou si un nouveau plan est nécessaire.

Pour une comparaison détaillée fonctionnalité par fonctionnalité, consultez la note de publication interne : [Note de version de l’extension Mistral Vibe](https://www.notion.so/mistralai/Mistral-Vibe-Extension-release-note-3276ba59a7fe81949a46da97ec77c277).

<SectionTab as="h2" sectionId="migration-steps">Étapes de la migration</SectionTab>

1. **Sauvegardez tous les paramètres Mistral Code personnalisés** que vous souhaitez conserver (préférences de modèles, prompts personnalisés, etc.).
2. **Désinstallez ou désactivez l’extension Mistral Code** dans l’onglet Extensions de VS Code.
3. **Installez Mistral Vibe** en suivant [Installer et s’authentifier](/vibe/code/vs-code-extension/install-authenticate).
4. **Authentifiez-vous** avec votre clé API Mistral existante, ou suivez le processus de configuration.
5. **Recréez vos configurations personnalisées** (agents, Skills, serveurs MCP) dans `~/.vibe/config.toml`. Voir [Configuration](/vibe/code/cli/configuration).

<SectionTab as="h2" sectionId="not-carried-over">Ce que la migration ne reprend pas</SectionTab>

- **Préférences par modèle** de Mistral Code (Vibe utilise un seul agent).
- **Paramètres de l’extension Mistral Code**.
- **Historique de chat sauvegardé** issu de Mistral Code.
