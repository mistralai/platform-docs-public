# Gérer les espaces de travail et les clés API

Organisez votre compte Mistral avec des espaces de travail, des clés API et des contrôles de dépenses.

- Les **espaces de travail** isolent les clés API et les métriques d'utilisation par équipe ou environnement
- Les **clés API** sont limitées à un espace de travail pour le suivi et le contrôle d'accès
- Les **limites de dépenses** évitent les coûts imprévus par espace de travail

**Temps nécessaire :** ~10 minutes

<SectionTab as="h2" sectionId="prerequisites">Prérequis</SectionTab>

- Rôle admin dans votre organisation Mistral
- Facturation activée (les clés API sont inactives tant qu'un moyen de paiement n'est pas enregistré)

<SectionTab as="h2" sectionId="step-1">Étape 1 : Créer un espace de travail</SectionTab>

Les espaces de travail isolent les clés API, les métriques d'utilisation et la facturation. Utilisez-les pour séparer le développement de la production, ou pour suivre l'utilisation par équipe.

1. Ouvrez <AppLink href="https://admin.mistral.ai/organization/workspaces" app="admin">Gérer › espaces de travail</AppLink>.
2. Cliquez sur **Créer un espace de travail**.
3. Saisissez un nom : par exemple, `Production` ou `Recherche ML`.
4. Ajoutez éventuellement une icône et une description pour faciliter l'identification.
5. Cliquez sur **Créer**.

Créez au moins deux espaces de travail (développement et production) pour que le trafic de test ne consomme pas les quotas de production.

<SectionTab as="h2" sectionId="step-2">Étape 2 : Ajouter des membres à l'espace de travail</SectionTab>

Contrôlez l'accès à chaque espace de travail de manière indépendante.

1. Ouvrez l'espace de travail que vous venez de créer.
2. Cliquez sur **Ajouter des membres**.
3. Sélectionnez les membres de l'équipe dans votre organisation.
4. Attribuez leur rôle dans l'espace de travail :

5. Cliquez sur **Confirmer**.

<SectionTab as="h2" sectionId="step-3">Étape 3 : Générer des clés API</SectionTab>

Chaque espace de travail possède ses propres clés API. Nous suivons et facturons l'utilisation par espace de travail.

1. Ouvrez <AppLink href="https://console.mistral.ai/api-keys" app="studio">Clés API</AppLink>.
2. Sélectionnez l'espace de travail cible dans le menu déroulant des espaces de travail.
3. Cliquez sur **Créer une nouvelle clé**.
4. Nommez la clé de manière descriptive : par exemple, `prod-backend` ou `staging-chatbot`.
5. Copiez la clé immédiatement. Elle n'est affichée qu'une seule fois.

```bash
# Set the key as an environment variable
export MISTRAL_API_KEY="your-key-here"
```

:::warning
Ne commitez jamais les clés API dans le contrôle de version. Utilisez des variables d'environnement ou un gestionnaire de secrets pour les stocker.
:::

<SectionTab as="h2" sectionId="step-4">Étape 4 : Définir les limites d'utilisation</SectionTab>

Évitez les coûts imprévus en définissant des limites de dépenses mensuelles par espace de travail.

1. Ouvrez <AppLink href="https://admin.mistral.ai/organization/billing" path={["Admin Panel", "Subscriptions", "Billing"]} />.
2. Sous l'espace de travail, cliquez sur **Définir une limite**.
3. Saisissez un budget mensuel en USD : par exemple, `$500`.
4. Lorsque l'espace de travail atteint la limite, les requêtes API retournent `429 Too Many Requests` jusqu'au prochain cycle de facturation.

Suivez l'utilisation en temps réel depuis le tableau de bord de facturation pour détecter les tendances avant d'atteindre les limites.

<SectionTab as="h2" sectionId="verify">Vérifier</SectionTab>

La configuration de votre espace de travail est terminée si :

- L'espace de travail apparaît dans le menu déroulant des espaces de travail dans <AppLink href="https://console.mistral.ai">Studio</AppLink>
- Les membres de l'équipe ajoutés à l'espace de travail peuvent le voir dans leur console
- Les clés API générées pour l'espace de travail retournent des réponses réussies
- L'utilisation apparaît sous le bon espace de travail dans le tableau de bord de facturation

Testez une clé avec un appel API rapide :

```bash
curl https://api.mistral.ai/v1/models \
  -H "Authorization: Bearer $MISTRAL_API_KEY"
```

Une réponse réussie retourne une liste JSON des modèles disponibles.
