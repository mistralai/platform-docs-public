# Agents

:::warning
**Attention.** Il s’agit d’une fonctionnalité héritée, encore disponible dans l’onglet **Chat**. Son fonctionnement reste inchangé, inutile de vous presser. Consultez [Prise en main de Work](/vibe/work/get-started) pour démarrer.

Dans Vibe Work, ce concept existe désormais sous forme de **[Skills](/vibe/work/skills)** : testez-les quand vous serez prêt.

Les deux coexistent pour l’instant. Aucune date de suppression prévue — nous vous tiendrons informé.
:::

Un Agent est un **assistant spécialisé** que vous configurez une seule fois et réutilisez dans plusieurs conversations. Donnez-lui des instructions, des outils et des sources de connaissance : il s’occupe du reste.

:::tip
Si vous vous surprenez à **répéter les mêmes prompts**, joindre les mêmes fichiers ou activer systématiquement les mêmes outils à chaque nouvelle conversation, il est temps d’utiliser un Agent.
:::

<SectionTab as="h2" sectionId="use-cases">Exemples d’utilisation</SectionTab>

Les Agents sont particulièrement adaptés lorsque vous souhaitez appliquer de façon cohérente les mêmes instructions, le même ton ou la même base de connaissances :

- **Correcteur du ton de marque** : collez n’importe quel brouillon, il sera réécrit dans le ton de votre entreprise, avec les mêmes règles de mise en forme à chaque fois.
- **Vérificateur de clauses juridiques** : téléchargez un contrat et demandez à l’Agent de signaler les clauses non conformes à vos règles internes (voir [Bibliothèque](/vibe/work/libraries)).
- **Assistant préparation de réunion** : partagez le site d’un prospect ou un bref mémo pour obtenir des arguments structurés, chiffres clés et questions ouvertes.
- **FAQ technique** : joignez la documentation produit comme [Bibliothèque](/vibe/work/libraries) et partagez l’Agent à votre équipe pour proposer des réponses précises et sourcées.
- **Traduction contextuelle** : un Agent réalise des traductions respectant votre glossaire et la terminologie de marque, rattachée comme une [Bibliothèque](/vibe/work/libraries).

<SectionTab as="h2" sectionId="creating-an-agent">Créer un Agent</SectionTab>

Créer un Agent est simple : accessible à tous les utilisateurs.

1. Sélectionnez l’onglet **Chat** dans la barre latérale.
2. Cliquez sur `Agents` dans la barre à gauche.
3. Cliquez sur `Créer un Agent` pour partir de zéro ou choisissez un **modèle pré-rempli** pour démarrer.
4. Configurez l’Agent (voir [section suivante](#configuration)).
5. Testez-le dans le **panneau d’aperçu** à droite de l’écran.

Les Agents sont enregistrés automatiquement à mesure que vous les configurez. Il n’y a pas de bouton d’enregistrement.

:::tip
Les modèles couvrent les usages courants : synthétiseur, traduction, assistant code… Démarrez avec un modèle puis personnalisez-le plutôt que de repartir de zéro.
:::

<SectionTab as="h2" sectionId="configuration">Configuration</SectionTab>

La configuration d’un Agent se décompose en **paramètres principaux** et options avancées. Vous pouvez modifier ces réglages à tout moment.

<ExplorerTabs>
<ExplorerTab value="main-settings" label="Paramètres principaux" default>

Chaque Agent débute avec quatre champs :

- **Nom** : un nom clair et descriptif que votre équipe retrouvera facilement. Exemple : `Qualification support client` ou `Générateur de notes de vente`.
- **Description** (optionnelle) : bref descriptif des fonctions de l’Agent, visible dans le sélecteur d’Agents.
- **Avatar** (optionnel) : choisissez un avatar parmi ceux proposés ou générez-en un personnalisé.
- **Instructions** : consignes détaillées sur le comportement attendu. C’est ici que vous définissez son rôle, son périmètre, et le format des réponses. Tapez `/` dans le champ pour référencer des outils ou sources de connaissances à la volée.

</ExplorerTab>
<ExplorerTab value="guardrails-tone" label="Règles et ton">

Ces réglages sont facultatifs, mais permettent de **préciser le mode de communication de l’Agent**.

- **Règles** : définissez les limites à respecter. Exemple : *« Toujours utiliser les unités du Système international »* ou *« Ne jamais communiquer d’informations tarifaires hors grille homologuée »*.
- **Ton et personnalité** : développez la rubrique `Ton` pour préciser son style. Un Agent client adoptera un ton professionnel, chaleureux ; un Agent d’analyse de code interne, un style plus direct.

</ExplorerTab>
<ExplorerTab value="tools-knowledge" label="Outils et connaissances">

Les Agents peuvent être enrichis avec des **outils préconfigurés** et vos **sources de connaissance**.

**Outils**

- **[Recherche web](/vibe/work/web-search-open-url)** : récupérer des infos à jour sur le web.
- **[Code Interpreter](/vibe/chat-legacy/code-interpreter)** : exécuter du Python pour calculs, analyses, graphiques.
- **[Canvas](/vibe/work/files-and-canvas)** : générer et éditer fichiers, code, feuilles de calcul ou présentations.
- **[Génération d’images](/vibe/work/image-generation)** : créer des images à partir de descriptions textuelles.

**Connaissances**

- **[Bibliothèques](/vibe/work/libraries)** : associez une ou plusieurs Bibliothèques pour donner accès à vos documents importés.
- **[Connecteurs](/vibe/work/connectors)** : connectez des sources externes (Google Drive, GitHub, Notion, etc.) pour enrichir l’Agent.

:::note
Bibliothèques et connecteurs de connaissance (Google Drive, SharePoint) ne peuvent pas être utilisés simultanément sur un même Agent. Les autres connecteurs (Gmail, Google Calendar…) fonctionnent avec l’une ou l’autre option.
:::

</ExplorerTab>
</ExplorerTabs>

<SectionTab as="h2" sectionId="using-agents">Utiliser les Agents dans Chat</SectionTab>

<SectionTab as="h3" variant="secondary" sectionId="starting-conversation">Démarrer une conversation avec un Agent</SectionTab>

1. Sélectionnez l’onglet **Chat** dans la barre latérale.
2. Cliquez sur `+` puis sélectionnez `Agents` ou tapez `/` dans la zone de saisie.
{/* TODO : Il s’agit encore de @ dans l’ancienne interface */}
3. Cherchez ou sélectionnez un Agent. Son nom apparaît en haut de la boîte de discussion.
4. Envoyez votre message. L’Agent répond selon sa configuration.

<SectionTab as="h3" variant="secondary" sectionId="switching-agents">Basculer d’Agent</SectionTab>

Cliquez sur la flèche à côté du nom de l’Agent ou recommencez la procédure ci-dessus et choisissez un Agent différent.

:::info
Un seul **Agent peut être actif à la fois** dans une conversation.
:::

<SectionTab as="h3" variant="secondary" sectionId="returning-default-model">Revenir au modèle par défaut</SectionTab>

Cliquez sur la croix à côté du nom d’Agent pour le retirer. La conversation se poursuit avec le modèle Vibe par défaut. Vous pouvez aussi démarrer un nouveau chat, toujours initialisé sur le modèle par défaut.

<SectionTab as="h2" sectionId="writing-instructions">Rédiger des instructions efficaces</SectionTab>

De bonnes instructions font la différence entre un chatbot générique et un outil fiable pour l’équipe. Quelques principes :

- **Définir clairement le rôle.** Commencez par préciser le périmètre de l’Agent : *« Vous êtes un relecteur de code. Soyez attentif aux bonnes pratiques Python, à la sécurité, à la performance… »*
- **Spécifier le format attendu.** Indiquez si vous souhaitez des bullet points, du JSON, du texte libre, un tableau, etc. Ajoutez un exemple bref si besoin.
- **Définir explicitement les limites.** Précisez ce que l’Agent ne doit pas faire : *« Ne donnez pas de conseil juridique »* ou *« Ne citez que les documents référencés dans la Bibliothèque jointe »*.
- **Référencer les outils dans le texte.** Tapez `/` dans le champ instructions pour mentionner un outil ou une base de connaissances : l’Agent saura comment s’y connecter.
- **Itérer petit à petit.** Modifiez un paramètre à la fois, testez dans l’aperçu, recommencez. L’écriture d’instructions procède souvent par ajustements successifs.

:::info
Pour une introduction complète à la rédaction de prompts, consultez notre [guide d’ingénierie de prompts](/models/best-practices/prompt-engineering).
:::

<SectionTab as="h2" sectionId="sharing">Partage et droits d’accès</SectionTab>

Par défaut, un nouvel Agent est **privé** : vous seul pouvez le voir et l’utiliser.

<SectionTab as="h3" variant="secondary" sectionId="sharing-specific-people">Partage avec des personnes spécifiques</SectionTab>

1. Ouvrez la carte de l’Agent sur la page `Agents`.
2. Cliquez sur l’étiquette d’état de partage (`Privé`, `Organisation` ou `x personnes`).
3. Cherchez les utilisateurs ou groupes, puis attribuez un niveau de droit :
   - **Collaborateur** : peut utiliser, modifier et supprimer l’Agent.
   - **Lecteur** : peut utiliser l’Agent dans ses conversations, mais ne peut ni le modifier ni le supprimer.

Les modifications sont enregistrées automatiquement. Fermez la fenêtre quand vous avez terminé.

<SectionTab as="h3" variant="secondary" sectionId="sharing-organization">Partage à toute l’organisation</SectionTab>

Dans la même fenêtre de partage, activez `Toute l’organisation` et choisissez si tout le monde obtient le rôle Collaborateur ou Lecteur.

:::note
Nous n’envoyons pas de notification lors du partage d’un Agent. Prévenez directement vos collègues pour qu’ils puissent l’utiliser.
:::

<SectionTab as="h2" sectionId="managing-agents">Gérer les Agents</SectionTab>

<SectionTab as="h3" variant="secondary" sectionId="duplicating-agent">Dupliquer un Agent</SectionTab>

Ouvrez le menu trois points (`⠇`) sur la carte de l’Agent et choisissez `Dupliquer mon Agent`. Cela crée une copie indépendante que vous possédez. Les modifications sur l’un n’affectent pas l’autre.

:::tip
La duplication est aussi la solution pour transférer un Agent à un collègue. Partagez-le en Collaborateur, demandez-lui de le dupliquer, puis supprimez l’original si besoin.
:::

<SectionTab as="h3" variant="secondary" sectionId="deleting-agent">Supprimer un Agent</SectionTab>

1. Ouvrez le menu trois points (`⠇`) sur la carte de l’Agent.
2. Cliquez sur `Supprimer` et confirmez.

:::caution
La suppression d’un Agent est définitive et irréversible. S’il était partagé, il devient inaccessible pour tous.
:::

:::note
Les Agents sont inclus dans toutes les formules. Si vous modifiez ou résiliez votre abonnement, vos Agents et leurs paramètres de partage sont conservés. Certains outils ou sources de connaissance peuvent devenir indisponibles selon le niveau d’offre.
:::

<SectionTab as="h2" sectionId="agents-and-api">Agents et l’API</SectionTab>

Les Agents créés dans Chat sont conçus pour l’interface chat et ne peuvent pas être utilisés via l’API. Pour API, créez des Agents dans <AppLink href="https://console.mistral.ai/build/agents" app="studio">Agents</AppLink> ou via la [REST API](https://docs.mistral.ai/api/#tag/beta.agents).

Les Agents Studio peuvent être déployés dans Chat et apparaissent alors dans une section dédiée de votre liste d’Agents.
