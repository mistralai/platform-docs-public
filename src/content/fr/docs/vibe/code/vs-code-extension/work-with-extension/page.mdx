# Utiliser l’extension

Utilisez l’extension VS Code pour profiter de Vibe Code dans **le même espace de travail que vos fichiers, sélections, différences et contrôle de version**, sans changer de contexte et sans terminal supplémentaire.

<SectionTab as="h2" sectionId="open-vibe">Ouvrir Vibe dans VS Code</SectionTab>

Ouvrez l’extension à partir de l’un de ces points d’entrée :

- Cliquez sur l’icône **Mistral Vibe** dans la barre d’activités.
- Exécutez **Mistral Vibe : Ouvrir Mistral Vibe** depuis la palette de commandes.
- Utilisez l’option du titre de l’éditeur lorsqu’elle apparaît pour le fichier actif.

L’extension maintient le panneau Vibe dans une webview de VS Code et exécute l’agent Vibe ACP inclus en arrière-plan.

<SectionTab as="h2" sectionId="editor-context">Utiliser le contexte de l’éditeur</SectionTab>

L’extension suit votre **éditeur actif et votre sélection en cours**, vous évitant ainsi de copier les chemins ou de coller des extraits :

| Contexte | Fonctionnement |
|---|---|
| **Fichier actif** | Vibe reçoit une mention de l’espace de travail pour le fichier actif. |
| **Lignes sélectionnées** | Vibe reçoit la mention du fichier avec les numéros de début et de fin de ligne. |
| **Texte sélectionné** | Vibe inclut le texte sélectionné lorsque la sélection contient assez de contenu non blanc et reste sous la limite de taille. |

:::tip
Sélectionnez le code sur lequel vous souhaitez échanger, puis posez votre question à Vibe en langage naturel. L’extension relie automatiquement le bon contexte.
:::

<SectionTab as="h2" sectionId="prompt">Envoyer un prompt</SectionTab>

Rédigez une demande précise dans le panneau Vibe. Indiquez le résultat attendu et toute contrainte importante.

Exemples :

```text
Expliquez comment l’authentification est structurée dans le fichier sélectionné.
```

```text
Refactorez cette fonction pour rendre la gestion des erreurs explicite. L’API publique ne doit pas changer.
```

```text
Relisez ce diff pour repérer les cas limites avant que j’ouvre une pull request.
```

<SectionTab as="h2" sectionId="review-actions">Actions de relecture</SectionTab>

Quand Vibe Code a besoin d’une autorisation, l’extension affiche **des contrôles d’approbation dans la webview**. Relisez l’action proposée avant de l’approuver. Pour les modifications de fichiers, utilisez les vues diff et contrôle de version de VS Code pour vérifier le résultat avant de valider ou d’ouvrir une pull request.

Pour le modèle d’approbation partagée, voir [Sécurité, demandes d’autorisation et permissions](/vibe/code/safety-approvals-permissions).

<SectionTab as="h2" sectionId="limits">Limites actuelles</SectionTab>

L’extension n’expose pas toutes les commandes CLI ni tous les workflows.

| Domaine | Comportement de l’extension |
|---|---|
| **Commandes slash** | Affiche un sous-ensemble des commandes slash du CLI. Voir [Commandes et slash commands](/vibe/code/vs-code-extension/commands-slash-commands). |
| **Installation du CLI** | Non requise pour un usage standard de l’extension. |
| **Mode programmatique** | Non pris en charge depuis l’extension. Pour des exécutions sans interaction, installez le CLI et utilisez `vibe --prompt`. |
