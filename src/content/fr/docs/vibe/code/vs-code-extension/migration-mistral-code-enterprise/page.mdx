# Migration depuis Mistral Code Enterprise

Cette page explique à quoi vous attendre lors de la transition de l’**extension Mistral Code** vers **Vibe Code**. Les deux extensions peuvent coexister si vous devez continuer à utiliser Mistral Code tout en testant Vibe.

Pour la plupart des équipes, la migration se résume à : **installer Vibe, s’authentifier et éventuellement désinstaller Mistral Code**. Aucune migration automatique des paramètres ou des sessions n’est possible entre les deux : Mistral Code nécessitait plusieurs modèles spécialisés, alors que Vibe n’en utilise qu’un seul.

<SectionTab as="h2" sectionId="key-differences">Différences clés</SectionTab>

- **Un seul modèle, et non plusieurs.** Vibe propose un agent unique basé sur un seul modèle, ce qui élimine la sélection de modèles requise par Mistral Code.
- **Sessions partagées entre la CLI et l’extension.** Votre configuration, vos agents, vos compétences et votre session active sont partagés entre la CLI Vibe et l’extension Vibe pour VS Code.
- **Nouveau modèle de configuration.** La configuration de Vibe est stockée dans `~/.vibe/` (et dans `.vibe/` par projet), et non dans les anciens chemins utilisés par Mistral Code.
- **Plans distincts.** Mistral Code Enterprise et Vibe Code sont facturés via des **plans différents**. Si vous êtes client Mistral Code Enterprise, contactez votre équipe de compte pour discuter de l’ajout de Vibe.

<SectionTab as="h2" sectionId="migration-steps">Étapes de migration</SectionTab>

1. **Sauvegardez les paramètres Mistral Code personnalisés** que vous souhaitez conserver (préférences de modèles, prompts personnalisés, etc.).
2. **Désinstallez ou désactivez l’extension Mistral Code** depuis la vue Extensions de VS Code.
3. **Installez Mistral Vibe** en suivant [Installer et s’authentifier]([Install and authenticate](/vibe/code/vs-code-extension/install-authenticate)).
4. **Authentifiez-vous** avec votre clé API Mistral existante ou suivez le flux de configuration.
5. **Reconstruisez toute configuration personnalisée** (agents, compétences, serveurs MCP) dans `~/.vibe/config.toml`. Consultez [Configuration](/vibe/code/cli/configuration).

<SectionTab as="h2" sectionId="not-carried-over">Ce qui n’est pas transféré</SectionTab>

- **Les préférences par modèle** de Mistral Code (Vibe utilise un agent unique).
- **Les paramètres de l’extension Mistral Code**.
- **L’historique de conversation** sauvegardé dans Mistral Code.
