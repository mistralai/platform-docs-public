En tant que fichier JSONL, chaque ligne représente une requête vers l'Endpoint et le Modèle de l'API.

<SectionTab as="h2" variant="secondary" sectionId="ocr-explained">Explication</SectionTab>

Le corps de la requête suivra le même format que l'endpoint pour lequel vous souhaitez exécuter votre traitement par batch, à l'exception de l'identifiant du modèle qui sera fourni uniquement lors de la création du job pour démarrer l'exécution du batch. Ci-dessous, nous fournissons un exemple de ligne avec OCR :

<Tabs>
    <TabItem value="file" label="Exemple" default>

```py
{
    "custom_id": "2", # Un ID utilisé comme métadonnée qui sera retourné dans le fichier de sortie pour identifier la requête
    "body": { # Le corps de la requête
        "document": { # Le document à traiter
            "image_url": "data:image/jpeg;base64,<base64_image>" # Ici, nous utilisons une image encodée en base64, mais vous pouvez également utiliser une URL vers un fichier, et document_url pour les PDF ou autres types de documents
        }
    }
}
```
