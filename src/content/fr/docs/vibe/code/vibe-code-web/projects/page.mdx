# Projets

Un **projet** est l’espace de travail autour duquel Vibe Code Web organise tout : un ou plusieurs dépôts GitHub, les sessions que vous y exécutez, et l’historique de toutes les exécutions précédentes. Cette page explique comment créer, gérer et supprimer des projets.

<SectionTab as="h2" sectionId="create-a-project">Créer un projet</SectionTab>

Les projets apparaissent dans la barre latérale gauche de Vibe Code Web. Pour en créer un :

1. Cliquez sur le bouton **+** en haut de la liste **Projets**.
2. Sélectionnez les dépôts sur lesquels le projet doit travailler. Tous les dépôts doivent appartenir au **même propriétaire GitHub** (compte utilisateur ou organisation). Vous pouvez en ajouter un ou plusieurs.
3. Cliquez sur **Continuer**.
4. Saisissez un nom de projet.
5. Cliquez sur **Créer le projet**.

Le projet s’ajoute à votre barre latérale et est prêt pour lancer des sessions.

:::note
Pour travailler sur des dépôts d’un autre propriétaire GitHub, créez un projet séparé. Voir [Dépôts GitHub et permissions](/vibe/code/vibe-code-web/github-repositories-permissions) pour la contrainte liée au propriétaire unique.
:::

<SectionTab as="h2" sectionId="start-a-session">Démarrer une session dans un projet</SectionTab>

Ouvrez un projet depuis la barre latérale pour voir ses sessions et en lancer une nouvelle. Pour démarrer une session :

- Saisissez un prompt dans le champ en bas de la vue projet, puis envoyez-le. Une session est créée automatiquement.
- Ou cliquez sur **+ Nouvelle session** et saisissez votre prompt dans la fenêtre qui s’ouvre.

Toutes les sessions démarrées dans le projet apparaissent sous celui-ci dans la barre latérale, formant un historique de chat. La barre latérale affiche toutes les sessions ; la sélection d’un projet filtre sur les sessions de ce projet uniquement. Vous pouvez rouvrir à tout moment une session précédente pour consulter les commandes, modifications de fichiers, branche et pull request.

<SectionTab as="h2" sectionId="manage-sessions">Gérer les sessions</SectionTab>

Depuis la vue projet, vous pouvez :

- **Rouvrir une session précédente** pour examiner son fil d’activité, les modifications de fichiers, la branche et la pull request. Les anciennes sessions sont en lecture seule : une fois la sandbox terminé, la session ne peut plus être reprise. Pour continuer, lancez une nouvelle session.
- **Supprimer une session** pour la retirer de l’historique des sessions de Vibe Code Web. Les sessions actives ne peuvent pas être supprimées : attendez d’abord la fin de la session.

:::warning
**La suppression est définitive et irréversible.** Supprimer une session ne retire que l’historique Vibe Code Web. Les branches, commits et pull requests que la session a créés restent dans votre dépôt.
:::

<SectionTab as="h2" sectionId="edit-project-details">Modifier les détails du projet</SectionTab>

Cliquez sur **Voir les détails du projet** pour ouvrir les paramètres du projet. Vous pouvez alors :

- **Renommer** le projet.
- **Détacher** un dépôt associé.
- **Ajouter un autre dépôt** du même propriétaire GitHub.

:::info
Les modifications s’appliquent **uniquement aux nouvelles sessions**. Les sessions déjà en cours conservent leur périmètre d’origine.
:::

<SectionTab as="h2" sectionId="delete-a-project">Supprimer un projet</SectionTab>

Pour supprimer un projet, ouvrez **Voir les détails du projet** et faites défiler jusqu’à la section **Zone de danger**. Tapez le **nom complet du projet** pour confirmer, puis cliquez sur supprimer. Un projet ayant des sessions actives ne peut pas être supprimé : attendez d’abord leur fin.

:::warning
**La suppression est définitive et irréversible.** Le projet Vibe Code Web et tout son historique de sessions sont supprimés. Les branches, commits, pull requests et les dépôts eux-mêmes restent intacts sur GitHub.
:::
