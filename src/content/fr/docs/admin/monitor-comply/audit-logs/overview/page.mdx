# Journaux d’audit

Les journaux d’audit fournissent un **enregistrement chronologique des actions** effectuées au sein de votre Organisation dans Studio et Vibe. Ils permettent aux administrateurs de suivre l’activité des utilisateurs, l’utilisation des clés API et les événements de sécurité.

Disponible sur les plans **Enterprise**. Activé par défaut pour tous les espaces de travail.

<SectionTab as="h1" sectionId="accessing">
  Accéder aux journaux d’audit
</SectionTab>

Ouvrez <AppLink href="https://admin.mistral.ai/audit-logs" path={["Console d’administration", "Administration", "Journaux d’audit"]} />.

:::info
Aucune configuration n’est nécessaire. Les journaux d’audit sont activés automatiquement sur les plans Enterprise.
:::

<SectionTab as="h1" sectionId="what-is-logged">
  Ce qui est enregistré
</SectionTab>

Les journaux d’audit capturent les actions effectuées par les utilisateurs et les clés API. Chaque entrée de journal inclut :

- **Horodatage** (`CreatedAt`) : date et heure de l’événement.
- **Acteur** : celui qui a effectué l’action (utilisateur humain, clé API ou système).
- **Événement** : l’action spécifique (par exemple `batch_job.create`, `le_chat.conversation.deleted`).
- **Cible** : la ressource concernée (par exemple, job batch, conversation, clé API).
- **Métadonnées** : Contexte supplémentaire sur l’acteur, l’événement et la cible.

**Activités enregistrées** :

- Événements d’authentification (connexions, flux SSO)
- Utilisation et gestion des API et des clés (création, suppression)
- Modifications de l’Organisation et des espaces de travail
- Actions de gestion des utilisateurs (invitations, changements de rôle, suppressedions)
- Modifications des paramètres
- Interactions dans Vibe (création, suppression de conversations)

<SectionTab as="h1" sectionId="search-and-filter">
  Rechercher et filtrer
</SectionTab>

Utilisez les commandes de filtrage au-dessus du tableau des journaux pour affiner les résultats :

1. Filtrez par **Acteur** (humain, clé API), par **type d’Événement** ou par **ressource Cible**.
2. Combinez plusieurs filtres pour trouver des activités spécifiques (par exemple, toutes les suppressions de conversations effectuées par un utilisateur précis).
3. Personnalisez les colonnes visibles en cliquant sur **Colonnes** et en activant ou désactivant les champs.

Le tableau se met à jour automatiquement lorsque vous appliquez des filtres.

<SectionTab as="h1" sectionId="use-cases">
  Cas d’usage
</SectionTab>

- **Surveillance de la sécurité** : détectez les accès non autorisés ou les schémas d’activité inhabituels.
- **Enquête sur un incident** : retracez la séquence d’actions à l’origine d’un problème.
- **Conformité** : conservez des enregistrements pour les exigences réglementaires.

<SectionTab as="h1" sectionId="limitations">
  Limitations
</SectionTab>

- L’export des journaux n’est pas pris en charge.
- Les journaux d’audit ne sont disponibles que sur les plans Enterprise.
