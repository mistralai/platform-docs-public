# Authentication and Admin API keys

Admin API requests authenticate with a dedicated **Admin API key**. Learn how those keys work and how to call the API.

<SectionTab as="h1" sectionId="admin-api-keys">Admin API keys</SectionTab>

An Admin API key is created and managed in the **Backoffice** (<AppLink href="https://backoffice.mistral.ai">backoffice.mistral.ai</AppLink>). Each key is tied to a **single organization** and grants access to that organization's admin endpoints. Keys are managed from the Enterprise Account Backoffice for convenience, but their scope is always one organization.

:::warning
A user's standard API key never grants admin access, regardless of the user's role. Admin endpoints require an Admin API key created in the Backoffice. This is different from the regular [API keys](/admin/identity-access/api-keys) used for the inference API.
:::

<SectionTab as="h1" sectionId="create-key">Create an Admin API key</SectionTab>

1. Open the <AppLink href="https://backoffice.mistral.ai">Backoffice</AppLink> and go to **API keys**.
2. Click **Create key**.
3. Choose a name, select the organization the key belongs to, and optionally set an expiration date.
4. Copy the key and store it securely.

:::warning
The key is shown in clear **only once**, at creation. If you lose it, create a new one.
:::

<SectionTab as="h1" sectionId="make-a-request">Make a request</SectionTab>

- **Base URL**: `https://console.mistral.ai/api/admin`
- **Authentication header**: `x-api-key: <ADMIN_API_KEY>`

```bash
curl https://console.mistral.ai/api/admin/users \
  -H "x-api-key: $ADMIN_API_KEY"
```
