# Vibe Code Web pour Slack : guide de démarrage

:::info
**Contenu généré par IA :** Cette application utilise l'IA pour générer des réponses et du code. Les résultats peuvent être inexacts, incomplets ou trompeurs. Ne vous y fiez pas pour des conseils factuels, juridiques, médicaux, financiers ou professionnels. Vérifiez toujours les informations importantes par d'autres moyens. Voir notre [politique de confidentialité](https://mistral.ai/terms#privacy-policy).
:::

Vibe Code Web pour Slack permet à votre Team de lancer des sessions de codage IA à distance directement depuis un thread Slack. L'agent analyse le contexte du thread, travaille dans une sandbox cloud isolée et ouvre une pull request GitHub — sans quitter Slack.

<CtaButton href="https://admin.mistral.ai/organization/connectors">
  Connecter Slack à votre organisation Mistral
</CtaButton>

Utilisez ce guide une fois que l'application Slack Vibe Code Web a été installée et liée à votre compte Vibe et à vos dépôts GitHub.

<SectionTab as="h1" sectionId="configuration">
  Configuration
</SectionTab>

:::warning
Vous devez être Admin de votre organisation Mistral pour configurer l'intégration Slack.
:::

1. Ouvrez les [**paramètres Admin**](https://admin.mistral.ai/organization/connectors)
2. **Vérifiez que vous êtes dans la bonne organisation Mistral.** Une organisation Mistral ne peut être liée qu'à un seul espace de travail Slack.
3. Dans la section **Connexions d'applications**, cliquez sur **Connecter à Slack**.
4. Effectuez la procédure OAuth pour finaliser la connexion. Si vous n'êtes pas propriétaire de votre espace de travail Slack, il se peut que vous deviez demander l'installation de l'application au propriétaire de l'espace de travail.
5. **Mistral Vibe** est maintenant installé dans votre espace de travail Slack. Vous pouvez [connecter Slack MCP](#slack-mcp-usage) ou [démarrer votre première session](#start-first-session).

<SectionTab as="h1" sectionId="slack-mcp-usage">
  Utiliser Slack MCP
</SectionTab>

Une fois l'application Slack installée dans votre espace de travail, suivez ces instructions :

1. Ouvrez [**Vibe Work**](https://chat.mistral.ai/connections)
2. Cliquez sur **Connecteur Slack**, puis sur **Connecter**
3. Finalisez l'authentification OAuth pour connecter le bon espace de travail Slack
4. Cliquez sur [**Nouveau Chat**](https://chat.mistral.ai/chat?integrations=019cfb41-5e8a-7350-b162-b2b80604fe5a)
5. Profitez de vos expériences avec Slack !

<SectionTab as="h1" sectionId="what-you-can-do">
  Ce que vous pouvez faire depuis Slack
</SectionTab>

:::warning
Cette fonctionnalité fait l'objet d'un déploiement progressif. Elle sera bientôt disponible, restez informé ;
:::

Vibe Code Web permet de lancer une session de codage distante sur la base du contexte déjà présent dans un thread Slack. L'agent exploite le contexte du thread, travaille dans une sandbox cloud isolée et crée une session Vibe Code Web liée. L'agent de codage peut ouvrir une pull request pour relecture.

Utilisez Slack lorsque la tâche de code commence :

- à partir d'un thread d'incident ou d'astreinte
- d'une notification de CI ou d'alerte
- d'une escalade utilisateur ou support
- d'une discussion produit ou technique
- d'un résumé de relecture dans Slack
- d'une petite décision déjà validée dans le thread

<SectionTab as="h1" sectionId="start-first-session">
  Démarrer votre première session
</SectionTab>

1. **Trouvez un thread Slack avec suffisamment de contexte pour la tâche de codage.** Un bon contexte inclut :
   - le nom du dépôt ou du projet
   - le bug, le test en échec ou le comportement attendu
   - des liens vers des logs, PR, issues ou captures d'écran
   - des contraintes, comme _"ne pas modifier l'API"_ ou _"ne toucher qu'à la documentation"_
2. **Mentionnez l'application Vibe Code dans le thread.**
   ```text
   @Vibe analyse le problème décrit dans ce thread et ouvre une PR si une correction claire est identifiée.
   ```
3. **Si Vibe a besoin de précisions, répondez directement dans le thread.** Exemples de questions formulées :
   - Quel dépôt dois-je utiliser ?
   - Quelle branche dois-je cibler ?
   - Dois-je ouvrir une PR ou mener une simple investigation ?
   - Est-ce sans risque ou faut-il fournir un résumé d'abord ?
4. **Ouvrez le lien de session Vibe Code Web publié par l'application.** Utilisez la session web pour suivre la progression, consulter la sortie console, orienter l'agent et effectuer la relecture de la branche GitHub ou de la pull request.
5. **Relisez le code final dans GitHub.** GitHub reste la source de vérité pour la relecture du code, les vérifications, les commentaires, les validations et les décisions de merge.

<SectionTab as="h1" sectionId="good-first-prompts">
  Bons premiers prompts
</SectionTab>

Utilisez des prompts ciblés, actionnables et susceptibles de conduire rapidement à une branche ou une pull request.

| Cas d'usage                         | Prompt Slack                                                                                                                                        |
| ------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Analyser un test en échec**        | `@Vibe analyse le test en échec dans ce thread. Si la correction est claire, ouvre une PR avec la plus petite modification sécurisée.`              |
| **Corriger un bug signalé par client** | `@Vibe exploite le rapport ci-dessus pour identifier le bug probable. Ouvre une PR si tu arrives à le reproduire ou à détecter une cause claire.`  |
| **Traiter une alerte**                | `@Vibe analyse cette alerte et vérifie s'il existe une correction de code ou de configuration. Commence par un résumé, puis ouvre une PR seulement si le changement est sans risque.` |
| **Appliquer une modification validée** | `@Vibe mets en place la modification de contenu/config validée ci-dessus et ouvre une PR.`                                                        |
| **Ajouter une couverture de test**    | `@Vibe ajoute un test de régression pour le cas limite décrit dans ce thread. Ne modifie pas l'implémentation, sauf si le test révèle un bug.` |
| **Mettre à jour la doc**             | `@Vibe mets à jour la doc en fonction du comportement décrit ci-dessus. Ouvre une PR avec uniquement la modification doc.`                         |
| **Petit refactoring**                | `@Vibe refactorise le helper discuté ci-dessus. Ne change pas le comportement et exécute les tests concernés avant d'ouvrir la PR.`                 |

<SectionTab as="h1" sectionId="prompt-template">
  Modèle de prompt
</SectionTab>

Pour de meilleurs résultats, mentionnez le dépôt, l'objectif et le périmètre de la relecture.

```text
@Vibe [tâche]

Dépôt/projet : [nom du dépôt]
Objectif : [ce qui doit changer]
Contraintes : [ce qui ne doit pas être modifié]
Validation : [test/vérification à effectuer si précisé]
Sortie : ouvrir une PR / seulement enquêter / résumer d'abord
```

Exemple :

```text
@Vibe corrigez le bug de persistance des paramètres décrit ci-dessus.

Dépôt/projet : web-app
Objectif : l'interrupteur d'alertes doit rester activé après actualisation.
Contraintes : ne pas changer la structure de l'API.
Validation : exécuter les tests de paramètres si existants.
Sortie : ouvrir une PR si la correction est claire ; sinon résumer les conclusions.
```

<SectionTab as="h1" sectionId="what-happens-next">
  Étapes suivantes
</SectionTab>

Après mention de Vibe Code :

1. L'application lit le contexte du thread Slack.
2. Elle associe la demande à un projet Vibe Code Web et un dépôt GitHub.
3. Si le dépôt est ambigu, elle vous demande de choisir.
4. Elle crée une session Vibe Code Web et publie un lien dans Slack.
5. L'agent travaille à distance dans une sandbox gérée.
6. Le résultat est une branche ou une pull request GitHub si une modification de code est apportée.

<SectionTab as="h1" sectionId="slack-vs-web-vs-cli">
  Quand utiliser Slack, Web ou CLI
</SectionTab>

| Commencer depuis     | À utiliser quand                                                                                              |
| ------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **Slack**           | Le contexte existe déjà dans un thread Slack : incidents, tickets support, alertes ou décisions d'équipe.         |
| **Vibe Code Web**   | Vous connaissez le dépôt et vous souhaitez lancer une tâche distante directement.                               |
| **Vibe CLI**        | Vous travaillez localement, ou la tâche dépend des fichiers, services, secrets ou paramétrages locaux.             |

<SectionTab as="h1" sectionId="best-practices">
  Bonnes pratiques
</SectionTab>

- Privilégiez des tâches petites et ciblées.
- Précisez dès que possible le dépôt ou le projet concerné.
- Indiquez si vous souhaitez une pull request ou simplement un résumé.
- Ajoutez le test en échec, le message d'erreur, la stack ou le comportement attendu.
- Évitez de demander de grandes réécritures à partir de Slack.
- Ne collez pas de secrets dans Slack ni dans le prompt de l'agent.
- Relisez la branche ou la pull request dans GitHub avant de valider.

<SectionTab as="h1" sectionId="good-tasks">
  Bonnes tâches Slack
</SectionTab>

Les tâches bien adaptées à Slack :

- exploitent assez de contexte dans le thread
- sont liées à un dépôt ou projet connu
- peuvent tourner dans une sandbox isolée
- peuvent être vérifiées par test, compilation ou inspection du code
- aboutissent à une branche GitHub, une pull request ou un résumé

<SectionTab as="h1" sectionId="less-ideal-tasks">
  Tâches moins adaptées à Slack
</SectionTab>

Évitez d'utiliser Slack pour :

- de grands chantiers d'architecture sans responsable identifié
- des tâches nécessitant des services, secrets, matériels ou réseaux privés locaux
- des prompts vagues comme _"corriger ceci"_ sans logs ni comportement attendu
- du travail GitHub multi-équipe si le lien projet/dépôt n'est pas clair
- des changements qui doivent être conçus manuellement auparavant

<SectionTab as="h1" sectionId="data-privacy">

- Pour toutes les informations sur la gestion des données, la durée de conservation et vos droits, consultez la [politique de confidentialité Mistral](https://mistral.ai/terms#privacy-policy) et le [trust center](https://trust.mistral.ai/).
