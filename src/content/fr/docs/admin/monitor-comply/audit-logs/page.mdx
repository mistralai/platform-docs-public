# Journaux d’audit

Les journaux d’audit fournissent un **historique chronologique des actions** réalisées au sein de votre organisation, sur Studio et Vibe. Ils permettent aux administrateurs de suivre l’activité des utilisateurs, l’utilisation des clés API et les événements de sécurité.

Disponible avec les offres **Enterprise**. Activé par défaut pour tous les espaces de travail.

<SectionTab as="h1" sectionId="accessing">Accéder aux journaux d’audit</SectionTab>

Ouvrez le <AppLink href="https://admin.mistral.ai" app="admin">panneau d’administration</AppLink> puis cliquez sur **Journaux d’audit** dans le menu de gauche, ou accédez directement à <AppLink href="https://admin.mistral.ai/audit-logs" app="admin">Journaux d’audit</AppLink>.

Aucune configuration requise. Les journaux d’audit sont activés automatiquement dans les offres Enterprise.

<SectionTab as="h1" sectionId="what-is-logged">Contenu des journaux</SectionTab>

Les journaux d’audit enregistrent les actions effectuées par les utilisateurs et les clés API. Chaque entrée comprend :

- **Horodatage** (`CreatedAt`) : moment où l’événement a eu lieu.
- **Acteur** : entité ayant réalisé l’action (utilisateur humain, clé API ou système).
- **Événement** : action précise (ex. : `batch_job.create`, `le_chat.conversation.deleted`).
- **Cible** : ressource concernée (par ex. : batch job, conversation, clé API).
- **Métadonnées** : contexte supplémentaire sur l’acteur, l’événement et la cible.

Les activités consignées incluent :

- Événements d’authentification (connexions, processus SSO)
- Utilisation de l’API et gestion des clés (création, suppression)
- Modifications de l’organisation ou de l’espace de travail
- Actions de gestion des utilisateurs (invitations, changements de rôle, suppressions)
- Modifications des paramètres
- Activités sur Vibe (création et suppression de conversations)

<SectionTab as="h1" sectionId="search-and-filter">Recherche et filtrage</SectionTab>

Utilisez les filtres situés au-dessus du tableau pour affiner les résultats :

1. Filtrez par **acteur** (humain, clé API), type d’**événement** ou type de **cible**.
2. Combinez plusieurs critères pour retrouver des actions précises (ex. : toutes les suppressions de conversations par un utilisateur donné).
3. Personnalisez les colonnes affichées en cliquant sur **Colonnes** et en activant ou désactivant les champs souhaités.

Le tableau se met à jour automatiquement à chaque modification des filtres.

<SectionTab as="h1" sectionId="use-cases">Cas d’usage</SectionTab>

- **Surveillance de la sécurité** : détecter des accès non autorisés ou des comportements inhabituels.
- **Enquête d’incidents** : retrouver la séquence d’actions ayant conduit à un incident.
- **Conformité** : conserver un historique pour répondre aux exigences réglementaires.

<SectionTab as="h1" sectionId="limitations">Limites</SectionTab>

- L’export des journaux n’est pas disponible pour le moment.
- Les journaux d’audit sont accessibles uniquement avec Enterprise.
